# 客服系统客车注销费查询功能测试用例

## 1. UI功能展示测试

### 1.1 按钮展示基础功能测试

#### TC-001 客车注销状态按钮展示测试
- **前置条件**：用户已登录客服系统，进入客车模块，存在已注销状态的客车记录
- **测试步骤**：
  1. 进入客车模块页面
  2. 在车辆列表中选择状态为【注销】的客车记录
  3. 验证操作栏中是否显示【注销费查询】按钮
  4. 检查按钮的样式和位置是否与货车模块保持一致
  5. 验证按钮的可点击状态
- **预期结果**：选中注销状态的客车时，操作栏显示【注销费查询】按钮，按钮样式与货车模块一致，按钮可正常点击

#### TC-002 客车非注销状态按钮隐藏测试
- **前置条件**：用户已登录客服系统，进入客车模块，存在非注销状态的客车记录
- **测试步骤**：
  1. 进入客车模块页面
  2. 在车辆列表中选择状态为【正常】【锁定】【停用】等非注销状态的客车记录
  3. 验证操作栏中是否不显示【注销费查询】按钮
  4. 检查其他功能按钮的正常显示
- **预期结果**：选中非注销状态的客车时，操作栏不显示【注销费查询】按钮，其他功能按钮正常显示

#### TC-003 客车状态切换按钮动态显示测试
- **前置条件**：用户已登录客服系统，客车列表中同时存在注销和非注销状态的车辆
- **测试步骤**：
  1. 先选择非注销状态的客车记录，记录操作栏按钮状态
  2. 切换选择注销状态的客车记录，观察按钮变化
  3. 再次切换回非注销状态的客车记录，观察按钮变化
  4. 多次切换不同状态的客车记录，验证按钮显示的动态性
- **预期结果**：按钮根据选中车辆的状态动态显示和隐藏，切换响应及时，无显示延迟或错误

### 1.2 按钮交互功能测试

#### TC-004 注销费查询按钮点击响应测试
- **前置条件**：用户已选中注销状态的客车，【注销费查询】按钮已显示
- **测试步骤**：
  1. 点击【注销费查询】按钮
  2. 验证是否弹出查询结果弹窗或页面
  3. 检查弹窗/页面的标题和布局
  4. 验证加载状态的提示信息
  5. 检查查询请求是否正常发送
- **预期结果**：点击按钮后正常弹出查询结果界面，界面布局合理，有加载提示，查询请求正常发送

#### TC-005 注销费查询结果展示测试
- **前置条件**：用户点击【注销费查询】按钮，查询请求已发送
- **测试步骤**：
  1. 等待查询结果返回
  2. 验证查询结果的展示格式和内容
  3. 检查是否包含注销费用明细信息
  4. 验证金额显示的准确性和格式
  5. 检查时间信息的显示格式
  6. 验证车辆信息的关联显示
- **预期结果**：查询结果正确展示组合注销费用明细，包含金额、时间、车辆信息等关键字段，数据格式正确

#### TC-006 查询结果弹窗交互测试
- **前置条件**：注销费查询结果已正常展示
- **测试步骤**：
  1. 验证弹窗的关闭功能（X按钮、取消按钮、点击空白区域）
  2. 检查弹窗内容的滚动功能（如果内容较多）
  3. 验证弹窗的大小调整功能（如果支持）
  4. 测试弹窗内的其他交互元素（如复制、导出等）
- **预期结果**：弹窗交互功能正常，关闭方式多样化，内容展示友好，交互体验良好

## 2. 权限控制功能测试

### 2.1 权限配置一致性测试

#### TC-007 客车注销费查询权限配置测试
- **前置条件**：系统中已配置货车注销费查询权限，需要为客车配置相同权限
- **测试步骤**：
  1. 进入权限管理后台
  2. 查找货车注销费查询权限的配置项
  3. 验证客车注销费查询权限配置项是否存在
  4. 对比两个权限配置的字段和规则
  5. 检查权限继承关系和依赖关系
- **预期结果**：客车注销费查询权限配置与货车保持一致，权限名称、编码、规则等参数匹配

#### TC-008 权限角色分配测试
- **前置条件**：已配置客车注销费查询权限，存在不同角色的用户账号
- **测试步骤**：
  1. 为测试角色分配客车注销费查询权限
  2. 为对照角色不分配该权限
  3. 验证权限分配结果的正确保存
  4. 检查权限分配的生效时间
  5. 验证权限继承机制（如果存在上下级角色）
- **预期结果**：权限角色分配功能正常，权限状态正确保存，分配生效及时

### 2.2 权限验证功能测试

#### TC-009 有权限用户访问测试
- **前置条件**：用户账号已分配客车注销费查询权限
- **测试步骤**：
  1. 使用有权限的用户账号登录
  2. 进入客车模块，选择注销状态的车辆
  3. 验证【注销费查询】按钮是否正常显示
  4. 点击按钮验证功能是否可正常使用
  5. 检查查询结果是否能正常获取和展示
- **预期结果**：有权限用户能够看到并正常使用注销费查询功能，所有操作无权限限制

#### TC-010 无权限用户访问测试
- **前置条件**：用户账号未分配客车注销费查询权限
- **测试步骤**：
  1. 使用无权限的用户账号登录
  2. 进入客车模块，选择注销状态的车辆
  3. 验证【注销费查询】按钮是否被隐藏或禁用
  4. 尝试通过其他方式访问注销费查询功能
  5. 检查是否有相应的权限提示信息
- **预期结果**：无权限用户无法看到或使用注销费查询功能，按钮被隐藏或禁用，有明确的权限提示

#### TC-011 权限变更即时生效测试
- **前置条件**：用户当前拥有客车注销费查询权限，且正在使用系统
- **测试步骤**：
  1. 用户正常使用注销费查询功能
  2. 管理员撤销该用户的注销费查询权限
  3. 用户刷新页面或重新操作
  4. 验证权限变更是否立即生效
  5. 检查系统的权限提示信息
- **预期结果**：权限变更立即生效，用户再次操作时无法使用该功能，有合适的权限变更提示

### 2.3 权限边界测试

#### TC-012 权限依赖关系测试
- **前置条件**：客车注销费查询可能依赖其他权限（如客车查看权限）
- **测试步骤**：
  1. 分析注销费查询功能的权限依赖关系
  2. 为用户分配注销费查询权限，但不分配依赖权限
  3. 验证功能是否能正常使用
  4. 检查权限冲突的处理机制
  5. 验证权限提示信息的准确性
- **预期结果**：权限依赖关系清晰，缺少依赖权限时有明确提示，权限冲突处理合理

#### TC-013 权限级别测试
- **前置条件**：可能存在不同级别的注销费查询权限（如只读、完整等）
- **测试步骤**：
  1. 为用户分配不同级别的权限
  2. 验证各级别权限对应的功能范围
  3. 检查权限级别的升级和降级效果
  4. 验证权限级别的优先级规则
- **预期结果**：不同级别权限功能范围明确，权限变更效果正确，优先级规则合理
