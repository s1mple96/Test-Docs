# 任通行通过MQ推送账单到新发行ETC逻辑优化测试用例

## 1. 需求背景分析

### 1.1 需求概述
- **发现问题**: 20240802发现预警群内蒙账单与运营商不一致
- **根本原因**: 任通行异步发送账单到MQ服务时出现异常，导致账单未推送成功
- **核心问题**: 阿里云MQ对uncheck消息有最大投递次数限制和消费时长限制，导致消息在超过最大投递次数后被丢弃

### 1.2 优化方案
1. **MQ异常预警**: MQ消息如果没有正常推送到阿里云服务，进行报警
2. **账单存储优化**: 客车新发行账单增加单独表结构存储，增加推送状态相关维护
3. **回调机制**: 新发行账单落库增加相关回调，回调业务方确认消息
4. **补偿机制**: 客车新发行账单需要增加定时任务补偿，对于超过时间未收到新发行回调账单进行重新推送

### 1.3 涉及系统
- **发版项目**: rtx-quartz, rtx-app, fenmi-etc-job
- **数据库**: rtx库和fenmi_etc库
- **MQ系统**: RabbitMQ (192.168.1.58:5673)
- **监控工具**: Arthas (192.168.1.60服务器)

### 2.3 关键表结构

#### 2.3.1 账单表 (fenmi_etc.etc_deduct_bill)
```sql
- id: bigint(32) PRIMARY KEY
- bill_date: varchar(12) 账单日期
- operator_code: varchar(10) 运营商代码 (TXB=任通行)
- bill_amt: decimal(13,2) 账单金额
- bill_status: int(1) 账单状态 (0=待推送)
- pay_status: int(1) 支付状态 (0=未支付)
```

#### 2.3.2 MQ错误日志表 (rtx.sys_mq_error_log) - 优化后
```sql
- id: bigint(20) PRIMARY KEY 主键id
- biz_id: varchar(64) 业务id
- correlation_id: varchar(64) 投递id
- msg: text 消息内容
- msg_type: int 发送类型 (0=字符串, 1=对象)
- class_name: varchar(128) 全类名
- routing_key: varchar(100) 路由键
- status: int 状态
- send_time: datetime 最近一次发送时间
- retry_max_count: int 最大重试次数
```
