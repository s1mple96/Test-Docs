# 【任通行】无扣款方式下黑功能测试用例

## 1. 无扣款方式下黑功能测试

### 1.1 下黑触发条件验证测试

#### TC-001 钱包模式用户不触发下黑测试
- **前置条件**：用户为钱包2.0代扣模式，且账户状态为正常或待激活
- **测试步骤**：
  1. 创建钱包2.0代扣模式的测试账户
  2. 设置账户为无有效银行卡和无微信代扣状态
  3. 执行定时任务ID231（无扣款方式下黑任务）
  4. 查询该账户车辆的黑名单状态
  5. 验证是否触发无扣款方式下黑
- **预期结果**：钱包2.0代扣模式用户不会被标记为无扣款方式下黑，车辆保持原有状态

#### TC-002 有有效银行卡用户不触发下黑测试
- **前置条件**：用户有至少一张有效的绑定银行卡，但未开通微信代扣
- **测试步骤**：
  1. 创建测试账户并绑定有效银行卡
  2. 设置账户为非钱包2.0代扣模式
  3. 执行定时任务ID231
  4. 查询该账户车辆的黑名单状态
  5. 检查系统日志中的判断逻辑
- **预期结果**：有有效银行卡的用户不会被标记为无扣款方式下黑，系统正常放行

#### TC-003 已开通微信代扣用户不触发下黑测试
- **前置条件**：用户车辆已开通微信代扣，但无有效银行卡
- **测试步骤**：
  1. 创建测试账户并开通车辆微信代扣
  2. 设置账户为无有效银行卡状态
  3. 执行定时任务ID231
  4. 查询该账户车辆的黑名单状态
  5. 验证微信代扣是否作为有效扣款方式
- **预期结果**：已开通微信代扣的车辆不会被标记为无扣款方式下黑，正常使用

#### TC-004 正常用户无扣款方式触发下黑测试
- **前置条件**：用户非钱包2.0代扣模式，无有效银行卡，未开通微信代扣，车辆状态为正常
- **测试步骤**：
  1. 创建符合下黑条件的测试账户
  2. 确认车辆状态为正常且非黑名单
  3. 执行定时任务ID231
  4. 查询该账户车辆的黑名单状态和类型
  5. 检查数据库中黑名单类型标记
- **预期结果**：车辆被成功标记为无扣款方式下黑，黑名单类型正确记录

#### TC-005 待激活用户无扣款方式触发下黑测试
- **前置条件**：用户非钱包2.0代扣模式，无有效银行卡，未开通微信代扣，车辆状态为待激活
- **测试步骤**：
  1. 创建车辆状态为待激活的测试账户
  2. 确认无任何有效扣款方式
  3. 执行定时任务ID231
  4. 查询该账户车辆的黑名单状态和类型
  5. 验证待激活状态是否支持下黑
- **预期结果**：待激活状态车辆被成功标记为无扣款方式下黑

#### TC-006 已黑名单用户不触发无扣款方式下黑测试
- **前置条件**：用户车辆已经是黑名单状态（欠费黑名单或其他黑名单类型）
- **测试步骤**：
  1. 创建已在黑名单中的测试账户
  2. 设置账户为无有效扣款方式
  3. 执行定时任务ID231
  4. 查询该账户车辆的黑名单类型
  5. 验证黑名单类型是否被覆盖
- **预期结果**：已有黑名单状态不被覆盖，保持原有黑名单类型，无扣款方式下黑不生效

### 1.2 黑名单优先级和覆盖关系测试

#### TC-007 欠费下黑覆盖无扣款方式下黑测试
- **前置条件**：用户车辆当前为无扣款方式下黑状态，且产生了欠费账单
- **测试步骤**：
  1. 创建无扣款方式下黑的测试账户
  2. 为该账户创建欠费账单
  3. 执行欠费下黑定时任务
  4. 查询该账户车辆的黑名单类型变化
  5. 验证黑名单类型是否从无扣款方式下黑变更为欠费下黑
- **预期结果**：欠费下黑成功覆盖无扣款方式下黑，黑名单类型更新为欠费黑名单

#### TC-008 无扣款方式下黑不覆盖欠费黑名单测试
- **前置条件**：用户车辆当前为欠费黑名单状态，且扣款方式已失效
- **测试步骤**：
  1. 创建欠费黑名单的测试账户
  2. 删除该账户的所有有效扣款方式
  3. 执行定时任务ID231
  4. 查询该账户车辆的黑名单类型
  5. 验证黑名单类型是否保持为欠费黑名单
- **预期结果**：欠费黑名单状态保持不变，无扣款方式下黑不会覆盖

#### TC-009 无扣款方式下黑不覆盖其他类型黑名单测试
- **前置条件**：用户车辆为其他类型黑名单（如违规黑名单、风控黑名单等）
- **测试步骤**：
  1. 创建其他类型黑名单的测试账户
  2. 设置账户为无有效扣款方式
  3. 执行定时任务ID231
  4. 查询该账户车辆的黑名单类型
  5. 确认黑名单类型是否被保留
- **预期结果**：其他类型黑名单状态保持不变，无扣款方式下黑不会覆盖

### 1.3 小程序消息推送内容测试

#### TC-010 无扣款方式下黑消息内容验证测试
- **前置条件**：用户账户满足无扣款方式下黑条件，且小程序消息推送功能正常
- **测试步骤**：
  1. 创建符合下黑条件的测试账户（无有效银行卡、未开通微信代扣）
  2. 执行定时任务ID231使车辆下黑
  3. 查询小程序消息推送记录表或消息队列
  4. 获取推送的消息内容
  5. 验证消息内容是否为"账号无有效扣款方式，车辆已下黑"
- **预期结果**：下黑成功后发送小程序消息，消息内容准确为"账号无有效扣款方式，车辆已下黑"

#### TC-011 欠费下黑消息内容验证测试
- **前置条件**：用户账户产生欠费并触发欠费下黑
- **测试步骤**：
  1. 创建测试账户并产生欠费账单
  2. 执行欠费下黑任务
  3. 查询小程序消息推送记录表或消息队列
  4. 获取推送的消息内容
  5. 验证消息内容是否为"账号已欠费，车辆已下黑"
- **预期结果**：欠费下黑发送的消息内容准确为"账号已欠费，车辆已下黑"

#### TC-012 两种下黑类型消息内容对比测试
- **前置条件**：系统支持欠费下黑和无扣款方式下黑两种类型
- **测试步骤**：
  1. 准备两个测试账户，一个触发欠费下黑，一个触发无扣款方式下黑
  2. 分别执行对应的下黑任务
  3. 查询两个账户的小程序消息推送记录
  4. 对比两条消息的内容差异
  5. 验证消息内容是否符合各自的下黑类型
- **预期结果**：
  - 欠费下黑：消息内容为"账号已欠费，车辆已下黑"
  - 无扣款方式下黑：消息内容为"账号无有效扣款方式，车辆已下黑"
  - 两种消息内容明确区分，不会混淆

#### TC-013 欠费下黑覆盖无扣款方式下黑后的消息推送测试
- **前置条件**：用户车辆当前为无扣款方式下黑状态，产生欠费后触发欠费下黑
- **测试步骤**：
  1. 创建无扣款方式下黑的测试账户
  2. 为该账户创建欠费账单
  3. 执行欠费下黑任务，覆盖原有的无扣款方式下黑
  4. 查询小程序消息推送记录
  5. 验证推送的消息内容是否为欠费下黑的内容
- **预期结果**：欠费下黑覆盖无扣款方式下黑时，推送的消息内容为"账号已欠费，车辆已下黑"

#### TC-014 消息推送时机验证测试
- **前置条件**：用户账户触发下黑条件
- **测试步骤**：
  1. 创建测试账户并触发无扣款方式下黑
  2. 记录下黑操作的执行时间
  3. 查询小程序消息推送的发送时间
  4. 对比两个时间的间隔
  5. 验证消息是否在下黑成功后立即发送
- **预期结果**：下黑成功后立即（1分钟内）发送小程序消息通知

#### TC-015 消息推送成功状态验证测试
- **前置条件**：用户账户触发下黑，且用户已绑定小程序
- **测试步骤**：
  1. 创建已绑定小程序的测试账户
  2. 执行定时任务ID231触发无扣款方式下黑
  3. 查询消息推送记录表
  4. 检查消息发送状态字段
  5. 验证消息是否成功送达
- **预期结果**：消息推送状态为"成功"，消息内容为"账号无有效扣款方式，车辆已下黑"

#### TC-016 消息推送失败处理测试
- **前置条件**：用户未绑定小程序或小程序推送服务异常
- **测试步骤**：
  1. 创建未绑定小程序的测试账户
  2. 执行定时任务ID231触发无扣款方式下黑
  3. 查询消息推送记录表
  4. 检查消息发送状态和失败原因
  5. 验证是否有重试机制或补偿措施
- **预期结果**：消息推送失败时有明确的失败状态和原因记录，支持重试或人工补发

#### TC-017 多次下黑消息推送测试
- **前置条件**：用户账户多次触发下黑（如从解黑后再次下黑）
- **测试步骤**：
  1. 创建测试账户并触发无扣款方式下黑
  2. 为该账户添加有效扣款方式并解黑
  3. 再次删除扣款方式触发无扣款方式下黑
  4. 查询小程序消息推送记录
  5. 验证每次下黑是否都发送消息
- **预期结果**：每次触发无扣款方式下黑都发送消息通知，消息内容均为"账号无有效扣款方式，车辆已下黑"

## 2. 无扣款方式解黑功能测试

### 2.1 解黑触发条件验证测试

#### TC-013 新增有效银行卡自动解黑测试
- **前置条件**：用户车辆为无扣款方式下黑状态，且名下无欠款账单
- **测试步骤**：
  1. 创建无扣款方式下黑的测试账户
  2. 确认该账户名下无欠款账单
  3. 为该账户新增绑定有效银行卡
  4. 执行钱包代扣2.0解黑定时任务
  5. 查询该账户车辆的黑名单状态
- **预期结果**：新增有效银行卡后，车辆自动解黑，黑名单状态清除

#### TC-014 开通微信代扣自动解黑测试
- **前置条件**：用户车辆为无扣款方式下黑状态，且名下无欠款账单
- **测试步骤**：
  1. 创建无扣款方式下黑的测试账户
  2. 确认该账户名下无欠款账单
  3. 为该车辆开通微信代扣
  4. 执行钱包代扣2.0解黑定时任务
  5. 查询该账户车辆的黑名单状态
- **预期结果**：开通微信代扣后，车辆自动解黑，黑名单状态清除

#### TC-015 有欠款账单时不自动解黑测试
- **前置条件**：用户车辆为无扣款方式下黑状态，但名下存在欠款账单
- **测试步骤**：
  1. 创建无扣款方式下黑的测试账户
  2. 为该账户创建未结清的欠款账单
  3. 为该账户新增有效扣款方式（银行卡或微信代扣）
  4. 执行钱包代扣2.0解黑定时任务
  5. 查询该账户车辆的黑名单状态
- **预期结果**：虽然新增了有效扣款方式，但因存在欠款账单，车辆不会自动解黑

#### TC-016 非无扣款方式下黑类型不触发解黑逻辑测试
- **前置条件**：用户车辆为其他类型黑名单（如欠费黑名单），且新增了有效扣款方式
- **测试步骤**：
  1. 创建欠费黑名单的测试账户
  2. 为该账户新增有效扣款方式
  3. 执行钱包代扣2.0解黑定时任务
  4. 查询该账户车辆的黑名单状态
  5. 验证解黑逻辑是否只针对无扣款方式下黑类型
- **预期结果**：欠费黑名单等其他类型不会通过此逻辑解黑，需走各自的解黑流程

### 2.2 解黑后状态验证测试

#### TC-017 解黑后车辆状态恢复验证测试

#### TC-018 解黑后再次满足下黑条件的测试
