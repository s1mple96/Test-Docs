# 【任货行】诉讼还款注销测试用例

> 关联需求：【任货行清欠用户快速还款】
> 前置条件：用户通过清欠快速还款成功后触发
> **重要变更：采用延迟MQ通知注销（延迟1小时）**

## 1. 注销订单创建测试

### 1.1 自动创建注销订单

#### TC-001 单车辆用户-延迟1小时自动创建注销订单
- **前置条件**: 
  - 系统: 任货行小程序
  - 用户名下车辆: 1辆(粤B12345)
  - 钱包标签: PURGE_Wallet
  - 钱包余额: -2000
  - 完成清欠还款支付成功
- **测试步骤**:
  1. 用户完成清欠快速还款(记录时间T)
  2. 立即查询hcb_hlj_cancelbussine表
  3. 等待30分钟后查询
  4. **等待1小时后查询**
- **预期结果**: 
  - 时间T: 注销订单未创建
  - 时间T+30分钟: 注销订单未创建
  - **时间T+1小时: 注销订单已创建**
  - BUSSTYPE=诉讼还款注销对应值
  - PAYMENT_STATUS=已支付
  - PAY_AMT=0
  - PAYMENT_METHOD=无需支付

#### TC-002 多车辆用户-延迟1小时批量创建注销订单
- **前置条件**: 
  - 系统: 任货行小程序
  - 用户名下车辆: 3辆(粤B12345、粤B67890、粤A11111)
  - 完成清欠还款支付成功
- **测试步骤**:
  1. 用户完成清欠快速还款
  2. **等待1小时后查询hcb_hlj_cancelbussine表**
- **预期结果**: 
  - 1小时后自动创建3条注销订单
  - 每辆车各一条
  - 所有订单BUSSTYPE=诉讼还款注销
  - 所有订单创建时间≈还款时间+1小时

#### TC-003 部分车辆已注销-只为未注销车辆创建订单
- **前置条件**: 
  - 系统: 任货行小程序
  - 用户名下车辆: 3辆

  2. 查询注销订单
- **预期结果**:

#### TC-004 车辆已有进行中注销订单-不重复创建
