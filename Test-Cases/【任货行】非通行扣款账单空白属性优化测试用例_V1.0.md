## 1. 新增生成逻辑优化测试（MQ监听器）

### 1.1 三月前历史账单扣款成功场景

#### TC-001 三月前账单扣款成功 - 历史归档账单能正确填充 truckuser_id 和 bill_id
- **前置条件**:
  - 系统: 任货行定时任务（hcb-timer）
  - 数据库: hcb
  - 场景: 通行账单已归档到历史表（三月前数据），MQ 扣款成功消息触发
  - 测试账单: 选取一条 `deduction_time` 在三月前、历史备份表（MongoDB: `HCB_HEIDEDUCRECORD_BACKUP`）中存在对应记录的账单
  - 测试环境: 192.168.1.60
- **测试步骤**:
  1. 确认该账单的 `deductionExtend` 对象中 `truckUserId` 和 `deductBillNo` 均为空
  2. 触发扣款成功（或通过定时任务 `deductionHisFailtTask` 自然触发）
  3. MQ 消息被 `DeductBillSuccessRabbitMqListener` 处理
  4. 查询 `hcb_nonpassabledeductionbillflow` 对应记录的 `truckuser_id` 和 `bill_id`
- **预期结果**: 新生成的非通行扣款账单记录中 `truckuser_id` 和 `bill_id` 均不为空，值来源于历史备份表

#### TC-002 三月前账单扣款成功 - 历史备份表中无对应记录
- **前置条件**:
  - 系统: 任货行定时任务（hcb-timer）
  - 数据库: hcb / MongoDB
  - 场景: 通行账单已归档，但历史备份表（MongoDB）中无该 `deductionId` 对应记录
  - 测试环境: 192.168.1.60
- **测试步骤**:
  1. 构造一条历史备份表中不存在对应记录的扣款成功场景
  2. 触发 MQ 扣款成功消息处理
  3. 查询 `hcb_nonpassabledeductionbillflow` 对应记录
  4. 查看日志 `/home/tomcat/tomcat-hcbTimer-bill/logs/catalina.out`
- **预期结果**: 日志中出现 `填充非通行扣款账单,查无历史通行账单信息` 报错日志；新生成的非通行扣款账单记录仍缺失用户信息（非阻断性异常，不影响其他账单处理）

#### TC-003 三月内（近期）账单扣款成功 - 通行账单未归档，走正常逻辑
- **前置条件**:
  - 系统: 任货行定时任务（hcb-timer）
  - 数据库: hcb
  - 场景: 通行账单在三月内，通行账单表（`hcb_heideducrecord`）中存在对应记录
  - 测试环境: 192.168.1.60
- **测试步骤**:
  1. 选取 `deduction_time` 在近三月内的扣款失败重试账单
  2. 触发扣款成功
  3. 查询生成的 `hcb_nonpassabledeductionbillflow` 记录
- **预期结果**: `truckuser_id` 和 `bill_id` 均正确填充，与通行账单表中的值一致；原有逻辑不受影响
