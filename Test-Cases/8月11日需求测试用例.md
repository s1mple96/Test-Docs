## 1. 新发行ETC系统账单扣款优化测试

### 1.1 分页加载处理功能测试

#### TC-001 首次扣款任务分页处理测试
- **前置条件**：系统中存在大量未扣款账单数据（>1000条）
- **测试步骤**：
  1. 启动新发行ETC定时任务fenmi-etc-job项目
  2. 触发首次扣款定时任务
  3. 监控任务执行过程，观察内存使用情况
  4. 检查任务日志，验证是否按分页（每页2条）处理
  5. 验证所有未扣款账单是否被正确处理
- **预期结果**：任务按分页处理，内存使用稳定，不会发生内存溢出，所有账单正确扣款

#### TC-002 扣款失败账单重新扣款分页处理测试
- **前置条件**：系统中存在大量扣款失败的账单数据（>1000条）
- **测试步骤**：
  1. 准备扣款失败状态的账单数据
  2. 启动扣款失败重新扣款定时任务
  3. 监控任务执行过程和内存使用情况
  4. 检查任务日志，验证分页查询是否正确（每页2条）
  5. 验证所有失败账单是否被重新处理
- **预期结果**：失败账单重新扣款任务按分页执行，内存稳定，避免服务挂掉

#### TC-003 分页查询逻辑正确性测试
- **前置条件**：测试环境配置每页查询2条数据
- **测试步骤**：
  1. 准备10条测试账单数据
  2. 执行首次扣款或失败重新扣款任务
  3. 查看任务执行日志，验证分页查询日志输出
  4. 确认每次查询确实只处理2条数据
  5. 验证总共执行5次分页查询
- **预期结果**：分页查询逻辑正确，每页查询2条数据，总共5页，日志输出正确

#### TC-004 分页处理避免死循环测试
- **前置条件**：系统中存在各种状态的账单数据
- **测试步骤**：
  1. 准备包含已扣款、未扣款、扣款失败等各种状态的账单
  2. 执行定时扣款任务
  3. 监控任务执行时间和循环次数
  4. 验证任务是否会因为数据状态变化而死循环
  5. 检查任务最终是否正常结束
- **预期结果**：任务正常结束，不会出现死循环，分页处理逻辑正确

### 1.2 扣款功能回归测试

#### TC-005 首次扣款功能正常性测试
- **前置条件**：系统中存在未扣款的账单

#### TC-006 失败扣款重试功能正常性测试
- **前置条件**：系统中存在扣款失败的账单
