# 任货行任通行账单推送MQ回调处理测试用例

## 1. 账单推送MQ服务功能测试

### 1.1 正常推送流程测试

#### TC-001 任通行账单推送MQ正常流程测试
- **前置条件**：任通行系统正常运行，MQ服务可用，新发行ETC系统正常
- **测试步骤**：
  1. 准备内蒙账单数据，设置账单状态为待推送
  2. 任通行系统调用MQ服务推送账单
  3. 验证MQ消息发送成功
  4. 验证新发行ETC系统接收并处理消息
  5. 检查账单在新发行ETC系统中的落库情况
- **预期结果**：账单成功推送到MQ，新发行ETC系统成功接收并落库，数据一致性得到保证

#### TC-002 批量账单推送测试
- **前置条件**：存在多笔内蒙账单需要推送
- **测试步骤**：
  1. 准备10-20笔内蒙账单数据
  2. 任通行系统批量推送账单到MQ
  3. 验证所有账单都成功发送到MQ
  4. 检查新发行ETC系统接收处理情况
  5. 对比推送前后的账单数量和内容
- **预期结果**：所有账单都成功推送，新发行ETC系统完整接收，无遗漏无重复

#### TC-003 不同运营商账单推送测试
- **前置条件**：存在不同运营商的账单数据
- **测试步骤**：
  1. 准备内蒙、其他省份的账单数据
  2. 验证内蒙账单的推送逻辑
  3. 验证其他省份账单的处理逻辑
  4. 检查运营商标识的正确性
  5. 验证推送目标的准确性
- **预期结果**：内蒙账单正确推送到新发行ETC系统，其他运营商账单按原逻辑处理

### 1.2 推送异常场景测试

#### TC-004 MQ服务不可用时推送测试
- **前置条件**：模拟MQ服务不可用或网络异常

- **预期结果**：系统能正确处理MQ不可用异常，记录错误日志，账单状态标记为推送失败

#### TC-005 推送超时处理测试
- **前置条件**：模拟网络延迟导致推送超时
- **测试步骤**：
  1. 设置较短的推送超时时间

  1. 推送账单到MQ

  5. 确认最终的数据一致性处理
- **预期结果**：系统能处理回调超时，有主动查询或重推机制，确保数据最终一致

## 3. 数据一致性保障测试
