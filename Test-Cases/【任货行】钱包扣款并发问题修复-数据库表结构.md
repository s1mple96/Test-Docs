# 【任货行】钱包扣款并发问题修复 - 涉及数据库表

## 核心业务表

### 1. 钱包表：hcb_monthly_paylater_wallet

**表名**：`hcb.hcb_monthly_paylater_wallet`

**字段说明**：
| 字段名 | 类型 | 说明 |
|-----|-----|-----|
| id | bigint(32) | 主键ID |
| wallet_id | varchar(64) | 钱包ID（唯一索引） |
| id_code | varchar(32) | 身份证号 |
| user_name | varchar(32) | 用户名 |
| blance | decimal(13,2) | 钱包余额 |
| create_time | datetime | 创建时间 |
| update_time | datetime | 更新时间 |

**核心字段**：
- `wallet_id`：钱包唯一标识，用于加锁
- `blance`：钱包余额，并发操作的核心字段

**核心字段**：
- `wallet_id`：关联钱包
- `trade_amount`：交易金额

**数据一致性验证公式**：
```
钱包余额 = 初始余额 + SUM(充值金额) - SUM(扣款金额)
surplus_amount = before_amount ± trade_amount
```

## 测试相关查询SQL

### 1. 查询钱包余额
```sql
SELECT wallet_id, user_name, blance, update_time 
FROM hcb.hcb_monthly_paylater_wallet 
WHERE wallet_id = '钱包ID';
```

### 2. 查询钱包流水记录
```sql
SELECT 
  id, wallet_id, biz_type, trade_amount, 
  before_amount, surplus_amount, fund_direction, 
  order_no, create_time 
FROM hcb.hcb_monthly_paylater_wallet_flow 
WHERE wallet_id = '钱包ID' 
ORDER BY create_time DESC;
```

### 3. 统计钱包流水总额
```sql
SELECT 
  wallet_id,
  SUM(CASE WHEN fund_direction = 'IN' THEN trade_amount ELSE 0 END) AS total_in,
  SUM(CASE WHEN fund_direction = 'OUT' THEN trade_amount ELSE 0 END) AS total_out,
  COUNT(*) AS flow_count
FROM hcb.hcb_monthly_paylater_wallet_flow 
WHERE wallet_id = '钱包ID' 
GROUP BY wallet_id;
```

### 4. 验证余额与流水一致性
```sql
SELECT 
  w.wallet_id,
  w.blance AS current_balance,
  IFNULL(SUM(CASE WHEN f.fund_direction = 'IN' THEN f.trade_amount ELSE 0 END), 0) AS total_in,
  IFNULL(SUM(CASE WHEN f.fund_direction = 'OUT' THEN f.trade_amount ELSE 0 END), 0) AS total_out,
  (IFNULL(SUM(CASE WHEN f.fund_direction = 'IN' THEN f.trade_amount ELSE 0 END), 0) - 
   IFNULL(SUM(CASE WHEN f.fund_direction = 'OUT' THEN f.trade_amount ELSE 0 END), 0)) AS flow_balance
FROM hcb.hcb_monthly_paylater_wallet w
LEFT JOIN hcb.hcb_monthly_paylater_wallet_flow f ON w.wallet_id = f.wallet_id
WHERE w.wallet_id = '钱包ID'
GROUP BY w.wallet_id, w.blance;
```

### 5. 查询待扣款账单
```sql
SELECT 
  id, wallet_id, bill_amount, bill_status, 
  bill_date, create_time 
FROM hcb.hcb_monthly_paylater_wallet_bill 
WHERE wallet_id = '钱包ID' 
  AND bill_status = 待扣款状态值 
ORDER BY create_time;
```

### 6. 查询指定时间段内的流水
```sql
SELECT 
  wallet_id, biz_type, trade_amount, 
  before_amount, surplus_amount, fund_direction, 
  create_time 
FROM hcb.hcb_monthly_paylater_wallet_flow 
WHERE wallet_id = '钱包ID' 
  AND create_time >= '2025-11-25 00:00:00' 
  AND create_time <= '2025-11-25 23:59:59' 
ORDER BY create_time;
```

### 7. 查询异常流水（余额计算不匹配）
```sql
SELECT 
  id, wallet_id, trade_amount, before_amount, 
  surplus_amount, fund_direction,
  CASE 
    WHEN fund_direction = 'IN' THEN before_amount + trade_amount
    WHEN fund_direction = 'OUT' THEN before_amount - trade_amount
  END AS expected_balance,
  create_time
FROM hcb.hcb_monthly_paylater_wallet_flow 
WHERE wallet_id = '钱包ID'
  AND surplus_amount != (
    CASE 
      WHEN fund_direction = 'IN' THEN before_amount + trade_amount
      WHEN fund_direction = 'OUT' THEN before_amount - trade_amount
    END
  )
ORDER BY create_time;
```

## 测试验证要点

### 1. 余额一致性验证
- 钱包余额 = 初始余额 + 充值总额 - 扣款总额
- 流水记录的 `surplus_amount` 必须连续正确

### 2. 流水完整性验证
- 每笔扣款/充值操作必须有对应的流水记录
- 流水记录的 `before_amount` 和 `surplus_amount` 必须连贯

### 3. 并发数据验证
- 同时发起的多个操作，流水记录不能丢失
- `before_amount` 不能相同（说明并发读取了相同的旧值）
- 最终余额 = 初始余额 + 所有流水计算结果

### 4. 账单状态验证
- 扣款成功的账单状态必须更新
- 账单的 `pay_flow_id` 必须对应钱包流水记录
