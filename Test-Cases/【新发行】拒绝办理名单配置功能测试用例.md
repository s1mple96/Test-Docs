- **预期结果**：列表正常显示，包含身份证号、拒办业务线、拒办产品、拒办所属渠道、拒办提示、备注、操作等字段

#### TC-002 拒办名单列表搜索筛选测试
- **前置条件**：系统中有多条不同类型的拒办名单
- **测试步骤**：
  1. 使用身份证号进行精确搜索
  2. 按拒办业务线进行筛选（内蒙古ETC服务/江苏通行宝ETC服务）

  5. 验证搜索结果的准确性
- **预期结果**：搜索筛选功能正常，能准确返回符合条件的拒办名单

#### TC-003 拒办名单列表排序测试
- **前置条件**：系统中有多条拒办名单数据
- **测试步骤**：
  1. 按创建时间排序（升序/降序）
  2. 按身份证号排序

  4. 验证排序结果的正确性
  5. 检查排序性能
- **预期结果**：排序功能正常，结果正确，性能满足要求

### 1.2 新增拒办名单测试

#### TC-004 单条拒办名单新增正常流程测试
- **前置条件**：具有拒办名单管理权限
- **测试步骤**：
  1. 点击"新增"按钮
  2. 填写身份证号（18位有效身份证号）
  3. 选择拒办业务线（MTK-内蒙古ETC服务，TXB-江苏通行宝ETC服务）
  4. 填写拒办产品ID（多个用英文逗号隔开）
  5. 填写拒办所属渠道ID（多个用英文逗号隔开）
  6. 填写拒办提示信息
  7. 填写备注信息
  8. 点击保存
- **预期结果**：拒办名单新增成功，数据保存正确，列表中显示新增记录

#### TC-005 拒办名单字段校验测试
- **前置条件**：进入新增拒办名单页面
- **测试步骤**：
  1. 测试身份证号格式校验（15位/18位/无效格式）
  2. 测试拒办业务线必填校验
  3. 测试拒办产品ID格式校验（数字、逗号分隔）
  4. 测试拒办渠道ID格式校验（数字、逗号分隔）
  5. 测试拒办提示必填校验
  6. 测试备注字段长度限制
- **预期结果**：字段校验正确，无效数据无法保存，有明确的错误提示

#### TC-006 重复身份证号新增校验测试
- **前置条件**：系统中已存在某身份证号的拒办记录
- **测试步骤**：
  1. 尝试新增相同身份证号的拒办名单
  2. 选择相同的拒办业务线
  3. 验证系统的重复校验机制
  4. 检查错误提示信息
  5. 确认数据不会重复保存
- **预期结果**：系统正确识别重复记录，给出明确提示，不允许重复新增
