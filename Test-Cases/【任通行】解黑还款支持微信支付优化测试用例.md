# 任通行解黑还款支持微信支付优化测试用例

## 1. 测试概述

### 1.1 测试目的
验证任通行解黑还款功能新增微信支付方式的完整性、准确性和稳定性，确保数据一致性和业务流程正确性。

### 1.2 测试范围
- **涉及项目**：rtx-admin、rtx-quartz、rtx-app、report-admin、fenmi-pay
- **核心功能**：微信支付还款、数据表更新、定时任务处理、回调处理
- **关键数据表**：
  - `rtx_etc_deducrecord` (账单表)
  - `rtx_deduct_record_extend` (账单扩展表)  
  - `rtx_user_wallet` (钱包表)
  - `rtx_etc_balance_flow` (钱包流水表)
  - `rtx_deduct_record_repayment` (批量还款明细表)

### 1.3 测试重点
1. **数据关注点**：主动还款后核对5张关键表数据准确性
2. **成功失败场景**：支付成功、失败的完整流程测试
3. **重复支付防护**：同一笔账单多次支付的异常处理
4. **定时任务影响**：任通行定时任务和微信回调的业务逻辑
5. **流水重复处理**：防止同一订单重复加钱的机制
6. **扣款任务适配**：现有扣款任务对微信支付账单的处理
7. **钱包模式影响**：1.0/2.0钱包模式对微信支付的影响
8. **扣款方式影响**：代扣/余额扣款方式对微信支付的影响

### 3.2 数据关联性验证测试

#### TC-011 多表数据关联一致性测试
- **前置条件**：完成微信支付还款操作
- **测试步骤**：
  1. 以账单ID为关联键查询所有相关表数据
  2. 验证账单表与扩展表数据一致性
  3. 验证流水表记录与账单金额匹配
  4. 验证还款明细与账单记录对应
  5. 检查时间戳的逻辑合理性
- **预期结果**：多表数据关联正确，无数据不一致情况
- **验证脚本**：
```sql
-- 数据一致性检查
SELECT 
  d.id as deduct_id,
  d.pay_type as deduct_pay_type,
  d.status as deduct_status,
  e.pay_type as extend_pay_type,
  e.pay_status as extend_pay_status,
  COUNT(f.id) as flow_count,
  r.pay_status as repay_status
FROM rtx_etc_deducrecord_ltk d
LEFT JOIN rtx_deduct_record_extend_ltk e ON d.id = e.deduct_bill_id
LEFT JOIN rtx_etc_balance_flow f ON d.id = f.source_id
LEFT JOIN rtx_deduct_record_repayment r ON d.id = r.deduct_record_id
WHERE d.id = {测试账单ID}
GROUP BY d.id
```

```sql
FROM rtx_deduct_record_repayment 
WHERE deduct_record_id = {测试账单ID};

-- 综合数据一致性检查
SELECT 
  d.id as deduct_id,
  d.plate_num,
  d.total_fee,
  d.pay_type as deduct_pay_type,
  d.status as deduct_status,
  e.pay_type as extend_pay_type,
  e.pay_status as extend_pay_status,
  COUNT(f.id) as flow_count,
  SUM(CASE WHEN f.fund_direction = 'in' THEN f.trade_amount ELSE 0 END) as total_in,
  SUM(CASE WHEN f.fund_direction = 'out' THEN f.trade_amount ELSE 0 END) as total_out,
  r.pay_status as repay_status
FROM rtx_etc_deducrecord_ltk d
LEFT JOIN rtx_deduct_record_extend_ltk e ON d.id = e.deduct_bill_id
LEFT JOIN rtx_etc_balance_flow f ON d.id = f.source_id
LEFT JOIN rtx_deduct_record_repayment r ON d.id = r.deduct_record_id
WHERE d.plate_num = '黑A113355'
GROUP BY d.id
ORDER BY d.create_time DESC;
```
