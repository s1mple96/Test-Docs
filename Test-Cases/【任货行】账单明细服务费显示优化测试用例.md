# 【任货行小程序】账单明细服务费显示优化测试用例

## 1. 账单明细功能测试

### 1.1 服务费显示逻辑验证

#### TC-001 账单明细页面服务费流水隐藏验证
- **前置条件**：用户已登录任货行小程序，账户有包含通行费和服务费的账单记录
- **测试步骤**：
  1. 打开任货行小程序并登录账号
  2. 进入"账单明细"页面
  3. 查看账单明细列表
  4. 检查通行费相关的流水记录显示情况
  5. 验证服务费单独流水是否已隐藏
- **预期结果**：账单明细页面只显示包含服务费的通行费总金额流水，不单独显示服务费流水记录，避免用户误认为重复扣款

#### TC-002 历史账单明细页面服务费流水隐藏验证
- **前置条件**：用户已登录任货行小程序，账户有历史账单记录
- **测试步骤**：
  1. 打开任货行小程序并登录账号
  2. 进入"历史账单明细"页面
  3. 选择查看某个历史账单周期
  4. 查看历史账单明细列表
  5. 验证服务费单独流水是否已隐藏
- **预期结果**：历史账单明细页面同样只显示包含服务费的通行费总金额流水，不单独显示服务费流水记录

#### TC-003 通行费金额准确性验证
- **前置条件**：用户已登录任货行小程序，有包含通行费、服务费和通行保费的账单记录
- **测试步骤**：
  1. 进入"账单明细"页面
  2. 查看某条通行费流水记录
  3. 记录显示的通行费金额
  4. 通过后端接口或数据库查询该笔流水的通行费、服务费和通行保费明细
  5. 验证显示的通行费金额是否等于通行费+服务费+通行保费之和
- **预期结果**：显示的通行费金额包含了通行保费和服务费，总金额计算准确

### 1.2 数据准确性测试

#### TC-004 接口返回数据结构验证
- **前置条件**：有权限调用com.hcb.getRechargeList接口
- **测试步骤**：
  1. 准备测试账号和测试数据
  2. 调用com.hcb.getRechargeList接口获取账单明细
  3. 查看接口返回的数据结构
  4. 验证服务费流水记录的隐藏标识或过滤处理
  5. 确认返回的流水列表中不包含服务费单独流水
- **预期结果**：接口返回的数据已将服务费单独流水隐藏或过滤，不在返回列表中，通行费金额已包含服务费

#### TC-005 不同账单类型的费用显示验证
- **前置条件**：用户账户有多种类型的账单记录（通行费、充值、退款等）
- **测试步骤**：
  1. 进入"账单明细"页面
  2. 查看通行费类型的流水记录
  3. 查看充值类型的流水记录
  4. 查看退款类型的流水记录
  5. 验证各类型流水的显示是否符合预期
- **预期结果**：只有通行费类型的流水隐藏了服务费单独流水，其他类型流水显示正常，不影响其他功能

#### TC-006 多笔通行费流水的服务费处理验证
- **前置条件**：用户账户有多笔通行费交易记录
- **测试步骤**：
  1. 进入"账单明细"页面
  2. 查看多笔通行费流水记录
  3. 逐条检查每笔通行费流水
  4. 验证每笔流水都只显示通行费总金额（包含服务费）
  5. 确认没有任何服务费单独流水显示
- **预期结果**：所有通行费流水都正确隐藏了对应的服务费单独流水，显示逻辑一致

### 1.3 边界场景测试

#### TC-007 无服务费的通行费流水显示验证
- **前置条件**：存在不包含服务费的通行费记录（如特殊优惠订单）
- **测试步骤**：
  1. 准备不包含服务费的通行费测试数据
  2. 进入"账单明细"页面
  3. 查看该笔通行费流水
  4. 验证流水显示是否正常
  5. 确认不会因为没有服务费而出现异常
- **预期结果**：不包含服务费的通行费流水正常显示，金额准确，系统处理稳定

#### TC-008 服务费为0的通行费流水显示验证
- **前置条件**：存在服务费为0元的通行费记录
- **测试步骤**：
  1. 准备服务费为0的通行费测试数据
  2. 进入"账单明细"页面
  3. 查看该笔通行费流水
  4. 验证流水显示的金额准确性
  5. 确认服务费为0的流水是否被隐藏
- **预期结果**：服务费为0的通行费流水显示正常，服务费单独流水正确隐藏（即使金额为0）

#### TC-009 历史数据兼容性验证
- **前置条件**：数据库中存在优化前的旧账单数据
- **测试步骤**：
  1. 准备优化前产生的历史账单数据
  2. 进入"账单明细"或"历史账单明细"页面
  3. 查看历史数据的显示情况
  4. 验证旧数据是否也正确应用了隐藏逻辑
  5. 确认新旧数据显示一致性
- **预期结果**：历史数据也正确应用了服务费隐藏逻辑，新旧数据显示规则一致，不影响历史账单查看

### 1.4 用户体验验证

#### TC-010 费用明细展示清晰度验证
- **前置条件**：用户已登录任货行小程序
- **测试步骤**：
  1. 进入"账单明细"页面
  2. 查看通行费流水的显示方式
  3. 验证费用项目名称是否清晰明确
  4. 确认用户能否准确理解费用构成
  5. 评估是否存在重复扣款的误解可能
- **预期结果**：费用展示清晰明确，通行费展示为包含服务费的总金额，用户不会产生重复扣款的误解

#### TC-011 总金额统计准确性验证
- **前置条件**：用户账户有多笔账单记录
- **测试步骤**：
  1. 进入"账单明细"页面
  2. 查看页面显示的所有流水记录
  3. 手动计算所有流水的总金额
  4. 对比系统显示的总金额或余额变化
  5. 验证隐藏服务费流水后总金额是否准确
- **预期结果**：隐藏服务费单独流水后，账单总金额统计依然准确，账户余额变化正确

## 2. 接口层测试

### 2.1 接口功能验证

#### TC-012 getRechargeList接口服务费过滤功能验证
- **前置条件**：有权限调用com.hcb.getRechargeList接口
- **测试步骤**：
  1. 使用测试工具调用com.hcb.getRechargeList接口
  2. 传入有效的用户标识参数
  3. 获取接口返回的账单流水列表
  4. 检查返回列表中是否包含服务费单独流水
  5. 验证通行费金额是否已包含服务费
- **预期结果**：接口返回的流水列表已过滤掉服务费单独流水，通行费金额包含服务费和通行保费

#### TC-013 接口返回字段完整性验证
- **前置条件**：有权限调用com.hcb.getRechargeList接口
- **测试步骤**：
  1. 调用com.hcb.getRechargeList接口
  2. 查看返回的每条流水记录的字段信息
  3. 验证必需字段是否完整（流水号、金额、时间、类型等）
  4. 确认隐藏服务费后是否影响其他字段
  5. 检查金额字段的准确性和精度
- **预期结果**：接口返回的字段完整准确，隐藏服务费流水不影响其他数据的完整性，金额精度正确

#### TC-014 接口性能和稳定性验证
- **前置条件**：准备大量测试数据
- **测试步骤**：
  1. 批量调用com.hcb.getRechargeList接口
  2. 测试不同数据量下的接口响应时间
  3. 验证接口在高并发情况下的稳定性
  4. 检查内存和CPU使用情况
  5. 确认服务费过滤逻辑不影响性能
- **预期结果**：接口响应时间符合要求，高并发下稳定运行，服务费过滤逻辑不明显影响性能

- **预期结果**：无数据时显示友好的空状态提示，系统运行稳定，不出现错误

#### TC-016 接口异常处理验证
- **前置条件**：模拟接口异常场景
- **测试步骤**：
  1. 模拟接口超时情况
  2. 模拟接口返回错误状态码

- **预期结果**：接口异常时有明确的错误提示，系统不崩溃，用户体验友好

#### TC-017 数据异常容错验证
- **前置条件**：准备包含异常数据的测试场景
