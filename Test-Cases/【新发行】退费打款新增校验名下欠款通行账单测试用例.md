# 新发行退费打款新增校验名下欠款通行账单测试用例

## 1. 批量退费校验功能测试

### 1.1 正常校验流程测试

#### TC-001 无欠款通行账单的批量退费测试
- **前置条件**：钱包用户名下无未支付或支付失败的通行账单
- **测试步骤**：
  1. 登录ETC新发行系统-账单管理-通行对账单-退费对账单
  2. 选择需要退费的多笔账单
  3. 点击"一键对账"按钮
  4. 系统校验钱包用户名下通行账单状态
  5. 验证退费流程能正常进行
- **预期结果**：校验通过，批量退费流程正常执行，退费打款成功

#### TC-002 有欠款通行账单的批量退费拒绝测试
- **前置条件**：钱包用户名下存在未支付或支付失败的通行账单
- **测试步骤**：
  1. 登录ETC新发行系统-账单管理-通行对账单-退费对账单
  2. 选择需要退费的多笔账单（涉及有欠款的钱包用户）
  3. 点击"一键对账"按钮
  4. 系统校验钱包用户名下通行账单状态
  5. 验证系统拒绝退费并提示原因
- **预期结果**：校验失败，系统拒绝退费操作，显示明确的拒绝原因和欠款账单信息

#### TC-003 混合情况的批量退费部分处理测试
- **前置条件**：批量退费中部分钱包用户有欠款，部分无欠款
- **测试步骤**：
  1. 准备混合情况的退费账单（部分用户有欠款，部分无欠款）
  2. 执行批量退费操作
  3. 验证系统对不同用户的处理逻辑
  4. 检查退费结果和拒绝列表
  5. 确认处理结果的准确性
- **预期结果**：无欠款用户的退费正常处理，有欠款用户的退费被拒绝，系统提供详细的处理报告

### 1.2 跨年份通行账单查询测试

#### TC-004 跨多年份欠款账单查询测试
- **前置条件**：钱包用户在不同年份都有通行账单记录
- **测试步骤**：
  1. 准备跨多年的通行账单数据（2020-2024年）
  2. 在部分年份设置未支付或支付失败的账单
  3. 执行退费校验操作
  4. 验证系统查询所有年份通行账单表
  5. 检查欠款账单的识别准确性
- **预期结果**：系统能正确查询所有年份的通行账单表，准确识别各年份的欠款账单

#### TC-005 历史年份欠款账单校验测试
- **前置条件**：用户在历史年份（如2020年）有未支付账单，当前年份无欠款
- **测试步骤**：
  1. 设置用户在2020年有未支付通行账单
  2. 当前年份（2024年）无欠款账单
  3. 执行退费操作
  4. 验证系统能检测到历史年份的欠款
  5. 确认退费被正确拒绝
- **预期结果**：系统能检测到历史年份的欠款账单，正确拒绝退费申请

#### TC-006 当前年份欠款账单校验测试
- **前置条件**：用户仅在当前年份有未支付或支付失败的通行账单
- **测试步骤**：
  1. 设置用户仅在当前年份有欠款账单
  2. 历史年份无欠款记录
  3. 执行退费校验
  4. 验证当前年份欠款的识别
  5. 确认退费拒绝逻辑
- **预期结果**：系统能正确识别当前年份的欠款账单，拒绝退费申请
