#### TC-005 单个钱包-账单扣款与充值同时发生-余额一致性验证

**测试目标**：验证修复后的锁机制能防止并发数据覆盖

**涉及表**：
- `hcb_monthly_paylater_wallet`：钱包表
- `hcb_monthly_paylater_wallet_flow`：流水表
- `hcb_monthly_paylater_wallet_bill`：账单表

**前置条件**：
- 钱包余额：1000.00元（`blance` = 1000.00）
- 待扣款账单：1个，金额200.00元（`bill_status` = 0）
- 准备充值：500.00元

**测试步骤**：

**Step 1：记录并发测试前的数据快照**
```sql
-- 记录初始余额
SELECT wallet_id, blance AS initial_balance, update_time 
FROM hcb.hcb_monthly_paylater_wallet 
WHERE wallet_id = '测试钱包ID';
-- 记录：initial_balance = 1000.00, update_time_before

-- 记录流水基准数量
SELECT COUNT(*) AS flow_count_before 
FROM hcb.hcb_monthly_paylater_wallet_flow 
WHERE wallet_id = '测试钱包ID';
-- 记录：flow_count_before = 假设为5

-- 记录账单ID
SELECT id AS bill_id, bill_amount 
FROM hcb.hcb_monthly_paylater_wallet_bill 
WHERE wallet_id = '测试钱包ID' 
  AND bill_status = 0 
LIMIT 1;
-- 记录：bill_id, bill_amount = 200.00
```

**Step 2：同时发起并发操作（关键）**

使用 JMeter/Postman/Python 脚本：
- 线程1：账单扣款接口，扣款200.00元
- 线程2：钱包充值接口，充值500.00元
- 设置：同时启动，Ramp-Up=0秒

**Step 3：等待操作完成**
- 等待3-5秒，确保两个操作都完成

**Step 4：验证最终余额**
```sql
-- 查询最终余额
SELECT wallet_id, blance AS final_balance, update_time 
FROM hcb.hcb_monthly_paylater_wallet 
WHERE wallet_id = '测试钱包ID';
-- 预期：final_balance = 1300.00 (1000 - 200 + 500)
-- 验证：update_time > update_time_before
```

**Step 5：验证流水记录完整性**
```sql
-- 统计新增流水数量
SELECT COUNT(*) AS flow_count_after 
FROM hcb.hcb_monthly_paylater_wallet_flow 
WHERE wallet_id = '测试钱包ID';
-- 预期：flow_count_after = flow_count_before + 2

-- 查询并发期间的流水记录
SELECT 
  id, trade_amount, before_amount, surplus_amount, 
  fund_direction, biz_type, source_id, create_time 
FROM hcb.hcb_monthly_paylater_wallet_flow 
WHERE wallet_id = '测试钱包ID' 
  AND create_time >= '并发测试开始时间'
ORDER BY create_time;
-- 预期：2条流水记录
-- 流水1（扣款）：trade_amount=-200.00, fund_direction='OUT'
-- 流水2（充值）：trade_amount=500.00, fund_direction='IN'
```

**Step 6：验证流水连续性（重要）**
```sql
-- 查询流水的 before_amount 和 surplus_amount
SELECT 
  id,
  before_amount,
  trade_amount,
  surplus_amount,
  fund_direction,
  create_time,
  LAG(surplus_amount) OVER (ORDER BY create_time) AS prev_surplus_amount
FROM hcb.hcb_monthly_paylater_wallet_flow 
WHERE wallet_id = '测试钱包ID'
ORDER BY create_time DESC
LIMIT 5;
-- 关键验证点：
-- 1. 每条流水的 before_amount 应该等于上一条的 surplus_amount
-- 2. 如果 before_amount 出现重复值，说明并发读取了相同的旧值（BUG未修复）
```

**Step 7：统计流水金额验证一致性**
```sql
-- 统计充值和扣款总额
SELECT 
  SUM(CASE WHEN fund_direction = 'IN' THEN trade_amount ELSE 0 END) AS total_recharge,
  SUM(CASE WHEN fund_direction = 'OUT' THEN ABS(trade_amount) ELSE 0 END) AS total_deduct,
  COUNT(*) AS total_flow_count
FROM hcb.hcb_monthly_paylater_wallet_flow 
WHERE wallet_id = '测试钱包ID' 
  AND create_time >= '并发测试开始时间';
-- 预期：
-- total_recharge = 500.00
-- total_deduct = 200.00
-- total_flow_count = 2
```

**Step 8：验证账单状态**
```sql
-- 查询账单是否扣款成功
SELECT 
  id, bill_status, pay_success_time, pay_flow_id, actual_amount 
FROM hcb.hcb_monthly_paylater_wallet_bill 
WHERE id = 'bill_id';
-- 预期：
-- bill_status = 1 (扣款成功)
-- pay_flow_id = 钱包流水ID（不为空）
-- actual_amount = 200.00
```

**Step 9：最终一致性验证（核心）**
```sql
-- 验证公式：最终余额 = 初始余额 + 充值总额 - 扣款总额
-- 计算：1300 = 1000 + 500 - 200 ✓
```

**预期结果**：
- ✓ 最终余额：1300.00元
- ✓ 流水数量：新增2条（1条扣款 + 1条充值）
- ✓ 扣款流水：trade_amount=200.00, fund_direction='OUT'
- ✓ 充值流水：trade_amount=500.00, fund_direction='IN'
- ✓ 流水连续：before_amount无重复，surplus_amount连贯
- ✓ 账单状态：bill_status=1
- ✓ 一致性验证：1300 = 1000 + 500 - 200

**失败判定（BUG未修复的表现）**：
- ✗ 最终余额错误（如800或1500，说明数据被覆盖）
- ✗ 流水记录丢失（只有1条或0条）
- ✗ 流水的before_amount出现重复值（说明并发读到相同旧值）
- ✗ surplus_amount不连续（说明数据计算错误）

- 钱包余额：5000.00元

**Step 7：验证流水连续性**

- ✓ 扣款总额：600.00元
- ✓ 账单状态：3个账单均为已扣款（bill_status=1）
- ✓ 流水连续：before_amount无重复
- ✓ 一致性验证：4400 = 5000 - 600
