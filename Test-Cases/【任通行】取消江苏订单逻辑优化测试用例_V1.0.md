# 【任通行】取消江苏订单逻辑优化测试用例

## 需求背景

**问题描述**: 因程序判断问题,导致写卡后仍能取消订单

**优化方案**: 写卡或写OBU后都不能取消订单

**业务规则**: 
- 当订单的activeStatus包含指定指令代码时,订单不能被取消
- 需判断的指令代码:
  - 3021: OBU系统信息指令 (com.hcb.txb.3021)
  - 3022: OBU车辆信息指令 (com.hcb.txb.3022)
  - 3023: OBU发行确认 (com.hcb.txb.3023)
  - 3017: ETC0015指令 (com.hcb.txb.3017)
  - 3018: ETC0016指令(国密取消) (com.hcb.txb.3018)
  - 3019: ETC发行确认 (com.hcb.txb.3019)

**修改内容**: 四个取消订单入口的判断都改为包含判断(contains)

**取消订单入口**: 
1. **定时任务入口**: EtcAutomaticCancelTask.automaticCancelTask
2. **小程序入口**: /apply/order/extensionCancelOrder
3. **后台管理-取消ETC订单申请**: /etcapplyorder/cancelOrder
4. **后台管理-一键取消置换申办订单**: /displace/ordercancelAll/{id}

## 1. 四个入口的写卡指令状态判断测试

### 1.1 定时任务入口测试 (EtcAutomaticCancelTask.automaticCancelTask)

#### TC-001 定时任务-订单activeStatus包含3021指令时自动取消
- **前置条件**: 已存在待激活且已支付的ETC订单,该订单的activeStatus包含"3021",创建时间超过配置的自动取消天数
- **测试步骤**:
  1. 准备测试数据:创建时间超期、状态为"待激活"、支付状态为"已支付"、activeStatus包含"3021"的订单
  2. 查看系统字典配置order_cancel_config的取消天数
  3. 执行定时任务EtcAutomaticCancelTask.automaticCancelTask
  4. 查看定时任务执行日志
  5. 查询该订单的状态变化
- **预期结果**: 订单取消失败,订单状态保持"待激活"不变,定时任务日志记录"订单已开始写卡流程,无法取消"或跳过该订单

#### TC-002 定时任务-订单activeStatus包含多个写卡指令时自动取消
- **前置条件**: 已存在待激活且已支付的ETC订单,该订单的activeStatus包含"3021,3022,3023",创建时间超过配置的自动取消天数
- **测试步骤**:
  1. 准备测试数据:创建时间超期、状态为"待激活"、支付状态为"已支付"、activeStatus包含多个写卡指令的订单
  2. 执行定时任务EtcAutomaticCancelTask.automaticCancelTask
  3. 查看定时任务执行日志
  4. 查询该订单的状态变化
- **预期结果**: 订单取消失败,订单状态保持"待激活"不变,定时任务日志记录无法取消的原因

#### TC-003 定时任务-订单activeStatus为空时自动取消成功
- **前置条件**: 已存在待激活且已支付的ETC订单,该订单的activeStatus为空或null,创建时间超过配置的自动取消天数

- **预期结果**: 订单取消成功,订单状态变更为"已取消",etc_automatic_cancel表中插入取消记录
