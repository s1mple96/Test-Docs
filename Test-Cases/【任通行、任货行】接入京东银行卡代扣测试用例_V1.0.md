# 【任通行、任货行】接入京东银行卡代扣测试用例

> 需求：任通行、任货行接入分米支付京东代扣通道，支持京东代扣签约（发送短信、验证短信）、支付、退款、查询、回调；同时新增签约中状态的前端交互（倒计时30s弹框、主动查询）。

## 1. 前置配置验证

### 1.1 数据库配置检查

#### TC-001 货车银行渠道通道配置-京东代扣通道已配置
- **前置条件**:
  - 系统: hcb数据库
  - 表: `hcb_bank_withhold_channel`
- **测试步骤**:
  1. 查询：`SELECT * FROM hcb_bank_withhold_channel WHERE bank_channel_code='JD_QUICK_PAY'`
  2. 检查 `status` 字段是否为 `1`（启用）
  3. 检查 `bank_channel_name` 是否为 `京东代扣`
- **预期结果**: 查到记录，status=1，bank_channel_code=JD_QUICK_PAY

#### TC-002 客车字典配置-京东代扣字典已配置
- **前置条件**:
  - 系统: rtx数据库
  - 表: `sys_dict_data`
- **测试步骤**:
  1. 查询：`SELECT * FROM sys_dict_data WHERE dict_type='bank_way' AND dict_value='JD_QUICK_PAY'`
  2. 检查 `dict_label` 是否为 `京东代扣`
  3. 检查 `status` 是否为 `0`（正常）
- **预期结果**: 查到记录，dict_label=京东代扣，可在前端银行方式选项中正常显示

#### TC-003 客车支付渠道通道配置-京东代扣通道已配置
- **前置条件**:
  - 系统: rtx数据库
  - 表: `rtx_pay_channel_code`
- **测试步骤**:
  1. 查询：`SELECT * FROM rtx_pay_channel_code WHERE pay_channel_code='JD_QUICK_PAY'`
  2. 检查 `trade_request_url` 是否指向分米支付地址
  3. 检查 `notify_url` 是否配置正确
- **预期结果**: 查到记录，渠道配置完整，trade_request_url 和 notify_url 均已配置

#### TC-004 定时任务配置-代扣协议状态查询定时任务已配置
- **前置条件**:
  - 系统: rtx数据库
  - 表: `sys_job`
- **测试步骤**:
  1. 查询：`SELECT * FROM sys_job WHERE invoke_target='bankSignOrderTask.syncIngStatus'`
  2. 检查任务名是否为 `代扣协议状态查询`
  3. 检查 `status` 是否为 `1`（运行中）
  4. 检查 `cron_expression` 是否为 `0 0/5 * * * ?`（每5分钟）
- **预期结果**: 查到记录，任务状态=1，每5分钟执行一次
