# 任通行新增代扣银行卡扣款校验金额优化测试用例

## 1. 扣款校验金额变更测试

### 1.1 平安付普通代扣金额测试

#### TC-001 平安付普通代扣新增银行卡0.01元扣款测试
- **前置条件**：
  - 用户已完成任通行注册和实名认证
  - 用户未绑定银行卡
  - 银行卡账户余额充足（≥0.01元）
- **测试步骤**：
  1. 登录任通行APP
  2. 进入"我的"-"支付管理"-"银行卡管理"
  3. 点击"添加银行卡"按钮
  4. 选择平安付普通代扣方式
  5. 输入银行卡号、持卡人姓名、手机号等信息
  6. 提交绑定请求
  7. 系统发起0.01元扣款校验
  8. 输入短信验证码完成验证
- **预期结果**：
  - 扣款校验金额为0.01元（不是1元）
  - 绑定成功后，银行卡账户扣款0.01元
  - 交易订单记录金额为0.01元
  - 钱包流水记录金额为0.01元
  - 交易状态为"成功"

#### TC-002 平安付普通代扣扣款失败场景测试
- **前置条件**：
  - 用户已完成任通行注册和实名认证
  - 银行卡账户余额不足0.01元或银行卡状态异常
- **测试步骤**：
  1. 登录任通行APP
  2. 进入银行卡管理页面
  3. 尝试添加银行卡（平安付普通代扣）
  4. 提交绑定请求
  5. 系统发起0.01元扣款校验
  6. 观察扣款结果
- **预期结果**：
  - 扣款校验失败
  - 系统提示"扣款验证失败，请检查银行卡状态或余额"
  - 交易订单记录金额为0.01元，状态为"失败"
  - 银行卡绑定失败
  - 用户账户无扣款记录

#### TC-003 平安付普通代扣多次绑定尝试测试
- **前置条件**：
  - 用户多次尝试绑定同一银行卡
  - 前几次绑定失败
- **测试步骤**：
  1. 第一次绑定尝试，扣款失败
  2. 修正银行卡信息或确保余额充足
  3. 第二次绑定尝试
  4. 系统再次发起0.01元扣款校验
  5. 完成绑定
- **预期结果**：
  - 每次绑定尝试均发起0.01元扣款校验
  - 多次失败的扣款不会累加金额
  - 每笔扣款校验都有独立的交易订单记录
  - 最终绑定成功时，只有一笔0.01元成功扣款
