# 【任货行】出库任务全选过滤已选设备及故障设备表去重测试用例

## 核心业务逻辑说明

### 设备筛选核心条件

设备能被选择出库需要同时满足以下条件：

1. **库存状态筛选**（SQL WHERE 条件）：
   - `STOCK_STATUS = '0'` - 未出库（设备在库）
   - `STATUS = '0'` - 未使用/未激活

2. **任务占用检查**（前端 DISABLED 标记）：
   - 设备被其他进行中任务占用时，标记为 `DISABLED=1`
   - 这些设备会显示在列表中但不可勾选
   - 占用判断条件：
     - 设备在 `HCB_OUTBOUNDTASKDEVICE` 表中
     - 关联任务状态为 `0/1/2/3`（待出库/已出库/部分出库/已完成）
     - 设备在任务中的状态 `!= '2'`
     - 占用任务订单号 `!= 当前任务订单号`

### 关键字段说明

| 字段 | 表 | 含义 | 值说明 |
|------|---|------|--------|
| STOCK_STATUS | HCB_NEWSTOCK | 库存状态 | 0=未出库 ✅, 1=已出库 ❌ |
| STATUS | HCB_NEWSTOCK | 使用状态 | 0=未使用/未激活 ✅, 1=已激活/已使用 ❌ |
| STATUS | HCB_OUTBOUNDTASK | 任务状态 | 0=待出库, 1=已出库, 2=部分出库, 3=已完成, 4=已完结, 7=已取消 |
| STATUS | HCB_OUTBOUNDTASKDEVICE | 设备在任务中的状态 | 2=特殊状态（不算占用） |

### 查询逻辑流程

```sql
-- 1. 查询可选设备（SQL 层面过滤）
SELECT * FROM HCB_NEWSTOCK
WHERE TYPE = '1'                    -- 设备类型
  AND WAREHOUSE_TYPE = '4'          -- 仓库类型
  AND STOCK_STATUS = '0'            -- ⭐ 未出库
  AND STATUS = '0'                  -- ⭐ 未使用
  AND (CHECK_IN_FLAG IS NULL OR CHECK_IN_FLAG != '1');

-- 2. 检查设备占用情况（标记 DISABLED）
SELECT a.INTERNAL_DEVICE_NO, b.ORDER_NO
FROM HCB_OUTBOUNDTASKDEVICE a
INNER JOIN HCB_OUTBOUNDTASK b ON a.OUTBOUNDTASK_ID = b.OUTBOUNDTASK_ID
WHERE b.STATUS IN ('0', '1', '2', '3')  -- 进行中的任务
  AND a.STATUS != '2';                  -- 设备状态不为2
```

### 重要说明

1. **已出库设备（STOCK_STATUS=1）不会出现在查询结果中**，在 SQL 层面就被过滤
2. **已激活设备（STATUS=1）不会出现在查询结果中**，在 SQL 层面就被过滤
3. **被占用的设备会显示在列表中但标记为 DISABLED=1**，不可勾选
4. **当前任务已选的设备会被过滤**，不重复显示

### 1.2 设备字段状态说明

**重要字段说明**：
- **STOCK_STATUS**（库存状态）：
  - `0` = 未出库（设备在库）✅ 可选
  - `1` = 已出库（设备不在库）❌ 不可选
  
- **STATUS**（使用状态）：
  - `0` = 未使用/未激活 ✅ 可选
  - `1` = 已激活/已使用 ❌ 不可选

- **设备可选条件**：
  - 必须同时满足：`STOCK_STATUS = '0'` AND `STATUS = '0'`
  - 即：设备在库 且 未被激活使用

- **设备占用标记**：
  - 被其他进行中任务占用的设备会标记为 `DISABLED=1`
  - 这些设备会显示在列表中但不可勾选
  - 占用判断条件：设备在任务中 AND 任务状态为0/1/2/3 AND 设备在任务中的状态≠2

### 测试方法应用说明

本测试用例设计综合应用了以下测试方法：

1. **等价类划分法**：用于设备状态、设备数量、金额等输入测试
2. **边界值分析法**：用于设备数量边界、金额边界测试
3. **场景法**：用于出库流程、换货流程等业务场景测试
4. **状态迁移法**：用于设备状态变化测试
5. **错误推测法**：用于异常场景、并发场景、数据一致性测试

### 重点测试关注点

1. **防止重复出库**：
   - 确保 STOCK_STATUS=1 的设备不出现在可选列表（TC-001, TC-007, TC-017）
   - 确保 STATUS=1 的设备不出现在可选列表（TC-002）
   - 查询条件：`WHERE STOCK_STATUS='0' AND STATUS='0'`

2. **设备占用标记**：
   - 被其他任务占用的设备标记为 DISABLED=1（TC-020）
   - 这些设备显示但不可勾选
   - 占用判断：设备在进行中任务（状态0/1/2/3）且设备状态≠2

3. **换货库存去重**：
   - 确保同一设备号不重复生成换货库存记录（TC-011, TC-012, TC-013）

4. **额度准确性**：
   - 确保代理额度扣减准确，不出现多扣情况（TC-016, TC-017, TC-032）

5. **数据一致性**：
   - 确保设备状态、库存记录、额度记录三者数据一致（TC-031, TC-032）
   - STOCK_STATUS 字段正确更新

6. **并发安全性**：
   - 确保并发操作不会产生重复数据（TC-013, TC-021）

1. **测试环境**：独立测试环境，与生产环境隔离
