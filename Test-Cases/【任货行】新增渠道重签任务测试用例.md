# 任货行新增渠道重签任务测试用例

## 1. 管理后台渠道重签任务菜单功能测试

### 1.1 菜单访问与界面基础功能测试

#### TC-001 渠道重签任务菜单访问测试
- **前置条件**：管理员已登录后台管理系统，具有相应权限
- **测试步骤**：
  1. 登录管理后台系统
  2. 查找"渠道重签任务"菜单项
  3. 点击菜单项进入功能页面
  4. 验证页面加载和显示是否正常
  5. 检查页面标题和面包屑导航
- **预期结果**：菜单正常显示，点击后能成功进入渠道重签任务页面，页面元素加载完整

#### TC-002 渠道重签任务列表基础显示测试
- **前置条件**：系统中存在渠道重签任务数据
- **测试步骤**：
  1. 进入渠道重签任务页面
  2. 验证列表表头显示：序号、重签ID、协议名称、协议版本、是否强制、重签人数/已签人数、状态、创建时间、操作
  3. 验证列表数据的正确显示
  4. 检查数据排序（按创建时间降序）
  5. 验证分页功能是否正常
- **预期结果**：列表表头完整显示，数据按创建时间降序排列，分页功能正常

#### TC-003 渠道重签任务筛选功能测试
- **前置条件**：系统中存在不同协议名称和状态的重签任务
- **测试步骤**：
  1. 测试协议名称筛选功能，输入部分协议名称
  2. 验证筛选结果的准确性
  3. 测试状态筛选功能，分别选择"签约中"、"完成"、"取消"状态
  4. 验证各状态筛选结果的正确性
  5. 测试组合筛选（协议名称+状态）
  6. 测试筛选重置功能
- **预期结果**：协议名称筛选准确，状态筛选正确显示对应状态的任务，组合筛选结果正确，重置功能正常

## 2. 发起重签功能测试

### 2.1 发起重签弹框基础功能测试

#### TC-004 发起重签弹框显示测试
- **前置条件**：管理员已进入渠道重签任务页面
- **测试步骤**：
  1. 点击"发起重签"按钮
  2. 验证弹框正常弹出
  3. 检查弹框内容包含：签订协议、协议版本、是否强制、签订平台、签订用户、导入模板、导入文件、有效用户数、取消和确认按钮
  4. 验证各字段的初始状态
  5. 检查弹框样式和布局
- **预期结果**：弹框正常显示，包含所有必需字段，初始状态正确，界面布局合理

#### TC-005 签订协议选择功能测试
- **前置条件**：系统中存在多个协议模板
- **测试步骤**：
  1. 点击"签订协议"下拉框
  2. 验证协议列表正常显示
  3. 测试模糊搜索功能，输入部分协议名称
  4. 验证搜索结果的准确性
  5. 选择一个协议，验证选中状态
  6. 测试清空选择功能
- **预期结果**：协议列表正常显示，模糊搜索功能正常，能正确选择和清空协议

#### TC-006 协议版本选择功能测试
- **前置条件**：已选择协议，该协议存在多个版本
- **测试步骤**：
  1. 选择协议后，点击"协议版本"下拉框
  2. 验证版本列表是否为所选协议的版本
  3. 选择版本，验证选中状态
  4. 更换协议，验证版本列表是否相应更新
  5. 测试未选协议时版本框的状态
- **预期结果**：版本列表与所选协议匹配，更换协议时版本列表正确更新，未选协议时版本框不可选

#### TC-007 是否强制选项测试
- **前置条件**：弹框已打开
- **测试步骤**：
  1. 验证"是否强制"单选框的默认状态
  2. 分别选择"是"和"否"选项
  3. 验证选中状态的切换
  4. 检查该选项对后续流程的影响提示
- **预期结果**：单选框默认状态正确，能正常切换选择，有相应的说明提示

#### TC-008 签订平台选择测试
- **前置条件**：弹框已打开
- **测试步骤**：
  1. 验证签订平台单选框显示"任货行推广"和"优拓宝"选项
  2. 分别选择不同平台选项
  3. 验证选中状态
  4. 检查平台选择对有效用户数的影响
- **预期结果**：平台选项完整显示，能正常选择，选择不同平台时有效用户数相应变化

### 2.2 用户导入功能测试

#### TC-009 签订用户选择功能测试
- **前置条件**：已选择签订平台
- **测试步骤**：
  1. 验证"签订用户"显示"所有用户"和"导入用户"选项
  2. 选择"所有用户"，验证有效用户数的自动计算和显示
  3. 选择"导入用户"，验证导入模板、导入文件字段的激活状态
  4. 测试选项切换时界面的联动变化
  5. 验证有效用户数字段在两种模式下的不同显示逻辑
- **预期结果**：用户选择选项正确，选择"所有用户"时自动计算并显示用户数，选择"导入用户"时导入相关字段激活且有效用户数等待文件解析后显示

#### TC-010 模板下载功能测试
- **前置条件**：选择了"导入用户"选项
- **测试步骤**：
  1. 点击"下载模板"超链接
  2. 验证模板文件是否成功下载
  3. 检查模板文件格式和内容
  4. 验证模板是否包含"用户手机号"列
  5. 测试模板文件是否可正常打开
- **预期结果**：模板成功下载，文件格式正确，包含用户手机号列，文件可正常打开

#### TC-011 文件上传功能测试
- **前置条件**：已下载模板文件并填写了用户手机号数据
- **测试步骤**：
  1. 点击"导入文件"按钮，打开文件选择弹窗
  2. 选择正确格式的模板文件上传
  3. 验证文件解析和有效用户数的回显
  4. 测试上传错误格式文件的处理，验证错误提示信息
  5. 测试上传空文件的处理，验证相应提示
  6. 测试文件大小限制，验证限制提示
- **预期结果**：正确文件能成功上传并解析，有效用户数正确回显，各种错误情况都有明确的错误提示信息

#### TC-012 有效用户数计算测试
- **前置条件**：已选择签订平台和用户类型
- **测试步骤**：
  1. 测试"所有用户"模式：验证基于平台的有效用户数自动计算（排除注销状态用户）
  2. 测试"导入用户"模式：上传文件后验证解析出的有效用户数回显
  3. 验证重签过滤逻辑（排除已签订相同协议版本的用户）
  4. 测试用户数显示的实时更新
  5. 测试边界情况：0个有效用户、大量用户、全部已签订等
- **预期结果**：两种模式下有效用户数计算准确，排除逻辑正确，实时更新显示，边界情况处理合理

### 2.3 协议配置验证测试

#### TC-013 协议推广端配置验证测试
- **前置条件**：协议管理中存在不同推广端配置的协议
- **测试步骤**：
  1. 选择未包含当前平台推广端配置的协议
  2. 选择签订平台，验证系统的提示信息："当前协议配置推广端未包含所选平台"
  3. 选择包含当前平台推广端配置的协议
  4. 验证能正常继续操作
  5. 测试提示信息的准确性和友好性
- **预期结果**：配置不匹配时显示明确提示"当前协议配置推广端未包含所选平台"，配置匹配时能正常继续，提示信息准确友好

#### TC-014 发起签订确认流程测试
- **前置条件**：已填写完整的发起重签信息
- **测试步骤**：
  1. 点击"确认"按钮前验证必填项校验
  2. 点击"确认"按钮，验证二次确认框的显示
  3. 检查二次确认框提示信息："本次发起签订的用户数为：XXX"
  4. 验证显示的用户数是否与有效用户数一致
  5. 点击二次确认框的"取消"，验证返回发起签订弹框
  6. 点击二次确认框的"确认"，验证重签任务创建
- **预期结果**：必填项校验正确，二次确认框显示标准提示格式"本次发起签订的用户数为：XXX"，用户数准确，确认后成功创建重签任务
