#### TC-014 全量钱包表"余额"字段公式验证测试
- **前置条件**：全量钱包表有多条测试数据，包含扣款和退款场景
- **测试步骤**：
  1. 记录扣款前的所有相关字段值：
     - 钱包余额(wallet_amount) = 500元
     - 冻结金额(freeze_amount) = 1000元
     - 已付月管理费(paid_manage_amount) = 0元
     - 扣失败管理费(fail_manage_amount) = 0元
     - 已付违约保费(paid_breach_amount) = 0元
     - 已付先享月费(paid_silent_amount) = 0元
     - 余额(blance) = 500 + 1000 + 0 + 0 + 0 + 0 = 1500元
  2. 执行先享月费扣款操作（扣款100元）
  3. 查询全量钱包表，验证以下字段变化：
     - 钱包余额(wallet_amount) 保持不变（500→500）
     - 冻结金额(freeze_amount) 保持不变（1000→1000）
     - 已付先享月费(paid_silent_amount) 增加100元（0→100）
     - 余额(blance) 增加100元（1500→1600）
  4. 查询钱包流水表，验证流水记录：
     - 有冻结解冻流水记录（但不更新全量钱包表的freeze_amount）
     - 有余额加减流水记录（但不更新全量钱包表的wallet_amount）
  5. 验证余额公式：余额(blance) = 钱包余额(wallet_amount) + 冻结金额(freeze_amount) + 已付月管理费(paid_manage_amount) + 扣失败管理费(fail_manage_amount) + 已付违约保费(paid_breach_amount) + 已付先享月费(paid_silent_amount)
     - 扣款后：余额 = 500 + 1000 + 0 + 0 + 0 + 100 = 1600元，公式正确
  6. 执行先享月费退款操作（退款100元）
  7. 查询全量钱包表，验证以下字段变化：
     - 钱包余额(wallet_amount) 保持不变（500→500）
     - 冻结金额(freeze_amount) 保持不变（1000→1000）
     - 已付先享月费(paid_silent_amount) 减少100元（100→0）
     - 余额(blance) 减少100元（1600→1500）
- **预期结果**：先享月费扣款和退款操作中，全量钱包表的wallet_amount和freeze_amount字段保持不变（变化只记录在流水中），只有paid_silent_amount字段变化，从而导致blance字段变化，余额公式正确：余额(blance) = 钱包余额(wallet_amount) + 冻结金额(freeze_amount) + 已付月管理费(paid_manage_amount) + 扣失败管理费(fail_manage_amount) + 已付违约保费(paid_breach_amount) + 已付先享月费(paid_silent_amount)

### 2.1 账单生成正常流程测试

#### TC-026 风控任务配置测试
- **前置条件**：有风控任务管理页面的访问权限
- **测试步骤**：
  1. 登录货车后台管理系统
  2. 进入"风控任务管理"页面（路径：后台首页 → 业务管理 → 配置管理 → 风控任务管理）
  3. 点击"新增"按钮，填写任务配置：
     - 任务编码(config_key)：Fee_MonthlyPayLater_Wallet
     - 任务名称(config_name)：无适行钱包包用户月扣款、沉默后付月管理费
     - 任务描述(config_desc)：该任务执行时间为每4月、8月、12月的2号03:00
     - 任务要决(task_rule)：Silent_Wallet&STK:XTK
     - 任务状态(status)：启用
     - 备注(remark)：配置示例-Silent Wallet&STK:XTK表示无账单生成时的配置
  4. 点击"保存"
  5. 验证配置是否保存成功
- **预期结果**：风控任务配置成功保存，任务状态为"启用"

#### TC-027 定时任务读取风控配置并生成账单测试
- **前置条件**：
  - 风控任务已配置（TC-026完成），省份配置为"浙江,江苏,上海,安徽"
  - 系统有符合条件的沉默用户数据（有Silent_Wallet标签）
  - 全量钱包账单表(hcb_complete_wallet_bill)有失败月管理费记录（bill_type=1, bill_status=2）
- **测试步骤**：
  1. 准备测试数据：沉默用户在全量钱包账单表有失败月管理费记录
  2. 配置或手动触发"账单生成定时任务"（执行时间：4月2日、8月2日、12月2日）
  3. 验证定时任务是否读取到风控任务配置（查看日志：config_key=Fee_MonthlyPayLater_Wallet）
  4. **查询全量钱包账单表(hcb_complete_wallet_bill)**，验证按wallet_id+car_num汇总失败月管理费
  5. **查询先享月费账单表(monthly_fee_bill)**，验证账单生成数量和金额
  6. 检查账单状态(deduct_status)是否为"待扣款(1)"

- **预期结果**：定时任务成功读取风控任务配置，先查询全量钱包账单表，按wallet_id+car_num汇总失败金额，符合条件（有Silent_Wallet标签且省份在配置中）的用户都生成了账单，账单金额等于失败金额的汇总，失败条数等于失败记录的数量，账单状态为"待扣款"，退款状态为"未退款"
