- **预期结果**：用户可以完整滚动查看协议内容，文本格式正常

- **前置条件**：隐私政策协议弹框已显示

- **预期结果**：点击同意按钮后弹框关闭，用户可正常使用APP，同意状态已保存

#### TC-011 拒绝协议按钮功能测试

## 4. 协议同意状态持久化测试

### 4.1 协议同意状态保存测试

#### TC-028 协议同意状态本地保存测试
- **前置条件**：用户首次启动APP，隐私政策弹框已显示
- **测试步骤**：
  1. 点击同意隐私政策协议
  2. 完全退出APP（杀死进程）
  3. 重新启动APP
  4. 观察是否还会弹出协议弹框
- **预期结果**：协议同意状态已保存，重启APP不再弹出协议弹框

#### TC-029 协议同意状态清除测试
- **前置条件**：用户已同意过隐私政策协议
- **测试步骤**：
  1. 清除APP数据（卸载重装或清除缓存）
  2. 重新启动APP
  3. 观察是否弹出协议弹框
  4. 验证协议同意状态是否已重置
- **预期结果**：清除数据后协议同意状态重置，重新弹出协议弹框

#### TC-030 多设备协议同意状态同步测试
- **前置条件**：用户在多个设备上使用同一账号
- **测试步骤**：
  1. 在设备A上同意隐私政策协议
  2. 在设备B上登录同一账号
  3. 观察设备B是否还会弹出协议弹框
  4. 验证协议同意状态是否跨设备同步
- **预期结果**：根据业务设计，验证协议同意状态是否按预期同步或独立
