# 【任通行】通行账单实现类优化测试用例

## 测试背景

**优化目标**: 解决shardingjdbc和baomidou多数据源冲突导致的数据源切换失败问题

**优化方案**: 将通行记录实现类(EtcDeducrecordServiceImpl)中的其他业务操作统一移到业务接口上层

**涉及功能**:
1. 导入起诉用户数据
2. 对账单统计补偿任务(BillFileAnalyzeTask.deductBillStatisticsTask)

**测试重点运营商**: 广西捷通卡(JTK)、内蒙蒙通卡(MTK)
