## 3. 发行流程 - 用户政策初始化

### 3.1 产品有"不允许新增绑卡"规则

#### TC-007 新用户发行-产品配置不允许新增绑卡-发行成功后政策写入不允许
- **前置条件**：
  - 系统：分米新发行小程序 + 管理后台
  - 数据库：fenmi_etc
  - 使用产品：已关联「新增绑卡=不允许」规则的产品（如：江苏客车规则测试，product_code: 596876784584）
  - 准备新用户（未发行过该产品的手机号）
- **测试步骤**：
  1. 新用户通过小程序完成申办（绑定车牌、提交申请）
  2. 等待发行成功（订单状态变为已激活）
  3. 查询数据库验证政策明细：
     ```sql
     SELECT d.rule_key, d.rule_value 
     FROM etc_user_policy p 
     JOIN etc_user_policy_detail d ON p.id = d.policy_id 
     WHERE p.car_num = '<车牌号>' AND d.rule_key = 'activatedEnableBindBankCard' 
     LIMIT 1;
     ```
- **预期结果**：
  - `etc_user_policy_detail` 中存在 `rule_key = 'activatedEnableBindBankCard'`，`rule_value = '0'`（不允许）

### 3.2 产品有"允许新增绑卡"规则

#### TC-008 新用户发行-产品配置允许新增绑卡-发行成功后政策写入允许
- **前置条件**：
  - 使用已关联「新增绑卡=允许」规则的产品
  - 准备新用户
- **测试步骤**：
  1. 完成新用户发行流程
  2. 发行成功后查询数据库（同 TC-007 SQL）
- **预期结果**：`rule_value = '1'`（允许）

### 3.3 产品无此规则配置

#### TC-009 新用户发行-产品无"新增绑卡"规则-政策默认为允许
