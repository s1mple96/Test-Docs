# 【熵名片】名片夹及个人名片创建与列表展示测试用例

## 1. 主要功能模块测试

### 1.1 自定义名片创建功能测试

#### TC-001 自定义名片创建入口测试
- **前置条件**：用户已登录小程序，进入首页
- **测试步骤**：
  1. 点击首页的【切换名片】按钮
  2. 点击【新增名片】按钮
  3. 查看弹框中的选项
- **预期结果**：弹框显示【自定义名片】和【公司认证名片】两个选项

#### TC-002 自定义名片基本信息填写测试
- **前置条件**：进入自定义名片创建页面
- **测试步骤**：
  1. 输入姓名"张三"
  2. 输入职位"产品经理"
  3. 输入公司名称"深圳市测试科技有限公司"
  4. 输入电话"13800138000"
  5. 输入微信号"test123"
  6. 输入公司地址"深圳市南山区科技园"
  7. 输入邮箱"test@example.com"
  8. 点击保存
- **预期结果**：名片创建成功，所有信息正确保存并显示

#### TC-003 自定义名片必填项校验测试
- **前置条件**：进入自定义名片创建页面
- **测试步骤**：

  2. 职位留空，填写其他必填项，点击保存

- **前置条件**：用户已登录，当前名片有已同意的交换名片记录
- **测试步骤**：
  1. 点击首页的"名片夹"栏目
  2. 查看名片夹列表
- **预期结果**：名片夹列表显示所有【请求状态==已同意】的名片记录

#### TC-029 名片夹待处理列表加载测试
- **前置条件**：首页显示有待处理的交换名片请求
- **测试步骤**：
  1. 点击首页的"名片请求记录"
  2. 查看跳转后的页面
- **预期结果**：跳转至名片夹页面，自动加载并显示【待处理】状态的交换名片请求

#### TC-030 交换名片请求同意操作测试
- **前置条件**：名片夹中有待处理的交换名片请求
- **测试步骤**：
  1. 进入名片夹待处理列表
  2. 点击某条请求的"同意"按钮
  3. 返回名片夹列表
- **预期结果**：请求状态变更为"已同意"，两张名片互为好友，该名片显示在收藏夹中

#### TC-031 交换名片请求拒绝操作测试
- **前置条件**：名片夹中有待处理的交换名片请求
- **测试步骤**：
  1. 进入名片夹待处理列表
  2. 点击某条请求的"拒绝"按钮
  3. 查看名片夹列表
- **预期结果**：请求状态变更为"已拒绝"，被请求人的名片夹中不显示该记录

#### TC-032 重复交换名片校验测试
- **前置条件**：当前名片与目标名片已有成功的交换记录

  1. 查看目标名片
  2. 点击"交换名片"按钮

#### TC-033 已拒绝名片再次交换测试
- **前置条件**：某条交换名片请求状态为"已拒绝"

#### TC-034 交换名片请求唯一性测试

  2. 尝试再次发起交换请求
