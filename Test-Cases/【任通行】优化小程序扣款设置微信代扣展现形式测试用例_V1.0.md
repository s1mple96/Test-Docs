# 【任通行】优化小程序扣款设置微信代扣展现形式测试用例

## 测试基础信息

### 涉及接口
- **接口地址**: `/rtx-app/index/all-carNumber`
- **请求方式**: GET
- **功能说明**: 查询用户名下所有车牌，返回车辆基本信息及微信代扣开通状态

### 核心代码位置
- **Controller**: `java/rtx/rtx-app-server/src/main/java/com/rtx/app/controller/IndexController.java` (第120-287行)
- **关键逻辑**: 第252-274行（微信代扣状态判断）

### 数据库表
- **车辆表**: `rtx.rtx_etc_card_user`
  - car_num: 车牌号
  - status: 车辆状态（1=正常, 2=黑名单）
  - operator_code: 运营商编码
  - user_id: 用户ID
  
- **微信代扣表**: `rtx.rtx_wechat_withhold_users`
  - plate_num: 车牌号
  - status: 微信代扣状态（1=正常, 2=已暂停, 3=欠费, 4=未授权）
  - withhold_switch: 代扣开关（0=开, 1=关）
  - operator_code: 运营商编码
  - is_priority_use: 优先使用微信代扣（0=不优先, 1=优先）

### 返回字段说明
- **isOpenWithhold**: 是否开通微信代扣（0=未开通, 1=已开通）
- **isCanOpenWithhold**: 是否可开通微信代扣（0=不可开通, 1=可开通）
- **withholdSwitch**: 代扣开关状态（0=开, 1=关）
