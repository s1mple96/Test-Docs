# 【任货行、任通行、新江苏】新增自动插入异常库存测试用例

## 1. 自动插入异常库存功能测试

### 1.1 基础功能验证测试

#### TC-001 江苏设备自动插入异常库存正常流程测试
- **前置条件**：用户准备补换江苏运营商设备，设备号以32开头，且该设备不在系统库存中
- **测试步骤**：
  1. 登录任货行/任通行/新发行系统
  2. 进入补换设备功能页面
  3. 提交补换申请，输入设备号（32开头的江苏设备）
  4. 系统检测设备不在库存中，自动触发异常库存判断
  5. 验证设备自动加入异常库存，补换流程正常继续
- **预期结果**：系统自动将江苏设备（32开头）加入异常库存，补换流程正常进行，无错误提示

#### TC-002 非江苏设备不触发异常库存插入测试
- **前置条件**：用户准备补换非江苏运营商设备，设备号不以32开头，且该设备不在系统库存中
- **测试步骤**：
  1. 登录任货行/任通行/新发行系统
  2. 进入补换设备功能页面
  3. 提交补换申请，输入设备号（非32开头的设备）
  4. 系统检测设备不在库存中
  5. 验证系统不自动加入异常库存，提示设备不存在
- **预期结果**：系统不自动插入异常库存，提示设备不存在库存中，补换申请失败

#### TC-003 设备已存在库存中不触发异常库存插入测试
- **前置条件**：用户准备补换的设备已在系统库存中
- **测试步骤**：
  1. 登录任货行/任通行/新发行系统
  2. 进入补换设备功能页面
  3. 提交补换申请，输入已存在库存中的设备号
  4. 系统检测设备存在于库存中
  5. 验证系统直接使用库存设备，不触发异常库存判断
- **预期结果**：系统使用正常库存设备进行补换，不触发异常库存插入逻辑，补换流程正常

#### TC-004 设备号边界值验证测试（32开头的有效性）
- **前置条件**：准备多种32开头的设备号进行测试
- **测试步骤**：
  1. 准备设备号：3200000001、3299999999、320、32（仅两位）
  2. 依次使用各设备号进行补换申请
  3. 验证设备号以32开头的判断逻辑准确性
  4. 检查异常库存插入条件判断的正确性
  5. 验证补换流程是否正常进行
- **预期结果**：所有32开头的有效设备号均能正确识别并自动插入异常库存，短设备号有合理提示
