# 【新发行】数据导出中心 - 通行账单导出测试用例

> 系统：新发行管理后台（fenmi-etc-admin）  
> 需求提交：fenmi 16e49fea2b  
> 测试环境：192.168.1.60  
> 与【任通行、任货行通行账单和钱包流水数据导出功能】基本一致，差异点见下表

---

## 与任通行导出功能的差异说明

| 差异项 | 任通行后台 | 新发行后台 |
|--------|-----------|-----------|
| 车辆类型选择 | 有（客车/货车） | 无 |
| 钱包流水导出 | 支持 | **不支持** |
| 配置邮箱按钮 | 有（前端可操作） | **无**（邮箱走系统配置，后端维护） |
| 车辆选择 | 可选（不选=全部） | **必填** |
| 下载/重发按钮展示 | 任务成功后显示 | **有权限且任务状态=成功(2)才显示** |

---

## 关键字段需求映射

| 需求原文描述 | 对应库.表.字段 | 说明 |
|-------------|--------------|------|

## 1. 身份证查询用户功能

### 1.1 基础查询

#### TC-001 有效身份证查询到用户
- **前置条件**:
  - 系统: 新发行管理后台 → 数据导出中心
  - 权限: 当前账号有 `center:export:query` 权限（需求：查询身份证触发接口）
  - 测试身份证: 数据库中存在对应 ETC 用户的身份证
  - 验证: `SELECT id, name, car_num, id_code FROM fenmi_etc.etc_user WHERE id_code = '目标身份证' LIMIT 10;`
  - 测试环境: 192.168.1.60
- **测试步骤**:
  1. 进入新发行管理后台数据导出中心
  2. 在身份证输入框中录入有效身份证号
  3. 使输入框失去焦点（点击其他区域）
- **预期结果**:
  - 查询成功，显示用户姓名及实名状态
  - 车辆选择下拉框中展示该身份证名下所有车辆的车牌号
  - **页面上无「客车/货车」类型切换控件**（需求：「没有车辆类型」）

#### TC-002 身份证不存在
- **前置条件**: 输入不存在于 `fenmi_etc.etc_user` 的身份证号
- **测试步骤**:
  1. 录入不存在的身份证
  2. 使输入框失去焦点
- **预期结果**: 提示"无该用户"或类似错误，车辆选择区域不显示/为空

#### TC-003 身份证格式验证
- **前置条件**: 输入格式错误的身份证（如：纯数字但不足18位、含非法字符）
- **测试步骤**: 录入格式错误身份证，失去焦点
- **预期结果**: 提示格式错误，不调用查询接口

## 5. 导出任务提交验证

### 5.1 表单校验

#### TC-011 未查询用户直接导出
- **前置条件**: 身份证输入框为空或查询结果为"无该用户"
- **测试步骤**: 直接点击「导出」按钮
- **预期结果**: 提示"请先查询有效用户"，不提交任务

#### TC-012 未勾选导出类型直接导出
- **前置条件**: 已查询有效用户，未勾选任何导出类型
- **测试步骤**: 直接点击「导出」按钮
- **预期结果**: 提示"请至少选择一种导出类型"

#### TC-013 已查询用户但未选择车辆（必填校验）
- **前置条件**: 已查询有效用户，已勾选通行账单，未选择车辆
- **测试步骤**: 点击「导出」
- **预期结果**: 提示"请选择要导出的车辆"（需求强制必填）

### 5.2 导出任务创建

#### TC-014 通行账单 - 正常导出任务创建
- **前置条件**:
  - 系统: 新发行管理后台
  - 权限: `center:export:export`
  - 已查询有效用户，已选择车辆，已勾选「通行账单记录」
  - 系统已配置接收邮箱: `SELECT config_value FROM fenmi_etc_admin.sys_config WHERE config_key = 'EXPORT_CENTER_EMAIL_RECEIVE' LIMIT 1;`
- **测试步骤**:
  1. 完成表单填写
  2. 点击「导出」按钮
  3. 观察导出任务记录列表
- **预期结果**:
  - 任务提交成功
  - 导出任务记录列表新增对应条数的记录（每辆车一条）
  - 数据库验证:
  ```sql
  SELECT id, type, status, export_source, export_name FROM fenmi_etc_admin.sys_export_download_task 
  WHERE create_by = '当前登录用户名' 
  ORDER BY create_time DESC LIMIT 5;
  ```
  - export_source=1（来源为导出中心），type=4501（通行记录导出），status 初始为处理中

#### TC-015 导出任务最终状态为成功
- **前置条件**: TC-014 任务已提交，等待任务处理完成
- **测试步骤**:
  1. 等待任务处理完成（轮询页面刷新）
  2. 观察任务状态
- **预期结果**:
  - 任务状态变为"成功"
  - 数据库验证: `SELECT status FROM fenmi_etc_admin.sys_export_download_task WHERE id = '任务ID' LIMIT 1;` 结果为 2（成功）
  - 下载按钮和重发邮件按钮在有权限的情况下出现（需求：「有权限且任务状态为成功（2）」）

## 10. 数据一致性验证

#### TC-027 导出数据行数与数据库比对
- **前置条件**:
  - 已成功导出指定车辆、指定时间范围的通行账单
  - 已知测试用户 userId 和时间范围
- **测试步骤**:
  1. 下载导出文件，统计数据行数（去除表头）
  2. 执行 SQL 验证（需结合分年查询表，确认 billYearList 范围）:
  ```sql
  SELECT COUNT(*) FROM fenmi_etc.etc_deduct_record_2024
  WHERE etc_user_id = '目标userId'
  AND create_time BETWEEN '开始时间' AND '结束时间 23:59:59'
  LIMIT 1;
  ```
- **预期结果**: 文件行数与 SQL 结果一致

#### TC-028 导出任务记录数据库验证
- **前置条件**: 已成功创建并完成导出任务
- **测试步骤**: 查询数据库确认任务信息:
  ```sql
  SELECT id, type, status, export_name, export_source, export_url 
  FROM fenmi_etc_admin.sys_export_download_task 
  WHERE create_by = '操作人用户名' 
  ORDER BY create_time DESC LIMIT 5;
  ```
- **预期结果**:
  - type=4501（CENTER_DEDUCT_RECORD_EXPORT）
  - status=2（成功）
  - export_source=1（导出中心来源）
  - export_name 包含车牌号和时间戳（非空）
  - export_url 非空（文件下载链接存在）

## 11. 边界与异常场景

#### TC-029 身份证下无任何车辆记录
- **前置条件**: 存在实名用户但名下无 ETC 车辆的身份证
- **测试步骤**: 输入该身份证查询，选择导出
- **预期结果**: 查询结果显示用户信息但车辆列表为空，无法提交导出（因车辆选择必填）

#### TC-030 指定时间范围内无通行数据
- **前置条件**: 选择一个该车辆无通行记录的时间段
- **测试步骤**: 完成表单填写，点击「导出」
- **预期结果**: 任务提交成功，导出文件内容为空（只有表头）或任务显示"数据量为0"

#### TC-031 并发提交 - 同一用户同一车辆重复提交
- **前置条件**: 已完成一次导出任务提交但任务未结束
- **测试步骤**: 立即再次提交相同参数的导出任务
- **预期结果**: 可以创建第二个任务，任务列表中显示两条独立记录（系统允许多任务并存，每次提交生成唯一任务ID）
