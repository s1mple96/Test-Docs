# 【任货行】渠道重签任务新增渠道公司签约功能测试用例

## 1. 发起重签弹窗新增功能测试

### 1.1 签订用户类型新增渠道公司选项测试

#### TC-001 签订用户类型单选框显示测试
- **前置条件**：管理员已登录后台管理系统，进入渠道重签任务页面
- **测试步骤**：
  1. 点击"发起重签"按钮打开弹窗
  2. 定位到"签订用户类型"字段
  3. 验证单选框选项包含：所有用户、导入用户、渠道公司
  4. 检查三个选项的显示顺序和样式
  5. 验证默认选中状态
- **预期结果**：签订用户类型显示三个单选项（所有用户、导入用户、渠道公司），样式正常，默认选中"所有用户"

#### TC-002 选择渠道公司选项界面联动测试
- **前置条件**：发起重签弹窗已打开
- **测试步骤**：
  1. 点击选择"渠道公司"单选项
  2. 验证导入模板、导入文件字段是否隐藏或禁用
  3. 检查是否显示渠道公司选择字段
  4. 验证有效用户数字段的显示状态
  5. 观察界面布局的变化
- **预期结果**：选择渠道公司后，导入相关字段隐藏，显示渠道公司选择下拉框，有效用户数字段等待选择后计算

#### TC-003 渠道公司下拉框显示测试
- **前置条件**：已选择"渠道公司"签订用户类型
- **测试步骤**：
  1. 点击渠道公司下拉框
  2. 验证渠道公司列表正常显示
  3. 检查列表数据的完整性（渠道公司名称、编码等）
  4. 测试下拉框的滚动和分页加载功能
  5. 验证渠道公司数据的准确性
- **预期结果**：渠道公司列表正常显示，数据完整准确，支持滚动查看，加载流畅

#### TC-004 渠道公司精确搜索功能测试
- **前置条件**：渠道公司下拉框已打开，系统中存在多个渠道公司（如"测试公司T01"）
- **测试步骤**：
  1. 在渠道公司搜索框中输入完整公司名称"测试公司T01"
  2. 点击搜索按钮或按回车键
  3. 验证搜索结果只显示"测试公司T01"
  4. 检查搜索结果的渠道公司ID和名称是否正确
  5. 验证其他渠道公司被正确过滤
- **预期结果**：精确搜索功能正常，只显示完全匹配的渠道公司，数据准确

#### TC-004-1 渠道公司模糊搜索功能测试
- **前置条件**：渠道公司下拉框已打开，系统中存在多个包含"测试"关键字的渠道公司
- **测试步骤**：
  1. 在搜索框中输入部分公司名称"测试"
  2. 验证搜索结果包含所有名称中包含"测试"的渠道公司
  3. 输入数字"01"进行搜索
  4. 验证搜索结果包含所有名称中包含"01"的渠道公司（如"测试公司01"、"测试公司T01"）
  5. 输入单个字符"1"进行搜索
  6. 验证搜索结果包含所有名称中包含"1"的渠道公司
- **预期结果**：模糊搜索功能正常，能匹配名称中任意位置包含关键字的渠道公司，结果准确完整

#### TC-004-2 渠道公司搜索大小写敏感性测试
- **前置条件**：渠道公司下拉框已打开，存在包含英文字母的渠道公司名称
- **测试步骤**：
  1. 输入小写字母"t01"进行搜索
  2. 验证能搜索到"测试公司T01"
  3. 输入大写字母"T01"进行搜索
  4. 验证搜索结果与小写搜索一致
  5. 确认搜索不区分大小写
- **预期结果**：搜索不区分大小写，大写和小写输入返回相同结果

#### TC-004-3 渠道公司搜索特殊字符测试
- **前置条件**：渠道公司下拉框已打开
- **测试步骤**：
  1. 输入特殊字符进行搜索（如"@"、"#"、"%"）
  2. 验证系统是否正常处理，不报错
  3. 输入SQL注入字符进行搜索（如"' OR '1'='1"）
  4. 验证系统安全性，搜索结果正确或返回空
  5. 输入空格进行搜索
  6. 验证系统处理逻辑（返回所有或提示输入关键字）
- **预期结果**：特殊字符搜索不报错，SQL注入攻击被防御，空格搜索有合理处理

#### TC-004-4 渠道公司搜索无结果测试
- **前置条件**：渠道公司下拉框已打开
- **测试步骤**：

  2. 点击搜索
  3. 验证搜索结果为空的提示信息
  4. 检查提示信息的友好性（如"未找到匹配的渠道公司"或"暂无数据"）
  5. 验证界面布局正常，无错误提示
- **预期结果**：搜索无结果时显示友好提示"未找到匹配的渠道公司"，界面布局正常

#### TC-004-5 渠道公司搜索清空和重置测试
- **前置条件**：已进行渠道公司搜索并有结果
- **测试步骤**：
  1. 在搜索框中输入"测试"并搜索，获得结果
  2. 点击清空或删除搜索框内容
  3. 验证搜索结果恢复为全部渠道公司列表
  4. 再次输入搜索关键字"01"
  5. 点击重置按钮（如果有）
  6. 验证搜索条件被清空，列表恢复初始状态
- **预期结果**：清空搜索框后列表恢复全部数据，重置功能正常，列表恢复初始状态

#### TC-004-6 渠道公司搜索性能测试
- **前置条件**：系统中存在大量渠道公司数据（605条）
- **测试步骤**：
  1. 输入常见关键字"测试"进行搜索
  2. 记录搜索响应时间
  3. 输入单个字符"1"进行搜索（可能返回大量结果）
  4. 记录搜索响应时间
  5. 连续快速输入多个字符，测试搜索防抖或节流
  6. 验证搜索过程中界面是否有加载提示
- **预期结果**：搜索响应时间<1秒，大量结果时响应时间<2秒，有防抖或节流机制，有加载提示

#### TC-004-7 渠道公司搜索结果排序测试
- **前置条件**：搜索结果包含多个渠道公司
- **测试步骤**：
  1. 输入"测试公司"进行搜索，获得多个结果
  2. 验证搜索结果的排序规则（按创建时间、按名称、按状态等）
  3. 检查是否优先显示状态为"启用"的渠道公司
  4. 验证排序的合理性和一致性
  5. 多次搜索相同关键字，验证排序是否稳定
- **预期结果**：搜索结果有明确的排序规则，优先显示启用状态的渠道公司，排序稳定一致

#### TC-004-8 渠道公司搜索状态过滤测试
- **前置条件**：系统中存在不同状态的渠道公司（启用/禁用）
- **测试步骤**：
  1. 输入"测试"进行搜索
  2. 验证搜索结果是否包含禁用状态的渠道公司
  3. 检查搜索逻辑：是否只显示启用状态（STATUS=1）的渠道公司
  4. 如果显示禁用公司，验证是否有明确的状态标识
  5. 尝试选择禁用状态的渠道公司，验证系统处理
- **预期结果**：搜索结果只显示启用状态的渠道公司，或禁用公司有明确标识且不可选择

#### TC-005 渠道公司选择和清空功能测试
- **前置条件**：渠道公司下拉框已打开
- **测试步骤**：

  2. 检查选中后下拉框的显示内容
  3. 验证有效用户数是否自动计算并显示

  5. 验证清空后有效用户数的变化
- **预期结果**：渠道公司能正常选择，选中后显示公司名称，有效用户数自动计算，清空功能正常

#### TC-006 渠道公司有效用户数计算测试
- **前置条件**：已选择签订协议、协议版本、签订平台和渠道公司
- **测试步骤**：
  1. 选择渠道公司后，观察有效用户数的计算
  2. 验证计算逻辑：该渠道公司下未签订当前协议版本的用户数
  3. 验证排除注销状态用户的逻辑
  4. 验证排除已签订相同协议版本用户的逻辑
  5. 更换不同渠道公司，验证用户数的实时更新
- **预期结果**：有效用户数计算准确，排除注销用户和已签约用户，更换渠道公司时实时更新

#### TC-007 渠道公司模式与其他模式切换测试
