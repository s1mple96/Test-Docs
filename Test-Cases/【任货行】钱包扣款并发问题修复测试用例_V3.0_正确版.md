#### TC-005 单个钱包-账单扣款与充值同时发生-余额一致性验证（核心测试）

- 钱包余额：1000.00元（`BALANCE` = '1000.00'）
- 待扣款账单：1个，金额200.00元（`STATUS` = 待扣款）
- 准备充值：500.00元

**测试步骤**：

**Step 2：同时发起并发操作（关键）**

使用 JMeter/Postman/Python 脚本：
- 线程1：账单扣款接口，扣款200.00元
- 线程2：钱包充值接口，充值500.00元
- **设置**：同时启动（Ramp-Up=0秒），模拟真实并发

**Python 脚本示例**：
```python
import threading
import requests
import time

WALLET_ID = "测试钱包ID"
BILL_ID = "账单ID"
BASE_URL = "http://测试环境地址"

def deduct_bill():
    try:
        response = requests.post(f"{BASE_URL}/api/deduct", json={
            "wallet_id": WALLET_ID,
            "bill_id": BILL_ID,
            "amount": 200.00
        })
        print(f"扣款结果: {response.status_code}")
    except Exception as e:
        print(f"扣款异常: {e}")

def recharge_wallet():
    try:
        response = requests.post(f"{BASE_URL}/api/recharge", json={
            "wallet_id": WALLET_ID,
            "amount": 500.00
        })
        print(f"充值结果: {response.status_code}")
    except Exception as e:
        print(f"充值异常: {e}")

# 创建线程
t1 = threading.Thread(target=deduct_bill)
t2 = threading.Thread(target=recharge_wallet)

# 同时启动
start_time = time.time()
t1.start()
t2.start()

# 等待完成
t1.join()
t2.join()
print(f"并发测试完成，耗时: {time.time() - start_time:.2f}秒")
```

**Step 3：等待操作完成**

-- 1. 每条流水的 SURPLUS_AMOUNT 应该等于 上一条的 SURPLUS_AMOUNT ± AMOUNT

-- 示例（正确）：

- ✓ 扣款流水：AMOUNT=-200.00, OPERATOR_TYPE='2'
- ✓ 充值流水：AMOUNT=500.00, OPERATOR_TYPE='1'
