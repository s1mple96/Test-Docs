  - 系统: 任通行APP + 后台定时任务
  - 场景: 用户手动扣款同时定时任务补扣

  3. 查看数据库账单状态变化
  4. 验证支付流水记录
- **预期结果**: 
  - 第一个请求将status='2'改为status='3'(扣款中)
  - 第二个请求发现status='3',不发起扣款
  - 仅产生一条支付流水
  - 避免一笔账单多扣场景

## 8. 风险提示

### 8.1 重点风险

1. **防止重复扣款**: 必须严格校验账单状态,扣款中状态不允许再次扣款
2. **状态流转控制**: 确保其他扣款任务(定时任务)会将状态改为扣款中
3. **并发锁机制**: 建议增加分布式锁,防止并发场景下重复扣款
4. **金额计算准确**: 1.0/2.0余额模式充值金额计算必须准确
5. **异常回滚**: 支付失败后账单状态必须正确回滚

### 8.2 测试重点

1. **状态校验**: 重点测试status状态流转
2. **并发场景**: 模拟用户手动扣款+定时任务补扣的并发场景
3. **下黑还款**: 确保支付组件共用不影响下黑还款功能
4. **金额计算**: 验证充值金额计算公式准确性
5. **异常处理**: 覆盖网络异常、支付异常、数据异常等场景
