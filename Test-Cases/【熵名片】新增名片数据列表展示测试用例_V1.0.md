# 【熵名片】新增名片数据列表展示测试用例_V1.0

## 1. 主要功能模块测试

### 1.1 "最近访客"列表展示测试

#### TC-001 访客列表项UI与字段测试
- **前置条件**：有其他用户访问过当前用户的名片
- **测试步骤**：
  1. 进入名片数据展示页面（"我的名片数据"）
  2. 查看"最近访客"列表区域
  3. 检查单条记录的字段完整性
- **预期结果**：
  1. **头像**：显示访客微信头像
  2. **姓名**：显示访客姓名（如"龚斌"）
  3. **职位**：显示访客职位（如"UI设计师"）
  4. **公司**：显示访客公司名称（如"深圳市分米互联科技有限公司"）
  5. **时间**：显示访问日期（格式如"2025.9.20"）
  6. **来源**：显示访问来源（如"微闪名片"）
  7. **电话**：显示访客电话（可能需脱敏或根据权限显示，需确认）

#### TC-002 数据统计看板测试
- **前置条件**：名片有历史访问数据
- **测试步骤**：
  1. 查看页面顶部的统计数据区域
- **预期结果**：
  1. **总访问量**：数字显示正确（如 930）
  2. **我看过的**：数字显示正确（如 26）
  3. **分享名片**：数字显示正确（如 670）
  4. 统计数据应与实际记录一致（或有一定延迟）

### 1.2 底部导航与功能入口测试

#### TC-003 底部导航栏切换测试
- **前置条件**：在小程序名片页
- **测试步骤**：
  1. 点击底部导航栏的"首页"
  2. 点击底部导航栏的"名片码"
  3. 点击底部导航栏的"我的"
- **预期结果**：
  1. 页面能正确切换，高亮状态跟随变化
  2. "名片码"应展示当前名片的二维码

#### TC-004 快捷功能入口测试
- **前置条件**：在名片详情页
- **测试步骤**：
  1. 点击"分享名片" -> 调起微信分享或生成海报
  2. 点击"编辑名片" -> 跳转至名片编辑页
  3. 点击"切换名片" -> 弹出名片切换/新增弹窗
- **预期结果**：各功能入口响应正确，跳转目标页面符合预期

### 1.3 列表交互与加载测试

#### TC-005 列表加载与分页测试
- **前置条件**：访客记录超过一页
- **测试步骤**：
  1. 下拉页面查看更多访客记录
- **预期结果**：
  1. 列表流畅滚动
  2. 到底部加载更多历史数据
  3. 无数据时显示缺省图

#### TC-006 实时性刷新测试
- **前置条件**：当前停留在列表页
- **测试步骤**：
  1. 使用另一手机访问该名片（产生新的访问记录）
  2. 手动下拉刷新当前页面
- **预期结果**：
  1. "总访问量" +1
  2. "最近访客"列表顶部新增一条刚才的访问记录

## 2. 兼容性与异常测试

#### TC-007 字段超长显示测试
- **前置条件**：访客公司名称特别长
- **测试步骤**：
  1. 查看该访客的列表记录
- **预期结果**：
  1. 公司名称应换行显示或省略号截断（如"深圳市分米互联..."），不遮挡其他信息
  2. 布局保持整齐

#### TC-008 缺失字段显示测试
- **前置条件**：访客未设置职位或公司
- **测试步骤**：
  1. 查看该访客记录
- **预期结果**：
  1. 对应字段留空或显示默认占位符（如"未填写"），不出现"null"或布局错位

## 测试用例总结

本测试用例文档基于需求文档《【熵名片】版本1.0的迭代：新增名片数据的列表展示-排期迭代-TAPD平台.pdf》编写，重点覆盖了**最近访客列表**的字段校验（头像/姓名/时间/来源等）、**数据统计看板**以及**底部导航交互**。
