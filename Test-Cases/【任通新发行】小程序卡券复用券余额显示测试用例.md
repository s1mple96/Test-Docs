# 任通、新发行小程序卡券复用券余额显示测试用例

## 1. 复用券余额显示功能测试

### 1.1 复用券余额正常显示测试

#### TC-001 复用券余额正常显示测试
- **前置条件**：用户拥有复用券，券有剩余金额，小程序卡券中心页面可正常访问
- **测试步骤**：
  1. 登录任通行或新内蒙小程序
  2. 进入卡券中心页面
  3. 查找复用券卡片
  4. 验证复用券卡片上是否显示"剩余金额"字段
  5. 检查剩余金额数值是否正确显示
  6. 验证剩余金额格式是否符合设计要求（如：剩余金额：XXX元）
- **预期结果**：复用券卡片正确显示剩余金额字段和具体数值，格式符合设计要求，数值准确

#### TC-002 复用券零余额显示测试
- **前置条件**：用户拥有复用券，券剩余金额为0
- **测试步骤**：
  1. 登录小程序进入卡券中心
  2. 查找剩余金额为0的复用券
  3. 验证零余额复用券的显示方式
  4. 检查是否仍显示"剩余金额：0元"或相应提示
  5. 验证用户交互体验是否合理
- **预期结果**：零余额复用券仍显示剩余金额字段，清晰显示0元或已用完等提示信息

#### TC-003 复用券大金额显示测试
- **前置条件**：用户拥有复用券，券剩余金额较大（如超过1000元）
- **测试步骤**：
  1. 准备剩余金额较大的复用券测试数据
  2. 登录小程序查看该复用券
  3. 验证大金额的显示格式
  4. 检查是否有千分位分隔符或其他格式化处理
  5. 验证显示的完整性和美观性
- **预期结果**：大金额正确显示，格式化处理符合用户习惯，界面美观整洁

### 1.2 复用券数据准确性测试

#### TC-004 复用券余额数据一致性测试
- **前置条件**：后台管理系统中有复用券数据，前端小程序可正常访问
- **测试步骤**：
  1. 在后台系统查询复用券的剩余金额数据
  2. 记录后台显示的剩余金额值
  3. 用户端登录小程序查看同一张复用券
  4. 对比前后台显示的剩余金额是否一致
  5. 验证数据同步的实时性
- **预期结果**：前后台显示的复用券剩余金额完全一致，数据同步及时准确

#### TC-005 复用券使用后余额更新测试
- **前置条件**：用户拥有复用券且有剩余金额，可进行消费使用
- **测试步骤**：
  1. 记录使用前复用券的剩余金额
  2. 使用复用券进行一次消费
  3. 消费完成后返回卡券中心
  4. 检查复用券剩余金额是否正确更新
  5. 验证扣减金额计算的准确性
  6. 测试余额更新的实时性
- **预期结果**：复用券使用后剩余金额正确扣减，计算准确，更新及时

#### TC-006 复用券退款后余额恢复测试
- **前置条件**：用户使用复用券消费后进行退款操作
- **测试步骤**：
  1. 记录退款前复用券的剩余金额
  2. 进行退款操作
  3. 退款完成后查看复用券剩余金额
  4. 验证剩余金额是否正确恢复
  5. 检查恢复金额计算的准确性
- **预期结果**：退款后复用券剩余金额正确恢复，计算准确，状态更新及时
