### 1. TC-088 用户有通行记录自动触发退款测试

#### 修正内容

**修正前的问题**：
- ❌ 前置条件不够明确："系统检测到通行记录"

- ❌ 没有体现触发时机：计费时

**修正后的改进**：
- ✅ 前置条件明确：通行账单已解析并完成计费

  1. 准备测试数据（具体金额和状态）

  3. **模拟通行账单解析和计费（关键步骤）**
  4. 验证退款结果
- ✅ 预期结果完整：包含退款状态、退款类型、操作人、标签、流水类型

#### 关键修正点

```markdown
【修正前】
2. 模拟用户车辆有ETC通行记录
3. 系统自动检测并触发退款

【修正后】

   - 插入通行记录到通行记录表
```

## ⚠️ 注意事项

### 1. 触发时机的准确性

```
测试时必须：
✅ 在通行账单计费完成后触发
❌ 不要在通行记录产生时立即触发
❌ 不要在通行账单解析时触发
```

### 2. 异步处理的等待

```
如果退款是异步处理：
- 触发计费后需要等待3-5秒
- 或使用轮询方式检查退款状态
- 确保退款完成后再验证结果
```

### 3. 批量退款的验证

```
TC-089测试时需要：
- 验证同一钱包下所有车辆的账单都退款
- 不能只退触发车辆的账单
- 需要查询并验证所有账单的退款状态
```
