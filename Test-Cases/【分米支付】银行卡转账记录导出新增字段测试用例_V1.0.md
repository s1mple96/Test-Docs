# 【分米支付】银行卡转账记录导出新增字段测试用例

## 1. 功能测试

### 1.1 导出功能基本验证

#### TC-001 导出功能-划扣成功记录导出验证新增字段
- **前置条件**: 
  - 系统: 分米支付管理后台
  - 数据库: fenmi_pay
  - 场景: 对账中心 → 银行划扣管理 → 银行入账划扣列表
  - 测试数据: 准备至少3条划扣成功(status=4)的转账记录,且关联有支付交易订单
  - 测试环境: 192.168.1.60
- **测试步骤**:
  1. 登录分米支付管理后台
  2. 进入 对账中心 → 银行划扣管理 → 银行入账划扣列表
  3. 查询划扣成功的记录
  4. 点击"导出"按钮
  5. 打开导出的Excel文件
  6. 检查Excel文件中的字段列
- **预期结果**: 
  - 导出成功,生成Excel文件
  - Excel文件中包含新增字段"订单编号"
  - Excel文件中包含新增字段"支付申请流水号"
  - 原有字段正常显示(入账记录id、入账订单号、转账订单号、转账申请流水号、转账确认流水号、转账金额、转账状态、备注等)

#### TC-002 导出功能-新增字段数据准确性验证
- **前置条件**: 
  - 系统: 分米支付管理后台
  - 数据库: fenmi_pay
  - 场景: 已有划扣成功且关联支付订单的测试数据
  - 准备数据: 
    ```sql
    -- 查询一条划扣成功的记录,记录其entry_record_id
    SELECT * FROM bank_account_transfer_record WHERE status='4' ORDER BY id DESC LIMIT 1;
    
    -- 通过entry_record_id查询入账记录的bank_order_no
    SELECT id, bank_order_no FROM bank_account_entry_record WHERE id = {entry_record_id};
    
    -- 通过bank_order_no查询支付交易订单的trade_order_no和id
    SELECT id, trade_order_no FROM pay_trade_order WHERE bank_order_no = {bank_order_no};
    ```
  - 测试环境: 192.168.1.60
- **测试步骤**:
  1. 执行前置条件中的SQL,记录查询结果: 订单编号(trade_order_no)和支付申请流水号(id)
  2. 登录分米支付管理后台
  3. 进入银行入账划扣列表
  4. 查询并导出包含该条记录的数据
  5. 打开Excel文件,找到对应的记录行
  6. 对比Excel中的"订单编号"和"支付申请流水号"与步骤1查询的结果
- **预期结果**: 
  - Excel中的"订单编号"字段 = 数据库中pay_trade_order.trade_order_no
  - Excel中的"支付申请流水号"字段 = 数据库中pay_trade_order.id
  - 数据准确无误

### 1.2 不同业务场景验证

#### TC-003 导出功能-有关联支付订单的记录
- **前置条件**: 
  - 系统: 分米支付管理后台
  - 数据库: fenmi_pay
  - 场景: 准备划扣记录,该记录通过入账记录关联到支付交易订单
  - 测试环境: 192.168.1.60
- **测试步骤**:
  1. 查询数据库,确认有关联支付订单的划扣记录:
     ```sql
     SELECT batr.*, pto.trade_order_no, pto.id as pay_trade_id
     FROM bank_account_transfer_record batr
     LEFT JOIN bank_account_entry_record baer ON batr.entry_record_id = baer.id
     LEFT JOIN pay_trade_order pto ON baer.bank_order_no = pto.bank_order_no
     WHERE batr.status='4' AND pto.id IS NOT NULL
     LIMIT 5;
     ```
  2. 登录管理后台导出包含这些记录的数据
  3. 检查Excel中这些记录的"订单编号"和"支付申请流水号"字段
- **预期结果**: 
  - 有关联数据的记录,新增字段正常显示订单编号和支付流水号
  - 字段值不为空,格式正确

#### TC-004 导出功能-无关联支付订单的记录(退款场景)
- **前置条件**: 
  - 系统: 分米支付管理后台
  - 数据库: fenmi_pay
  - 场景: 准备退款场景的划扣记录(bank_order_no以refund开头)
  - 测试环境: 192.168.1.60
- **测试步骤**:

     ```sql

     ```
  2. 登录管理后台导出包含这些记录的数据

  - 不影响导出功能,Excel文件正常生成
  - 原有字段正常显示
