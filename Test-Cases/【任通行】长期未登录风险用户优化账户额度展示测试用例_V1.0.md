## 测试数据说明

| 数据项 | 值 |
|--------|-----|
| 已打标测试钱包 | user_wallet_id = `1645305471005036545`（balance=100.14，give_gold_balance=0.14，accountBalance=100.00） |
| 风险用户标签 | `InactiveRiskWallet`（现有标签，触发打 RiskyDormantWallet 的前提） |

## 1. 定时任务测试（打标/解标）

### 1.1 addRiskyDormantWallet 月度打标任务

#### TC-001 打标-满足三条件的用户成功打标
- **前置条件**：
  - 系统：任通行后台定时任务（rtx-quartz）
  - 数据库：rtx
  - 准备一个钱包用户：带 `InactiveRiskWallet` 标签 + 钱包余额 > 0 + 最近小程序登录时间超过10天（args=10）+ 未带 `RiskyDormantWallet` 标签
  - 确认 `rtx_tag_config` 中 `RiskyDormantWallet` 标签已配置且 status=1、is_auto=1、args=10
- **测试步骤**：
  1. 确认目标用户满足以上条件（查询 rtx_tag_relation、rtx_user_wallet、rtx_driver_user_info）
  2. 手动触发打标任务（管理后台 → 定时任务 → `tagTask.addRiskyDormantWallet` → 立即执行）
  3. 查询 `rtx_tag_relation` 确认是否新增 `RiskyDormantWallet` 标签记录
- **预期结果**：目标用户被成功打标，`rtx_tag_relation` 新增一条 tag_key=`RiskyDormantWallet` 的记录

#### TC-002 打标-已带 RiskyDormantWallet 标签的用户不重复打标
- **前置条件**：
  - 用户已带 `InactiveRiskWallet` + `RiskyDormantWallet` 标签，钱包余额 > 0，登录时间超过阈值
  - 数据库：rtx
- **测试步骤**：
  1. 记录目标用户当前 `rtx_tag_relation` 中 `RiskyDormantWallet` 标签的条数
  2. 手动触发 `addRiskyDormantWallet` 任务
  3. 再次查询 `rtx_tag_relation` 中该用户 `RiskyDormantWallet` 标签条数
- **预期结果**：`RiskyDormantWallet` 标签条数不增加，不重复打标

#### TC-003 打标-钱包余额等于0的用户不打标
- **前置条件**：
  - 准备一个用户：带 `InactiveRiskWallet` 标签 + 钱包余额 = 0 + 登录时间超过阈值 + 未带 `RiskyDormantWallet` 标签
- **测试步骤**：
  1. 确认用户钱包 balance = 0
  2. 手动触发 `addRiskyDormantWallet` 任务
  3. 查询 `rtx_tag_relation` 是否新增 `RiskyDormantWallet` 标签
- **预期结果**：该用户不被打标，`rtx_tag_relation` 中无新增记录

#### TC-004 打标-登录时间未超过阈值的用户不打标
- **前置条件**：
  - 准备一个用户：带 `InactiveRiskWallet` 标签 + 钱包余额 > 0 + 最近小程序登录时间在10天以内 + 未带 `RiskyDormantWallet` 标签
- **测试步骤**：

#### TC-005 打标-没有小程序登录记录的用户直接打标
- **前置条件**：
  - 准备一个用户：带 `InactiveRiskWallet` 标签 + 钱包余额 > 0 + 钱包下所有车辆对应的 `rtx_driver_user_info.last_login_time` 全为 NULL + 未带 `RiskyDormantWallet` 标签

### 1.2 delRiskyDormantWallet 每日解标任务

#### TC-009 解标-钱包余额小于0时成功解标

#### TC-012 解标任务-钱包ID查询不到钱包信息时跳过不报错

  1. 手动触发 `tagTask.delRiskyDormantWallet` 任务
  2. 查看任务日志
- **预期结果**：任务正常执行完毕，异常钱包被跳过并打印 error 日志，不影响其他用户解标处理

## 2. isShowWalletBalance 接口测试

### 2.1 正常场景

#### TC-013 用户带 RiskyDormantWallet 标签-接口返回 false
- **前置条件**：
  - 系统：任通行 APP 接口
  - 数据库：rtx
  - 用户 user_wallet_id = `1645305471005036545`（已带 RiskyDormantWallet 标签）
  - 字典 `not_show_wallet_balance_type` 中 dict_value = `RiskyDormantWallet`
- **测试步骤**：
  1. 以该用户身份登录任通行小程序
  2. 调用接口 GET `/my/isShowWalletBalance`（无需额外参数，从 token 获取用户）
  3. 查看接口返回值
- **预期结果**：接口返回 `{"code":200,"data":false}`，表示隐藏账户余额

#### TC-014 用户不带 RiskyDormantWallet 标签-接口返回 true
- **前置条件**：
  - 使用一个不带 `RiskyDormantWallet` 标签的普通用户
  - 字典配置正常
- **测试步骤**：
  1. 以普通用户身份登录任通行小程序
  2. 调用接口 GET `/my/isShowWalletBalance`
  3. 查看返回值
- **预期结果**：接口返回 `{"code":200,"data":true}`，表示正常显示账户余额

### 2.2 字典配置相关

#### TC-015 字典值为空时-接口返回 true（默认显示）
- **前置条件**：
  - 将字典 `not_show_wallet_balance_type` 中 `not_show_wallet_balance_tag` 的 dict_value 修改为空字符串
  - 目标用户带 `RiskyDormantWallet` 标签
- **测试步骤**：
  1. 修改字典值为空
  2. 调用 GET `/my/isShowWalletBalance`
  3. 查看返回值
  4. 恢复字典配置
- **预期结果**：接口返回 true（无需隐藏标签时默认显示）

#### TC-016 字典配置多个标签（逗号分隔）-命中任一标签则返回 false
- **前置条件**：
  - 将字典 dict_value 修改为 `RiskyDormantWallet,OtherTag`（多标签）
  - 目标用户仅带 `RiskyDormantWallet` 标签（不带 OtherTag）
- **测试步骤**：
  1. 修改字典值为多标签（英文逗号分隔）
  2. 调用 GET `/my/isShowWalletBalance`
  3. 查看返回值
  4. 恢复字典配置
- **预期结果**：接口返回 false（命中其中一个配置标签即隐藏）

### 2.3 异常场景
