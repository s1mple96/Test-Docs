# 【ETC钱包2.0】欠费充值逻辑优化测试用例

## 📋 测试背景

### 问题描述
- **环境**: 生产环境
- **场景**: 用户开启ETC钱包2.0模式，使用余额扣款模式
- **问题**: 用户欠费-19.92元时，点击"立即付款"提示需要填写充值金额，但由于系统最低充值50元限制，导致用户无法直接充值欠费金额
- **影响**: 用户体验差，充值流程不顺畅

### 修复方案
**仅针对欠费场景的两个改动点**:
1. **按钮显示优化**: 欠费时按钮显示绝对值，不显示负数
2. **充值金额校验优化**: 欠费时，用户充值金额大于等于欠费金额即可

**重要说明**: 
- ✅ 不欠费用户的充值逻辑完全不变
- ✅ 仍然遵循原有的最低充值50元限制
- ✅ 改动范围仅限于欠费场景

---

## 🎯 测试目标

**核心测试目标** (仅针对欠费场景):
1. 验证欠费场景下按钮显示绝对值逻辑
2. 验证欠费充值金额校验优化逻辑
3. 验证充值成功后余额更新正确性

**回归测试目标** (确保不影响原有功能):
4. 验证非欠费用户充值逻辑不变
5. 验证最低充值50元限制仍然有效
