# 【任货行】黑名单导出优化测试用例

## 1. 数据归档功能测试

### 1.1 MongoDB流水归档测试

#### TC-001 用户最近下黑时间归档任务执行
- **前置条件**：MongoDB流水归档已完成,存在待归档的用户下黑时间数据
- **测试步骤**：
  1. 确认流水归档任务状态为已完成
  2. 触发用户最近下黑时间归档任务
  3. 查询归档后的汇总表数据
  4. 随机抽取5个用户,对比汇总表与原始流水的下黑时间
- **预期结果**：归档成功,汇总表数据与原始流水一致

#### TC-002 归档定时任务月度执行
- **前置条件**：已配置每月归档定时任务
- **测试步骤**：
  1. 等待定时任务触发
  2. 检查任务执行日志
  3. 验证归档数据更新时间
- **预期结果**：定时任务按时执行,归档数据正确更新

#### TC-003 流水归档未完成时触发下黑时间归档
- **前置条件**：流水归档任务未完成
- **测试步骤**：
  1. 确认流水归档状态为进行中或失败
  2. 尝试触发下黑时间归档任务
  3. 检查系统响应
- **预期结果**：系统阻止执行并提示流水归档未完成

#### TC-004 归档任务重复执行防护
- **前置条件**：归档任务正在执行中
- **测试步骤**：
  1. 启动归档任务
  2. 任务执行期间再次触发归档
  3. 观察系统行为
- **预期结果**：系统阻止重复执行,提示任务进行中

#### TC-005 归档数据为空场景
- **前置条件**：MongoDB中无待归档流水数据
- **测试步骤**：
  1. 执行归档任务
  2. 检查任务执行结果
- **预期结果**：任务正常完成,日志记录无数据归档

## 2. 路费欠款查询优化测试

### 2.1 下黑时间判断逻辑测试

#### TC-010 下黑时间在数据库保留期内查询
- **前置条件**：用户下黑时间<3个月(数据库迁移时间内)
- **测试步骤**：
  1. 选择近期下黑的用户
  2. 执行欠费金额查询
  3. 监控SQL执行日志,确认仅查询MySQL
  4. 记录查询耗时
- **预期结果**：仅查询MySQL,不访问MongoDB,查询耗时<500ms

#### TC-011 下黑时间超过数据库迁移时间查询
- **前置条件**：用户下黑时间>3个月
- **测试步骤**：
  1. 选择早期下黑的用户
  2. 执行欠费金额查询
  3. 监控查询日志,确认查询MongoDB汇总表
  4. 验证金额准确性
- **预期结果**：优先查询MongoDB汇总表,金额准确

#### TC-012 下黑时间等于迁移临界值
- **前置条件**：用户下黑时间恰好等于3个月前
- **测试步骤**：
  1. 准备临界时间点的测试用户
  2. 执行欠费金额查询
  3. 验证查询逻辑的边界处理
- **预期结果**：按既定规则选择数据源,金额准确

### 2.2 多数据源查询逻辑测试

#### TC-013 MySQL流水表获取最近下黑时间

  2. 查询MongoDB流水统计表(无结果)

  4. 验证返回的下黑时间

#### TC-016 所有数据源均无记录
- **前置条件**：用户在所有数据源均无流水记录
- **测试步骤**：
  1. 依次查询所有数据源
  2. 检查系统返回结果
- **预期结果**：返回欠费金额为0,无报错

#### TC-017 更新MongoDB最近一次下黑时间
- **前置条件**：从数据库查询到新的下黑时间
- **测试步骤**：
  1. 查询用户最新下黑时间

  3. 验证更新结果
- **预期结果**：MongoDB下黑时间更新成功
