# 【任通行】数据导出中心 - 通行账单和钱包流水导出测试用例

> 系统：任通行管理后台（rtx-admin-server）  
> 需求提交：b2b22b9733  
> 测试环境：192.168.1.60

---

## 关键字段需求映射

| 需求原文描述 | 对应库.表.字段 | 说明 |
|-------------|--------------|------|
| 客车 | rtx.rtx_etc_card_user.id_code | carType=1，查询 EtcCardUser，返回 walletId |
| 货车 | rtx.hcb_truck_user.id_code | carType=2，查询 TruckUser |
| 用户名下的车辆 | rtx.rtx_etc_card_user.car_num / hcb_truck_user.car_num | 查询该身份证下所有车辆 |
| 通行账单记录（客车）| rtx.rtx_etc_deducrecord | dict_value=1001，导出字段见 CenterExportEtcDeducrecordVO |
| 通行账单记录（货车）| 关联 HCB 账单数据 | dict_value=1002，导出字段见 CenterExportHcbHeideducrecordVO |
| 钱包流水记录（客车）| rtx 钱包流水表 | dict_value=1003，导出字段见 CenterExportEtcBalanceFlowVO |
| 钱包流水记录（货车）| rtx 货车钱包流水 | dict_value=1004，导出字段见 CenterExportTruckWalletFlowVO |
| 接收邮箱 | rtx.sys_config.config_value (config_key='EXPORT_CENTER_EMAIL_RECEIVE') | 邮箱走系统配置，配置页面写入此配置项 |
| 导出任务记录 | rtx.rtx_etc_export_record | export_source=1 区分导出中心来源 |
| 配置邮箱权限 | rtx.sys_menu.perms = 'center:export:email' | 需求：「单独权限控制」 |

## 1. 身份证查询用户功能

### 1.1 客车用户查询

#### TC-001 客车用户查询 - 有效身份证查询到用户
- **前置条件**:
  - 系统: 任通行管理后台 → 数据导出中心
  - 权限: 当前账号有 `center:export:query` 权限
  - 数据库: rtx
  - 车辆类型: 客车
  - 测试身份证：数据库中存在对应客车用户的有效身份证号
  - 测试环境: 192.168.1.60
- **测试步骤**:
  1. 进入数据导出中心页面
  2. 选择用户类型为「客车」
  3. 在身份证输入框中录入有效身份证号
  4. 使输入框失去焦点（点击其他区域）
  5. 观察查询结果
- **预期结果**:
  - 查询成功，显示用户姓名（如"张三（已实名）"）
  - 车辆选择下拉框中展示该身份证名下的所有客车车牌号
  - `SELECT car_num FROM rtx.rtx_etc_card_user WHERE id_code = '输入的身份证' LIMIT 20;` 结果与页面显示一致

#### TC-002 客车用户查询 - 身份证不存在
- **前置条件**:
  - 系统: 任通行管理后台 → 数据导出中心
  - 权限: `center:export:query`
  - 输入不存在于 rtx.rtx_etc_card_user 中的身份证号
- **测试步骤**:
  1. 选择用户类型「客车」
  2. 输入不存在的身份证号
  3. 使输入框失去焦点
- **预期结果**: 页面提示"无该用户"或类似错误提示，车辆选择区域不展示

#### TC-003 货车用户查询 - 有效身份证查询到用户
- **前置条件**:
  - 系统: 任通行管理后台 → 数据导出中心
  - 权限: `center:export:query`
  - 测试身份证：`SELECT id_code FROM rtx.hcb_truck_user LIMIT 1;` 取一个有效身份证
- **测试步骤**:
  1. 选择用户类型「货车」
  2. 录入货车用户身份证号
  3. 使输入框失去焦点
- **预期结果**: 查询成功，显示用户信息，车辆选择展示该用户名下所有货车车牌号

#### TC-004 货车用户查询 - 身份证不存在
- **前置条件**: 货车，输入不存在于 `rtx.hcb_truck_user` 的身份证
- **测试步骤**: 同 TC-002 操作
- **预期结果**: 提示"无该用户"

## 13. 数据一致性验证

#### TC-039 导出数据与数据库比对（客车通行账单）
- **前置条件**:
  - 已知一个客车用户身份证，有若干通行账单
  - 指定具体时间范围导出
- **测试步骤**:
  1. 下载导出文件，统计数据条数
  2. 执行以下 SQL 查询:
  ```sql
  SELECT COUNT(*) FROM rtx.rtx_etc_deducrecord 
  WHERE car_num IN (
    SELECT car_num FROM rtx.rtx_etc_card_user WHERE id_code = '测试身份证'
  )
  AND create_time BETWEEN '开始时间' AND '结束时间 23:59:59'
  LIMIT 1;
  ```
