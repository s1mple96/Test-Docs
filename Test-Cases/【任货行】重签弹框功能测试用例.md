# 【任货行】重签弹框功能测试用例

## 1. 重签弹框基础功能测试

### 1.1 首页重签弹框功能验证

#### TC-001 首页重签弹框正常显示测试
- **前置条件**：用户已登录且存在需要重签的协议，当天首次进入首页
- **测试步骤**：
  1. 启动任货行APP
  2. 使用测试账号登录系统
  3. 进入首页模块
  4. 观察页面加载完成后的弹框显示
  5. 检查弹框内容和按钮设置
- **预期结果**：首页加载完成后自动弹出重签弹框，显示需要重签的协议信息，包含立即签约和稍后再说按钮

#### TC-002 我的模块重签弹框正常显示测试
- **前置条件**：用户已登录且存在需要重签的协议，当天首次进入我的模块
- **测试步骤**：
  1. 启动任货行APP并登录
  2. 点击底部导航栏我的选项
  3. 进入我的模块页面
  4. 观察页面加载完成后的弹框显示
  5. 检查弹框内容的完整性
- **预期结果**：我的模块页面加载完成后自动弹出重签弹框，显示协议信息和操作按钮

#### TC-003 无需重签协议用户不显示弹框测试
- **前置条件**：用户已登录且已签署所有必要协议，不存在需要重签的协议
- **测试步骤**：
  1. 使用已完成所有协议签约的测试账号登录
  2. 分别进入首页和我的模块
  3. 观察页面加载过程和最终显示状态
  4. 确认无任何弹框出现
  5. 验证页面功能正常可用
- **预期结果**：进入首页和我的模块时均不弹出重签弹框，页面正常显示无干扰

### 1.2 弹框交互功能验证

#### TC-004 立即签约功能测试
- **前置条件**：用户已登录且存在需要重签的协议，重签弹框已显示
- **测试步骤**：
  1. 进入首页或我的模块触发重签弹框
  2. 点击弹框中的立即签约按钮
  3. 观察页面跳转和协议页面显示
  4. 验证协议内容的正确性
  5. 完成协议签约流程测试
- **预期结果**：点击立即签约后弹框关闭，跳转到协议签约页面，显示正确的协议内容，可正常完成签约

#### TC-005 稍后再说功能测试
- **前置条件**：用户已登录且存在需要重签的协议，重签弹框已显示
- **测试步骤**：
  1. 进入首页或我的模块触发重签弹框
  2. 点击弹框中的稍后再说按钮
  3. 观察弹框关闭后的页面状态
  4. 验证页面功能是否正常可用
  5. 检查当天弹出标记是否已记录
- **预期结果**：点击稍后再说后弹框关闭，停留在当前页面，页面功能正常，系统记录当天已弹出标记

#### TC-006 弹框关闭按钮功能测试
- **前置条件**：用户已登录且存在需要重签的协议，重签弹框已显示且包含关闭按钮
- **测试步骤**：
  1. 进入首页或我的模块触发重签弹框
  2. 点击弹框右上角的关闭按钮
  3. 观察弹框关闭效果
  4. 验证页面状态和功能可用性
  5. 确认关闭效果与稍后再说一致
- **预期结果**：点击关闭按钮后弹框关闭，停留在当前页面，效果等同于稍后再说功能

## 2. 频率控制机制测试

### 2.1 当天频率控制验证

#### TC-007 首页当天频率控制测试
- **前置条件**：用户已登录且存在需要重签的协议，当天已在首页弹出过重签弹框
- **测试步骤**：
  1. 第一次进入首页确认弹框出现并点击稍后再说
  2. 退出首页返回其他页面
  3. 再次进入首页观察弹框状态
  4. 重复进入首页操作多次
  5. 验证当天不再重复弹出
- **预期结果**：首次进入首页时弹出重签弹框，当天后续多次进入首页时不再弹出弹框

#### TC-008 我的模块当天频率控制测试
- **前置条件**：用户已登录且存在需要重签的协议，当天已在我的模块弹出过重签弹框
- **测试步骤**：
  1. 第一次进入我的模块确认弹框出现并点击稍后再说
  2. 退出我的模块切换到其他页面
  3. 再次进入我的模块观察弹框状态
  4. 重复进入我的模块操作多次
  5. 验证当天不再重复弹出
- **预期结果**：首次进入我的模块时弹出重签弹框，当天后续多次进入我的模块时不再弹出弹框

#### TC-009 跨模块频率控制测试
- **前置条件**：用户已登录且存在需要重签的协议，当天首次使用APP
- **测试步骤**：
  1. 进入首页观察弹框出现并点击稍后再说
  2. 切换到我的模块观察弹框状态
  3. 再次切换回首页验证弹框状态
  4. 再次切换到我的模块验证弹框状态
  5. 确认跨模块频率控制策略
- **预期结果**：根据产品需求确认是独立控制（各模块首次进入时弹出）还是全局控制（任一模块弹出后其他模块不再弹出）

### 2.2 跨天重置功能验证

#### TC-010 跨天重置弹框显示状态测试
- **前置条件**：用户已登录且存在需要重签的协议，昨天已弹出过重签弹框
- **测试步骤**：
  1. 确认昨天已在首页或我的模块弹出过弹框
  2. 等待跨过零点或修改系统时间模拟跨天
  3. 重新启动APP并登录
  4. 进入首页和我的模块
  5. 验证弹框重新显示功能
- **预期结果**：新的一天首次进入页面时重新弹出重签弹框，昨天的弹出记录已重置

## 3. 特殊场景处理测试

### 3.1 多协议和异常处理

#### TC-011 多个协议需要重签时的处理测试
- **前置条件**：用户已登录且存在多个需要重签的协议，当天首次进入页面
- **测试步骤**：
  1. 配置测试用户存在2-3个未签约协议
  2. 进入首页或我的模块触发弹框
  3. 观察弹框显示的协议信息
  4. 点击立即签约验证签约流程
  5. 确认是否能依次签署所有协议
- **预期结果**：弹框显示所有需要重签的协议或协议数量提示，点击立即签约可依次完成所有协议签署

#### TC-012 网络异常时重签弹框处理测试
- **前置条件**：用户已登录，网络连接异常或不稳定
- **测试步骤**：
  1. 断开设备网络连接
  2. 进入首页或我的模块观察处理方式
  3. 恢复网络连接
  4. 重新进入页面验证功能恢复
  5. 确认协议状态检查的网络依赖处理
- **预期结果**：网络异常时不弹出弹框或显示网络错误提示，网络恢复后正常检查协议状态并弹出弹框

#### TC-013 边界条件处理测试
- **前置条件**：准备各种边界条件测试场景
- **测试步骤**：
  1. 测试23:59分进入页面，0:00后再次进入的跨天处理
  2. 测试快速切换页面时的弹框控制
  3. 测试APP后台切换后的功能状态
  4. 测试设备内存不足时的处理机制
  5. 验证各种边界情况的稳定性
- **预期结果**：跨天时间点处理正确，快速切换不导致重复弹框，后台切换后功能正常，内存不足时优雅降级

### 3.2 性能和兼容性测试

#### TC-014 重签弹框功能性能测试
- **前置条件**：用户已登录且存在需要重签的协议
- **测试步骤**：
  1. 记录进入页面的时间戳
  2. 记录弹框显示的时间戳
  3. 重复测试10次以上统计平均时间
  4. 监控页面加载过程中的资源使用
  5. 验证弹框显示的流畅性
- **预期结果**：页面加载到弹框显示时间小于2秒，弹框显示流畅无卡顿，不影响页面正常加载速度

#### TC-015 不同平台兼容性测试
- **前置条件**：分别在任货行APP和优拓宝APP中进行测试
- **测试步骤**：
  1. 在任货行APP中完整测试重签弹框功能
  2. 在优拓宝APP中完整测试重签弹框功能
  3. 对比两个平台的功能表现和样式
  4. 验证功能逻辑的一致性
  5. 确认平台特定的设计规范符合性
- **预期结果**：两个平台的重签弹框功能表现一致，弹框样式符合各自平台设计规范，功能逻辑完全相同

## 4. 回归测试

### 4.1 原有功能回归验证

#### TC-016 登录时协议检查功能回归测试
- **前置条件**：新功能已集成到系统，存在需要签约的协议用户
- **测试步骤**：
  1. 使用需要签约协议的用户登录系统
  2. 验证登录时的协议检查机制是否正常
  3. 确认登录时协议提示功能未受影响
  4. 验证登录后的协议签约流程
  5. 确认与新增弹框功能的兼容性
- **预期结果**：原有登录时协议检查功能完全正常，与新增弹框功能兼容良好，不存在冲突

#### TC-017 协议签约流程回归测试
- **前置条件**：系统中存在各种类型的协议，用户具有签约权限
- **测试步骤**：
  1. 验证通过弹框进入的协议签约流程
  2. 验证通过其他入口的协议签约流程
  3. 确认协议内容显示的准确性
  4. 验证签约完成后的状态更新
  5. 检查签约记录的完整性
- **预期结果**：所有协议签约流程正常，协议内容准确，签约状态正确更新，记录完整

#### TC-018 页面性能回归测试
- **前置条件**：对比新功能上线前后的页面性能
- **测试步骤**：
  1. 测试首页加载时间和性能表现
  2. 测试我的模块加载时间和性能表现
  3. 对比新功能上线前后的性能差异
  4. 监控系统资源使用变化
  5. 评估性能影响的可接受性
- **预期结果**：首页和我的模块性能保持稳定，加载时间增加控制在500ms以内，系统资源使用合理
