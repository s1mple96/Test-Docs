## 关键字段需求映射

| 需求原文描述 | 对应库.表.字段 | 说明 |
|-------------|--------------|------|
| 车辆类型（客车） | rtx.rtx_customer_complaint.vehicle_type = '0' | 对应任通行/新发行客车用户 |
| 车辆类型（货车） | rtx.rtx_customer_complaint.vehicle_type = '1' | 对应任货行货车用户 |
| 未完结案件 | rtx.rtx_customer_complaint.handler_status IN ('0','1','3') | 待处理=0，跟进中=1，跟进结束=3；代码：`getNotCaseClosedStatus()` |
| 已结案 | rtx.rtx_customer_complaint.handler_status = '2' | 结案后不计入未完结 |
| 投诉来源ID | rtx.rtx_customer_complaint_label.id（label_type='1'） | 需从配置查询获取有效ID |
| 投诉标签ID | rtx.rtx_customer_complaint_label.id（label_type='0'） | 多个用英文逗号分隔 |
| 处理人ID | rtx.rtx_customer_complaint_label.id（label_type='2'） | 可不填 |
| 创建人固定为coze | rtx.rtx_customer_complaint.create_by = 'coze' | 需求：「在客诉案件中创建人为coze」；代码 `add.setCreateBy("coze")` |
| 客车用户关联 | rtx.rtx_etc_card_user.id（通过car_num查找） | vehicle_type=0时通过此表定位用户 |
| 货车用户关联 | hcb.hcb_truckuser.truckuser_id（通过car_num查找） | vehicle_type=1时通过此表定位用户 |

## 1. 校验未完结客诉案件接口

> 接口：`GET /apply/tools/customerComplaint/verifyUnresolvedCases`
> 参数：`vehicleType`（0=客车；1=货车），`carNum`（车牌号）
> 返回：`data=true` 存在未完结案件；`data=false` 不存在

  - 接口环境：`https://788360p9o5.yicp.fun/rtx-app`
  - 车牌 蒙AGTR98（vehicle_type=0）在 `rtx.rtx_customer_complaint` 中存在 handler_status IN ('0','1','3') 的案件（需求：「已存在则需要询问客服是否继续创建」）
  - 验证SQL：`SELECT id, handler_status FROM rtx.rtx_customer_complaint WHERE car_num='蒙AGTR98' AND handler_status IN ('0','1','3') LIMIT 5;`
- **测试步骤**：
  1. 发送 GET 请求：`/apply/tools/customerComplaint/verifyUnresolvedCases?vehicleType=0&carNum=蒙AGTR98`
  2. 查看响应结果
- **预期结果**：HTTP 200，响应 `code=200`，`data=true`
