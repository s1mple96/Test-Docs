# 【任通行】二手车工单业务迭代v2测试用例

## 1. 功能测试

### 1.1 客服代客下单流程

#### TC-001 后台创建二手车买卖工单-需支付
- **前置条件**：客服已登录，有工单管理权限
- **测试步骤**：
  1. 进入【客服系统-客户工单管理】-【新增】
  2. 选择业务类型"二手车买卖"，填写手机号、车牌号
  3. 勾选【是否支付注销费】为"需支付"
  4. 上传购车合同、身份证、行驶证
  5. 保存工单
- **预期结果**：
  - `rtx_customer_work_order.status='0'`(待提交)
  - `rtx_customer_work_order.source='0'`(后台下发)
  - `rtx_customer_work_order.pay_status='1'`(待支付)
  - `rtx_customer_work_order.payment_amount=0.00`
  - `rtx_customer_used_car_order.status='0'`(未支付)
  - `rtx_customer_used_car_order.source='2'`(任通行)

#### TC-001-1 后台创建二手车买卖工单-无需支付
- **前置条件**：客服已登录，有工单管理权限
- **测试步骤**：
  1. 进入【客服系统-客户工单管理】-【新增】
  2. 选择业务类型"二手车买卖"，填写手机号、车牌号
  3. 勾选【是否支付注销费】为"无需支付"
  4. 上传购车合同、身份证、行驶证
  5. 保存工单
- **预期结果**：
  - `rtx_customer_work_order.status='0'`(待提交)
  - `rtx_customer_work_order.source='0'`(后台下发)
  - `rtx_customer_work_order.pay_status='0'`(无须支付)
  - `rtx_customer_work_order.payment_amount=0.00`
  - `rtx_customer_used_car_order.status='0'`(未支付)
  - `rtx_customer_used_car_order.source='2'`(任通行)

## 核心业务流程说明

### 流程1：需支付场景
1. 后台创建工单（勾选"需支付"）→ `pay_status='1'`
2. 公众号查询获取H5链接
3. H5查询金额接口（根据 `bid_is_equipment_fee_required` 判断）
4. 用户支付 → `pay_status='4'`, `payment_amount > 0`, `out_trade_no` 有值
5. 后台审核通过 → 生成注销单
6. 或后台驳回 → 自动退款

### 流程2：无需支付场景
1. 后台创建工单（勾选"无需支付"）→ `pay_status='0'`
2. 公众号查询获取H5链接
3. H5直接提交（不查询金额，不拉起支付）
4. 后台审核通过 → 生成注销单
5. 或后台驳回 → 无需退款

### 流程3：申办已缴费场景
1. 后台创建工单（勾选"需支付"）→ `pay_status='1'`
2. 公众号查询获取H5链接
3. H5查询金额接口 → 返回0（因为 `bid_is_equipment_fee_required='1'`）
4. H5直接提交（金额为0，不拉起支付）
5. 后台审核通过 → 生成注销单
