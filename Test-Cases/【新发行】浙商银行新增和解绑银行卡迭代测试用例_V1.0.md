# 【新发行】浙商银行新增和解绑银行卡迭代测试用例

## 1. 功能测试

### 1.1 解绑银行卡校验（浙商银行卡数量校验）

#### TC-001 解绑银行卡-呼和浩特浙商-有效车辆-仅一张浙商卡-解绑失败
- **前置条件**: 
  - 系统: 新发行APP
  - 数据库: fenmi_etc
  - 场景: 用户名下有1台有效车辆（首次待激活/补换待激活/正常），外部合作银行=呼和浩特浙商（value=1）
  - 银行卡: 用户名下有且仅有1张浙商银行卡（card_status=1，del_flag=0）
  - 运营商: HHHT（呼和浩特）
  - 测试环境: 192.168.1.60
- **测试步骤**:
  1. 登录新发行APP
  2. 进入"我的银行卡"页面
  3. 选择唯一的浙商银行卡
  4. 点击"解绑"按钮
  5. 查看提示信息
- **预期结果**: 解绑失败，提示"您名下需启用至少一张浙商银行卡"，不允许解绑

### 1.3 新增绑定浙商银行卡时调用"签约结果通知"接口

#### TC-012 新增银行卡-呼和浩特浙商-绑定浙商卡-调用签约结果通知接口
- **前置条件**: 
  - 系统: 新发行APP
  - 数据库: fenmi_etc
  - 场景: 用户名下有有效车辆（首次待激活/补换待激活/正常），外部合作银行=呼和浩特浙商（value=1）
  - 银行卡: 准备新增一张浙商银行卡
  - 运营商: HHHT（呼和浩特）
  - 测试环境: 192.168.1.60
- **测试步骤**:
  1. 登录新发行APP
  2. 进入"我的银行卡"页面
  3. 点击"新增银行卡"按钮
  4. 输入浙商银行卡信息（卡号、持卡人姓名等）
  5. 点击"确认"按钮
  6. 绑定成功后，查看日志：`ssh_tail(server="test", file="/home/fenmi/fenmi-etc/log.txt", lines=200)`
  7. 搜索关键字："tetc_etcSignResult"、"签约结果通知"、"CZBankApiUtils"
- **预期结果**: 
  - 新增银行卡成功
  - 日志中显示调用"签约结果通知"接口（tetc_etcSignResult）
  - 接口参数包含：certNo（身份证号）、userId（钱包ID）、vehicleId（车辆ID）、accNo（银行卡号）、plateNum（车牌号）、plateColor（车牌颜色）、activeDate（激活日期）、shbh（商户编号：220001）
  - 接口返回成功

### 1.4 二类户产品发起ETC注销申请时调用"注销申请接口"

#### TC-017 ETC注销申请-二类户产品-呼和浩特浙商-调用注销申请接口
- **前置条件**: 
  - 系统: 新发行APP
  - 数据库: fenmi_etc
  - 场景: 用户是二类户产品，外部合作银行=呼和浩特浙商（value=1）
  - 车辆: 用户名下有有效车辆
  - 产品规则: 产品配置了"注销申请通知银行"（CANCEL_INFORMED_BANK=1）
  - 测试环境: 192.168.1.60
- **测试步骤**:
  1. 登录新发行APP
  2. 进入"车辆管理"页面
  3. 选择车辆
  4. 点击"注销申请"按钮
  5. 填写注销信息（车牌号、ETC卡号、银行卡号等）
  6. 提交注销申请
  7. 查看日志：`ssh_tail(server="test", file="/home/fenmi/fenmi-etc/log.txt", lines=200)`
  8. 搜索关键字："cancelInformedBank"、"注销申请接口"、"operType=1"
- **预期结果**: 
  - 注销申请提交成功
  - 日志中显示调用"注销申请接口"（CancelInformedBankParam，operType=1）
  - 接口参数包含：operType=1、vehicleId、userId、accNo（银行卡号）、plateNum、shbh（220001）
  - 接口返回成功
