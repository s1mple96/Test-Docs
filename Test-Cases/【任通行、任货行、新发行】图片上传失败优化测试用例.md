# 任通行、任货行、新发行图片上传失败优化测试用例

## 需求背景
货车反馈用户在进行申办时，传递的图片没有后缀名，导致报错。

## 需求内容
1. 后端在接口中对没有后缀名的文件进行拦截，并报错（接口：/common/getPresignedUrl）
2. 前端在组件使用的地方，进行提示优化，把之前的统一提示，改为优先取后端提示
3. 前端需要在H5项目中，检查后缀名的取值异常问题，参考任通行H5进行修改

**关键优化点**：
- 当前小程序上传图片失败统一提示为："上传失败,请重新上传"
- 需要修改为：优先取后端返回的错误提示，如果没有再返回统一提示

## 1. 后端接口优化测试

### 1.1 文件后缀名校验测试

#### TC-001 无后缀名文件上传拦截测试
- **前置条件**：后端已部署getPresignedUrl接口优化版本（0828上线）
- **测试步骤**：
  1. 准备无后缀名的测试文件（如"testfile"）
  2. 调用`/common/getPresignedUrl`接口上传文件
  3. 验证接口返回错误信息
  4. 检查错误信息的准确性和可读性
  5. 确认文件未被成功上传到存储系统
- **预期结果**：接口返回明确的错误提示"文件缺少后缀名"，HTTP状态码为400，文件未上传成功

#### TC-002 有效后缀名文件上传测试
- **前置条件**：后端已部署getPresignedUrl接口优化版本
- **测试步骤**：
  1. 准备有效后缀名的测试文件（如"test.jpg", "test.png", "test.pdf"）
  2. 调用`/common/getPresignedUrl`接口上传文件
  3. 验证接口正常返回预签名URL
  4. 使用预签名URL完成文件上传
  5. 验证文件上传成功
- **预期结果**：接口正常返回预签名URL，文件上传成功，后续业务流程正常

#### TC-003 特殊字符后缀名处理测试
- **前置条件**：后端已部署getPresignedUrl接口优化版本
- **测试步骤**：
  1. 准备包含特殊字符的文件名（如"test.jp g", "test..jpg", "test.JP G"）
  2. 调用`/common/getPresignedUrl`接口
  3. 验证接口对特殊字符的处理逻辑
  4. 检查是否正确识别有效后缀名
- **预期结果**：接口能正确处理特殊字符，识别有效后缀名，或给出合适的错误提示
