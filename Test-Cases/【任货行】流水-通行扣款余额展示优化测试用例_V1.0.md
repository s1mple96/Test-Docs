## 1. 功能测试

### 1.1 通行扣款余额展示-有服务费场景

#### TC-001 通行扣款关联服务费-余额取最低值
- **前置条件**: 
  - 系统: 任货行APP
  - 场景: 收支明细接口 `getRechargeList`
  - 数据准备: 存在一笔通行扣款流水(OPERATOR_TYPE=2)，关联通行服务费流水(OPERATOR_TYPE=83)
  - 数据库: hcb.hcb_etccardbalanceflow
  - 查询条件: queryType=1(3个月内)
- **测试步骤**:
  1. 调用接口查询收支明细
  2. 找到通行扣款流水记录(operatorType=2)
  3. 验证该流水关联的SOURCE_ID
  4. 查询数据库该SOURCE_ID下的通行服务费流水(OPERATOR_TYPE=83)
  5. 对比三笔流水的SURPLUS_AMOUNT: 通行扣款(7.00)、通行服务费(6.97)、代扣充值(9.00)
  6. 验证接口返回的surplusAmount字段
- **预期结果**: 
  - 接口返回的surplusAmount=6.97(三笔流水中最低值)
  - amt字段合并展示: -2.00 + (-0.03) = -2.03
  - serverAmt字段显示服务费金额: -0.03

  GROUP_CONCAT(

      ':', SURPLUS_AMOUNT

FROM hcb_etccardbalanceflow
WHERE SOURCE_ID IN (
  SELECT DISTINCT SOURCE_ID 
  FROM hcb_etccardbalanceflow 
  WHERE OPERATOR_TYPE = '2'
    AND CREATE_TIME >= '2026-01-01 00:00:00'
)
GROUP BY SOURCE_ID
HAVING COUNT(DISTINCT OPERATOR_TYPE) > 1;
