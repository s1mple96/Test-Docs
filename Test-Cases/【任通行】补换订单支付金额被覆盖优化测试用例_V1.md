### 1.2 数据一致性验证测试

#### TC-004 查询金额不一致的已支付订单
- **前置条件**：数据库中存在已支付订单
- **测试步骤**：
  1. 执行SQL查询金额不一致的订单：
     ```sql
     SELECT a.id, a.car_num, a.pay_amt, b.order_amount, b.order_status
     FROM rtx_replace_equipment a
     LEFT JOIN rtx_trade_order_202512 b ON a.id = b.biz_source_id
     WHERE a.pay_amt != b.order_amount 
       AND b.order_status = 'SUCCESS' 
       AND b.biz_type = '4'
       AND b.order_date >= '2025-12-16'
     ```
  2. 统计不一致订单数量
- **预期结果**：优化后新产生的订单无金额不一致情况

#### TC-005 批量订单金额一致性验证
- **前置条件**：系统中存在多个已支付补换订单
- **测试步骤**：
  1. 执行SQL统计金额一致性：
     ```sql
     SELECT 
       COUNT(*) as total,
       SUM(CASE WHEN a.pay_amt = b.order_amount THEN 1 ELSE 0 END) as consistent,
       SUM(CASE WHEN a.pay_amt != b.order_amount THEN 1 ELSE 0 END) as inconsistent
     FROM rtx_replace_equipment a
     LEFT JOIN rtx_trade_order_202512 b ON a.id = b.biz_source_id
     WHERE b.order_status = 'SUCCESS' AND b.biz_type = '4'
       AND b.order_date >= '2025-12-16'
     ```
  2. 计算一致性比率
- **预期结果**：一致性比率100%

### 验证SQL
```sql
-- 查询金额不一致的已支付订单
SELECT a.id, a.car_num, a.pay_amt, b.order_amount, b.order_status, b.order_date
FROM rtx_replace_equipment a
LEFT JOIN rtx_trade_order_202512 b ON a.id = b.biz_source_id
WHERE a.pay_amt != b.order_amount 
  AND b.order_status = 'SUCCESS' 
  AND b.biz_type = '4'
  AND b.order_date >= '2025-12-16'
ORDER BY b.order_date DESC;

-- 统计优化后订单金额一致性
SELECT 
  COUNT(*) as total_orders,
  SUM(CASE WHEN a.pay_amt = b.order_amount THEN 1 ELSE 0 END) as consistent_orders,
  SUM(CASE WHEN a.pay_amt != b.order_amount THEN 1 ELSE 0 END) as inconsistent_orders,
  ROUND(SUM(CASE WHEN a.pay_amt = b.order_amount THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) as consistency_rate
FROM rtx_replace_equipment a
LEFT JOIN rtx_trade_order_202512 b ON a.id = b.biz_source_id
WHERE b.order_status = 'SUCCESS' 
  AND b.biz_type = '4'
  AND b.order_date >= '2025-12-16';
```
