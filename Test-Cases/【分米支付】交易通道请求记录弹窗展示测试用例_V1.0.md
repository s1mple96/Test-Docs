# 【分米支付】交易通道请求记录弹窗展示测试用例

## 1. 功能测试

### 1.1 支付订单请求记录查询

#### TC-001 支付订单列表-显示【请求记录】按钮
- **前置条件**: 
  - 系统: 分米支付后台管理系统
  - 数据库: fenmi_pay (MySQL), jdbdb (MongoDB)
  - 页面: 支付订单列表
  - 测试环境: 192.168.1.60
- **测试步骤**:
  1. 登录分米支付后台管理系统
  2. 进入支付订单列表页面
  3. 查看列表操作列
  4. 验证每条支付订单记录都显示【请求记录】按钮
- **预期结果**: 支付订单列表每条记录的操作列都显示【请求记录】按钮

#### TC-002 支付订单-查询明通道请求记录
- **前置条件**:
  - 系统: 分米支付后台管理系统
  - 数据库: fenmi_pay, jdbdb
  - 场景: 支付订单使用明通道（如微信支付、支付宝支付）
  - 测试数据: 
    - 支付订单号: PAY202303141343000001
    - 商户支付单号: ZDKK2023031615362144
    - 明通道: 微信支付 (WeChatPay)
  - MongoDB集合: 对应明通道的请求记录集合
  - 测试环境: 192.168.1.60
- **测试步骤**:
  1. 在支付订单列表找到测试订单
  2. 点击【请求记录】按钮
  3. 系统根据订单查询明通道信息
  4. 系统根据通道确定MongoDB集合名称
  5. 系统根据商户支付单号（mch_order_no）查询MongoDB的out_order_no字段
  6. 弹窗展示请求记录列表
- **预期结果**: 
  - 弹窗正常打开
  - 显示该支付订单的所有请求记录
  - 记录按时间倒序排列

  - MongoDB响应参数中包含详细错误信息
  - 解决失败原因被覆盖的问题
