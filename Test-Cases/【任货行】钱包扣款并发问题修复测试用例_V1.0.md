# 【任货行】钱包扣款并发问题修复测试用例

## 1. 核心功能测试

### 1.1 账单扣款基础功能测试

#### TC-001 账单首次扣款-余额扣款方式优先-正常流程
- **前置条件**：钱包设置为余额扣款优先，钱包余额充足，存在待扣款账单
- **测试步骤**：
  1. 登录系统查看账单扣款前的钱包余额（记录为A）
  2. 执行账单扣款操作
  3. 查看扣款后的钱包余额（记录为B）
  4. 查询钱包流水记录，获取本次扣款金额（记录为C）
  5. 验证余额计算：B = A - C
- **预期结果**：扣款成功，钱包余额正确减少，钱包流水正确记录，账单状态更新为已扣款

#### TC-002 账单首次扣款-银行卡扣款方式优先-正常流程
- **前置条件**：钱包设置为银行卡扣款优先，已绑定银行卡，存在待扣款账单
- **测试步骤**：
  1. 登录系统查看账单扣款前的钱包余额（记录为A）
  2. 执行账单扣款操作
  3. 查看扣款后的钱包余额（记录为B）
  4. 查询钱包流水记录
  5. 查询账单扣款记录，确认扣款来源为银行卡
  6. 验证钱包余额未变化：B = A
- **预期结果**：银行卡扣款成功，钱包余额不变，账单状态更新为已扣款，扣款记录来源为银行卡

#### TC-003 账单扣款-余额不足自动切换银行卡扣款
- **前置条件**：钱包设置为余额扣款优先，钱包余额不足，已绑定银行卡，存在待扣款账单
- **测试步骤**：
  1. 设置钱包余额小于账单金额
  2. 执行账单扣款操作
  3. 查看钱包余额是否变化
  4. 查询账单扣款记录，确认扣款来源
- **预期结果**：自动切换为银行卡扣款，钱包余额不变，账单状态更新为已扣款

### 1.2 充值入金基础功能测试

#### TC-004 钱包充值-正常流程
- **前置条件**：用户钱包状态正常
- **测试步骤**：
  1. 查看充值前的钱包余额（记录为A）
  2. 执行钱包充值操作，充值金额为X
  3. 充值完成后查看钱包余额（记录为B）
  4. 查询钱包流水记录，获取本次充值金额（记录为C）
  5. 验证余额计算：B = A + C，且C = X
- **预期结果**：充值成功，钱包余额正确增加，钱包流水正确记录

## 2. 并发场景测试（核心）

### 2.1 账单扣款与充值并发测试

#### TC-005 单个钱包-账单扣款与充值同时发生-余额一致性验证
- **前置条件**：钱包余额为1000元，存在待扣款账单（金额200元）
- **测试步骤**：
  1. 记录初始钱包余额：1000元
  2. 同时发起账单扣款（200元）和充值操作（500元）
  3. 等待两个操作都完成
  4. 查询最终钱包余额（记录为finalBalance）
  5. 查询钱包流水记录，统计所有流水金额（记录为flowSum）
  6. 验证：finalBalance = 1000 - 200 + 500 = 1300元
  7. 验证：钱包流水相加 = finalBalance - 初始余额
- **预期结果**：最终余额为1300元，钱包流水记录完整，流水计算与余额一致，无数据覆盖

#### TC-006 单个钱包-多次账单扣款并发-余额一致性验证
- **前置条件**：钱包余额为5000元，钱包设置为余额扣款优先，存在3个待扣款账单（金额分别为100、200、300元）
- **测试步骤**：
  1. 记录初始钱包余额：5000元
  2. 同时发起3个账单的扣款操作
  3. 等待所有扣款操作完成
  4. 查询最终钱包余额（记录为finalBalance）
  5. 查询钱包流水记录，统计扣款流水金额总和（记录为deductSum）
  6. 验证：finalBalance = 5000 - 100 - 200 - 300 = 4400元
  7. 验证：钱包流水扣款总额 = 600元
  8. 验证：所有账单状态均为已扣款
- **预期结果**：最终余额为4400元，3笔扣款流水记录完整，流水计算与余额一致，3个账单均扣款成功

#### TC-007 单个钱包-账单扣款与充值高频并发-余额一致性验证
- **前置条件**：钱包余额为10000元，存在5个待扣款账单，每个账单金额为200元
- **测试步骤**：
  1. 记录初始钱包余额：10000元
  2. 在5秒内发起10次操作：5次账单扣款 + 5次充值（每次充值500元）
  3. 等待所有操作完成
  4. 查询最终钱包余额（记录为finalBalance）
  5. 查询钱包流水记录，统计扣款总额和充值总额
  6. 验证：finalBalance = 10000 - (5×200) + (5×500) = 11500元
  7. 验证：钱包流水相加 = finalBalance - 初始余额 = 1500元
- **预期结果**：最终余额为11500元，10笔流水记录完整无遗漏，流水计算与余额一致

#### TC-008 多个钱包-同时进行账单扣款与充值-互不影响验证
- **前置条件**：准备5个不同的钱包账户，每个钱包余额为2000元，各有1个待扣款账单（金额300元）
- **测试步骤**：
  1. 记录5个钱包的初始余额
  2. 同时对5个钱包发起扣款操作
  3. 同时对5个钱包发起充值操作（每个充值1000元）
  4. 等待所有操作完成
  5. 分别查询5个钱包的最终余额
  6. 验证每个钱包：余额 = 2000 - 300 + 1000 = 2700元
  7. 验证每个钱包的流水记录完整性
- **预期结果**：5个钱包余额均为2700元，每个钱包流水记录独立完整，钱包之间互不影响

### 2.2 银行卡扣款并发测试

### 2.3 混合扣款并发测试

- **前置条件**：钱包余额为500元，存在2个待扣款账单（每个金额400元），均设置余额扣款
- **测试步骤**：
  1. 记录初始钱包余额：500元
  2. 同时发起2个账单的扣款操作
  3. 等待扣款操作完成
  4. 查询最终钱包余额
  5. 查询两个账单的状态
  6. 查询钱包流水记录
- **预期结果**：只有一个账单扣款成功，余额为100元；另一个账单扣款失败或转为银行卡扣款，不会出现余额为负数

#### TC-013 并发充值与扣款-余额从不足到充足
- **前置条件**：钱包余额为100元，存在待扣款账单（金额500元，余额扣款优先）
- **测试步骤**：
  1. 记录初始钱包余额：100元
  2. 同时发起充值操作（金额1000元）和账单扣款操作

  5. 查询账单扣款方式和状态
- **预期结果**：根据并发先后顺序，余额最终为600元（充值先完成）或100元（扣款因余额不足转银行卡），数据一致性正确

### 3.2 极限并发测试

- **前置条件**：钱包余额为100000元，准备50个待扣款账单（每个金额100元），准备50次充值操作（每次金额200元）

  5. 查询钱包流水记录总数

  7. 验证：最终余额 = 100000 - 扣款总额 + 充值总额

- **预期结果**：所有操作成功，余额计算正确（105000元），流水记录完整（100条），无数据丢失或覆盖

## 4. 数据一致性验证测试

### 4.1 钱包余额与流水一致性验证

#### TC-015 钱包余额计算验证-单日所有流水
- **前置条件**：选择测试当日有多笔扣款和充值操作的钱包账户
- **测试步骤**：
  1. 查询钱包当日起始余额（记录为startBalance）
  2. 查询钱包当日所有流水记录
  3. 分别统计充值流水总额（记录为rechargeSum）和扣款流水总额（记录为deductSum）
  4. 查询钱包当日结束余额（记录为endBalance）
  5. 验证：endBalance = startBalance + rechargeSum - deductSum
- **预期结果**：余额计算完全一致，误差为0

#### TC-016 钱包流水完整性验证-对比业务记录
- **前置条件**：选择有多笔扣款和充值操作的钱包账户
- **测试步骤**：
  1. 查询钱包所有扣款流水记录（记录总数为A）
  2. 查询该钱包对应的所有已扣款账单记录（记录总数为B，筛选余额扣款方式）
  3. 查询钱包所有充值流水记录（记录总数为C）
  4. 查询该钱包对应的所有充值订单记录（记录总数为D）
  5. 验证：A = B，C = D
  6. 逐笔核对金额是否一致
- **预期结果**：流水记录与业务记录完全一致，无遗漏，无重复

#### TC-017 日终余额校验功能-并发场景下预警准确性
- **前置条件**：系统已上线日终余额校验功能，当日有并发扣款和充值操作
- **测试步骤**：
  1. 执行多次并发测试（TC-005至TC-014）
  2. 等待日终余额校验任务执行
  3. 查询日终余额校验预警结果
  4. 验证预警钱包列表是否为空
  5. 如有预警，逐个核对是否为真实的数据不一致
- **预期结果**：日终余额校验无预警，或预警为真实的历史遗留问题，新产生的并发操作不会导致余额不一致
