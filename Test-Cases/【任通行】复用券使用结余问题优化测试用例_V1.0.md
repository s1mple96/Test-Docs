## 1. 测试背景

### 1.1 需求背景
【任通行】系统在复用券使用过程中存在使用结余状态处理不完善的问题，主要体现在：
1. 过期券的定时任务没包含使用结余状态，导致券不会过期
2. 小程序端没展示使用结余的券，导致用户看不到券及余额
3. 使用结余的券锁定后再开启可能无法扣费

### 1.2 优化目标
- 完善定时任务逻辑，确保使用结余券能正常过期
- 优化小程序显示逻辑，让用户能看到使用结余券
- 修复锁定解锁扣费逻辑，确保功能正常

### 1.3 涉及模块
- RTX优惠券管理模块
- 定时任务调度模块  
- 小程序接口模块
- 扣费核心逻辑模块

### 3.4 TC004-数据一致性综合验证

#### 3.4.1 用例信息
| 项目 | 内容 |
|------|------|
| **用例编号** | TC004 |
| **用例标题** | 使用结余券数据一致性综合验证 |
| **用例级别** | P1 |
| **前置条件** | 系统中存在各种状态的券数据 |

#### 3.4.2 测试步骤
| 步骤 | 操作描述 | 预期结果 |
|------|----------|----------|
| 1 | 统计各状态券的数量分布 | 获得完整的状态分布图 |
| 2 | 检查使用结余券(status=8)的余额情况 | 所有status=8的券余额>0 |
| 3 | 验证余额字段一致性 | coupon_balance = remaining_amount |
| 4 | 检查过期时间与状态匹配性 | 已过期券不应该是status=8 |
| 5 | 验证复用券标识正确性 | status=8的券reuse_flag=1 |

#### 3.4.3 数据校验SQL
```sql
-- 1. 状态分布统计
SELECT status, COUNT(*) as count,
       CASE status WHEN -1 THEN '待发放' WHEN 0 THEN '未使用' 
                   WHEN 1 THEN '使用中' WHEN 2 THEN '已使用'
                   WHEN 3 THEN '已失效' WHEN 8 THEN '使用结余'
                   ELSE '其他' END as status_name
FROM rtx_etc_user_coupon_record GROUP BY status;

-- 2. 使用结余券余额检查
SELECT COUNT(*) as invalid_balance_count
FROM rtx_etc_user_coupon_record 
WHERE status = 8 AND (coupon_balance <= 0 OR remaining_amount <= 0);

-- 3. 余额一致性检查
SELECT COUNT(*) as inconsistent_count
FROM rtx_etc_user_coupon_record 
WHERE status = 8 AND ABS(coupon_balance - remaining_amount) > 0.01;

-- 4. 过期状态检查
SELECT COUNT(*) as expired_balance_count
FROM rtx_etc_user_coupon_record 
WHERE status = 8 AND coupon_expired_date < NOW();
```
