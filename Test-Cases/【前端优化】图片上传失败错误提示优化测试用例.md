# 前端图片上传失败错误提示优化测试用例

## 需求背景
货车反馈用户在进行申办时，传递的图片没有后缀名，导致报错。

## 需求内容
**前端优化重点**：把之前的统一提示"上传失败,请重新上传"，改为优先取后端提示

**关键优化逻辑**：
- **原来**：统一提示"上传失败,请重新上传"
- **现在**：优先显示后端返回的具体错误提示，如果没有再返回统一提示

## 测试用例总结

本测试用例文档共包含 **10个测试案例**，专门针对前端错误提示优化功能：

- **第一批优先发版测试**：4个测试案例
- **第二批发版测试**：1个测试案例
- **无需发版验证**：1个测试案例
- **回归测试**：2个测试案例
- **验收测试**：2个测试案例

**测试覆盖范围**：
- **第一批项目**：12个（小程序6个 + H5项目6个）
- **第二批项目**：13个（全部为H5推广项目）
- **无需发版项目**：4个（确认状态）

**核心验证重点**：
1. **错误提示优先级**：优先显示后端具体错误信息，无错误信息时降级显示统一提示
2. **业务场景区分**：申办流程证件上传 vs 个人中心头像上传
3. **平台一致性**：小程序与H5版本错误提示逻辑保持一致
4. **用户体验提升**：从模糊提示改为具体问题说明

**重点解决**：货车用户反馈的"传递的图片没有后缀名导致报错"问题，让用户能明确知道具体的错误原因，显著提升申办流程的用户体验。
