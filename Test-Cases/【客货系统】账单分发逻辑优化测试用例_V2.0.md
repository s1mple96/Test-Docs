# 【客货系统】账单分发逻辑优化测试用例 V2.0

## 测试背景

**需求版本**: V2.0 (基于20250828发版后的问题修复)  
**测试目标**: 验证优化后的账单分发逻辑，解决车牌颜色不一致导致的账单分发错误问题  
**测试范围**: 任货行账单分发逻辑的完整流程  
**数据库**: hcb数据库 (使用mysql-hcb工具)  
**目标运营商**: 广西JTK、华软三省(MTK/LTK/XTK)  

## 核心业务逻辑

### 新的账单分发流程：
1. **ETC卡号直查用户** → 找到则落库任货行 + 车牌颜色一致性检查
2. **ETC卡号查库存** → 判断客车/货车属性
3. **库存货车判断** → 车牌号查用户 + 颜色一致性处理
4. **颜色一致优先** → 相同颜色用户优先，按状态排序
5. **颜色不一致容错** → 不同颜色用户容错，按状态排序 + 报警
6. **库存未找到兜底** → 车牌号+颜色+运营商查询
