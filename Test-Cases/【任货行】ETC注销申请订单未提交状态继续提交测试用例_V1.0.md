# 【任货行】ETC注销申请订单未提交状态继续提交测试用例

## 1. 功能测试

### 1.1 正常流程测试

#### TC-001 未提交订单显示继续提交按钮
- **前置条件**：
  - 用户已登录任货行小程序
  - 存在STATUS='0'(未提交)的注销申请订单
  - 数据准备：`UPDATE hcb_hlj_cancelbussine SET STATUS='0' WHERE CANCELBUSSINE_ID='test001';`
- **测试步骤**：
  1. 进入"我的订单"或"注销申请列表"页面
  2. 查看未提交状态订单
  3. 验证是否显示"继续提交"按钮
- **预期结果**：未提交订单显示"继续提交"按钮

#### TC-002 点击继续提交跳转至提交页面
- **前置条件**：
  - TC-001执行成功
  - 订单数据：CANCELBUSSINE_ID='test001', PLATE_NUM='川BBB2A2', STATUS='0'
- **测试步骤**：
  1. 点击订单的"继续提交"按钮
  2. 验证页面跳转
  3. 验证车牌号是否自动填充
- **预期结果**：
  - 跳转至注销申请提交页面
  - 车牌号自动填充为"川BBB2A2"

#### TC-003 继续提交后数据完整性
- **前置条件**：
  - TC-002执行成功
  - 在提交页面填写其他必填项
- **测试步骤**：
  1. 检查页面是否带入原订单数据（车牌号、身份证号等）
  2. 补充填写其他信息（注销原因、图片等）
  3. 提交订单
  4. 查询数据库验证status变更
- **预期结果**：
  - 原订单数据正确带入
  - 提交成功，STATUS从'0'变更为'3'(已提交)
  - 数据完整性校验：`SELECT * FROM hcb_hlj_cancelbussine WHERE CANCELBUSSINE_ID='test001';`

#### TC-004 多个未提交订单场景
- **前置条件**：
  - 用户有2个未提交订单
  ```sql
  UPDATE hcb_hlj_cancelbussine SET STATUS='0' WHERE CANCELBUSSINE_ID IN ('test001', 'test002');
  ```
- **测试步骤**：
  1. 进入订单列表
  2. 验证两个订单都显示"继续提交"按钮
  3. 分别点击不同订单的按钮
  4. 验证带入的车牌号是否正确
- **预期结果**：
  - 所有未提交订单都显示按钮
  - 点击不同订单带入对应车牌号

### 1.2 异常处理测试

#### TC-005 已提交订单不显示继续提交按钮
- **前置条件**：
  - 订单STATUS='3'(已提交/待审核)
  - 数据准备：`SELECT * FROM hcb_hlj_cancelbussine WHERE STATUS='3' LIMIT 1;`
- **测试步骤**：
  1. 进入订单列表
  2. 查看已提交状态订单
  3. 验证按钮显示情况
- **预期结果**：已提交订单不显示"继续提交"按钮

#### TC-006 审核中订单不显示继续提交按钮
- **前置条件**：订单STATUS='4'(审核中)
- **测试步骤**：
  1. 进入订单列表
  2. 查看审核中状态订单
- **预期结果**：审核中订单不显示"继续提交"按钮

#### TC-007 车牌号为空的未提交订单
- **前置条件**：
  - 订单STATUS='0', PLATE_NUM=NULL
  ```sql
  UPDATE hcb_hlj_cancelbussine SET STATUS='0', PLATE_NUM=NULL WHERE CANCELBUSSINE_ID='test003';
  ```
- **测试步骤**：
  1. 点击继续提交按钮
  2. 进入提交页面
  3. 验证车牌号输入框
- **预期结果**：
  - 车牌号输入框为空
  - 允许用户重新输入车牌号

#### TC-008 订单数据不完整场景
- **前置条件**：
  - 订单STATUS='0', PIC_URL为空
  ```sql
  UPDATE hcb_hlj_cancelbussine SET STATUS='0', PIC_URL=NULL WHERE CANCELBUSSINE_ID='test001';
  ```
- **测试步骤**：
  1. 点击继续提交
  2. 检查页面数据
  3. 补充缺失数据后提交
- **预期结果**：
  - 缺失字段为空或默认值
  - 补充后可正常提交

#### TC-009 订单被删除或不存在
- **前置条件**：订单CANCELBUSSINE_ID不存在或已被删除
- **测试步骤**：
  1. 构造不存在的订单CANCELBUSSINE_ID
  2. 尝试访问提交页面
- **预期结果**：提示"订单不存在"或跳转至订单列表

#### TC-010 并发提交处理
- **前置条件**：同一未提交订单
- **测试步骤**：
  1. 同时在两个终端打开同一订单
  2. 几乎同时点击"继续提交"
  3. 在两个页面都进行提交操作
- **预期结果**：
  - 第一次提交成功，STATUS变为'3'
  - 第二次提交失败或提示订单已提交

### 1.3 边界值测试

#### TC-011 车牌号长度边界测试
- **前置条件**：各种长度的车牌号
  - 最短：7位（如"川A12345"）
  - 最长：8位（如"川A12345D"）
  - 新能源：8位（如"川AD12345"）
- **测试步骤**：
  1. 使用不同长度车牌号创建未提交订单
  2. 点击继续提交
  3. 验证车牌号正确带入
- **预期结果**：所有合法长度车牌号都能正确带入

#### TC-012 特殊字符车牌号
- **前置条件**：
  - 订单车牌号包含特殊字符（如：川A•12345）
- **测试步骤**：
  1. 点击继续提交
  2. 验证车牌号显示
- **预期结果**：特殊字符正确显示或转义处理

#### TC-013 状态值边界测试
- **前置条件**：
  - 测试STATUS='0'、'1'、'2'等各种状态
  ```sql
  SELECT DISTINCT STATUS FROM hcb_hlj_cancelbussine;
  ```
- **测试步骤**：
  1. 分别测试不同STATUS值的订单
  2. 验证哪些状态显示"继续提交"按钮
- **预期结果**：仅STATUS='0'显示按钮

## 3. 数据库测试

#### TC-016 数据查询准确性
- **前置条件**：数据库包含各种状态订单
- **测试步骤**：
  1. 执行查询：`SELECT COUNT(*) FROM hcb_hlj_cancelbussine WHERE STATUS='0';`

- **预期结果**：数据库记录数与小程序显示一致

#### TC-017 数据更新正确性
- **前置条件**：未提交订单CANCELBUSSINE_ID='test001'
- **测试步骤**：
  1. 继续提交前查询：`SELECT STATUS, UPDATE_TIME FROM hcb_hlj_cancelbussine WHERE CANCELBUSSINE_ID='test001';`
  2. 提交订单
  3. 再次查询STATUS和UPDATE_TIME
- **预期结果**：
  - STATUS从'0'变为'3'
  - UPDATE_TIME已更新

#### TC-018 关联字段验证
- **前置条件**：订单包含TRUCKUSER_ID、USER_ID
- **测试步骤**：
  1. 查询订单关联数据
  ```sql
  SELECT c.*, t.plate_num FROM hcb_hlj_cancelbussine c 
  LEFT JOIN hcb_truckuser t ON c.TRUCKUSER_ID=t.TRUCKUSER_ID 
  WHERE c.CANCELBUSSINE_ID='test001';
  ```
  2. 验证数据一致性
- **预期结果**：关联数据正确

## 测试数据准备SQL

```sql
-- 1. 创建未提交订单（用于TC-001到TC-004）
UPDATE hcb_hlj_cancelbussine 
SET STATUS='0', PLATE_NUM='川BBB2A2', IDCODE='413026198910256611' 
WHERE CANCELBUSSINE_ID='0039494b4ac14c4491de1fa11ad2bda9';

-- 2. 创建多个未提交订单
UPDATE hcb_hlj_cancelbussine 
SET STATUS='0' 
WHERE CANCELBUSSINE_ID IN ('0039494b4ac14c4491de1fa11ad2bda9', '00d0675f7b294c99ab65a714c41a1796');

-- 3. 创建车牌号为空的订单（用于TC-007）
UPDATE hcb_hlj_cancelbussine 
SET STATUS='0', PLATE_NUM=NULL 
WHERE CANCELBUSSINE_ID='01275efd7a364be3811dabc3a68e1271';

-- 4. 创建数据不完整订单（用于TC-008）
UPDATE hcb_hlj_cancelbussine 
SET STATUS='0', PIC_URL=NULL, CANCEL_REASON=NULL 
WHERE CANCELBUSSINE_ID='0039494b4ac14c4491de1fa11ad2bda9';

-- 5. 查询各状态订单数量
SELECT STATUS, COUNT(*) as count 
FROM hcb_hlj_cancelbussine 
GROUP BY STATUS;

-- 6. 查询未提交订单详情
SELECT CANCELBUSSINE_ID, LIST_NO, PLATE_NUM, STATUS, CREATE_TIME, UPDATE_TIME 
FROM hcb_hlj_cancelbussine 
WHERE STATUS='0' 
ORDER BY CREATE_TIME DESC 
LIMIT 10;
```
