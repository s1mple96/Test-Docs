# 【任通行】通行费多扣退款申请流程测试用例

## 1. 功能测试

### 1.1 流程配置测试

#### TC-001 通行费多扣退款流程配置
- **前置条件**：管理员权限，进入流程列表菜单
- **测试步骤**：
  1. 查看流程列表中是否存在业务类型：通行费多扣退款申请
  2. 查看审批流程配置
  3. 验证审核节点：客服一审→客服二审→财务一审→财务二审
  4. 验证审核模式为"或签"（有一人同意即可）
- **预期结果**：流程配置正确，审核模式为或签

#### TC-002 按钮权限验证
- **前置条件**：客服账号登录，进入客户信息-业务流程申请单
- **测试步骤**：
  1. 查询客车类型车辆
  2. 查看是否显示"通行费多扣退款申请"按钮
  3. 查询货车类型车辆
  4. 查看是否显示"通行费多扣退款申请"按钮
- **预期结果**：客车显示按钮（权限：bpmProcess:audit:tollRefund），货车不显示

### 1.2 申请单创建测试

#### TC-003 正常创建申请单-路费和服务费均退
- **前置条件**：客服账号，选择客车车辆

  1. 点击"通行费多扣退款申请"按钮
  2. 填写退款路费：100.00元
  3. 填写退通行服务费：50.00元

  5. 填写申请说明
  6. 提交申请
  7. 查询rtx_bpm_toll_refund表
  8. 查询rtx_bpm_process_management表
- **预期结果**：申请单创建成功，toll_amt=100.00，service_amt=50.00，total_amt=150.00，status=0（待审核）
