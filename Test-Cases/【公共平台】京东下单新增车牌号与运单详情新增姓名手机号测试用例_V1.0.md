## 1. 快递面单展示车牌号

### 1.1 正常场景

#### TC-001 补换订单下单 - 快递面单备注字段携带车牌号

- **前置条件**：
  - 系统：公共平台管理后台
  - 场景：补换订单已审批通过，补换方式为快递邮寄，使用京东物流（biz_source 为 JD 渠道）
  - 数据：`etc_replace_equipment` 存在一条 `car_num` 不为空的补换记录，`return_info_id` 已关联 `etc_rquipment_return_info`
  - 测试环境：192.168.1.60
- **测试步骤**：
  1. 确认补换记录的车牌号，查询：`SELECT id, car_num FROM etc_replace_equipment WHERE id = {id};`
  2. 在管理后台触发重新下单或后台流程触发京东下单
  3. 下单成功后，查询 `etc_rquipment_return_info.return_express_no` 获取运单号
  4. 用运单号到京东物流官网或后台查询运单打印预览，确认快递面单备注栏
- **预期结果**：
  - 京东快递面单的备注字段（remark）中显示车牌号，与 `etc_replace_equipment.car_num` 一致
  - 下单接口调用日志中可见 `remark` 字段值为车牌号

#### TC-002 补换订单下单 - 车牌号含特殊格式（新能源车牌）

- **前置条件**：
  - 系统：公共平台管理后台
  - 场景：补换订单的车辆为新能源车，车牌格式如 `粤AD12345`（6位）
  - 数据：`etc_replace_equipment.car_num` 为新能源车牌格式
- **测试步骤**：
  1. 查询新能源车牌的补换记录：`SELECT id, car_num FROM etc_replace_equipment WHERE LENGTH(car_num) = 8 LIMIT 5;`
  2. 触发该订单的京东下单流程
  3. 查看运单打印预览，确认面单备注
- **预期结果**：
  - 新能源车牌（如 `粤AD12345`）完整显示在面单备注字段中，字符不截断

#### TC-003 补换订单重新下单 - 面单备注仍携带车牌号

- **前置条件**：
  - 系统：公共平台管理后台
  - 场景：补换订单因某种原因需重新下单（旧运单取消后重新创建）
  - 数据：已存在的补换订单，`car_num` 不为空
- **测试步骤**：
  1. 找到一条已下单的京东补换订单，操作取消快递
  2. 重新触发下单
  3. 查看新运单的快递面单备注字段
- **预期结果**：
  - 重新下单后，面单备注字段依然正确携带车牌号，无丢失

### 1.2 异常场景

#### TC-004 补换订单下单 - car_num 为空时面单备注为空

- **前置条件**：
  - 场景：`etc_replace_equipment.car_num` 为空（异常数据，理论不应出现）
  - 数据：临时构造 `car_num` 为 NULL 或空字符串的记录
- **测试步骤**：
  1. 触发京东下单流程
  2. 查看下单请求日志中 `remark` 字段值
  3. 查看面单备注栏
- **预期结果**：
  - 下单请求中 `remark` 字段为空或 null，面单备注栏无异常内容，不影响下单成功

### 1.3 回归测试

#### TC-005 京东下单其他字段不受影响

- **前置条件**：
  - 系统：公共平台管理后台
  - 场景：正常补换京东下单流程
- **测试步骤**：
  1. 触发京东下单
  2. 验证 senderContact（寄件人姓名、手机、地址）正确

#### TC-015 数据库数据验证 - 车牌号来源正确

- **前置条件**：
  - 测试环境数据库可查询
- **测试步骤**：
  1. 查询已成功创建京东运单的补换订单：
     ```sql
     SELECT e.id, e.car_num, r.return_express_no, r.return_send_name, r.return_send_phone, r.return_receive_name, r.return_receive_phone
     FROM etc_replace_equipment e
     JOIN etc_rquipment_return_info r ON e.return_info_id = r.id
     WHERE r.return_express_company = '京东' AND r.return_express_no IS NOT NULL
     LIMIT 10;
     ```
  2. 确认 `car_num` 字段有值
  3. 确认 `return_send_name`、`return_send_phone`、`return_receive_name`、`return_receive_phone` 字段数据完整
- **预期结果**：
  - 查询结果显示车牌号、寄收件人姓名和手机号均有值
  - 数据符合预期，无明显数据异常
