# 任货行主动还款防重处理测试用例

## 1. 批量主动还款功能测试

### 1.1 批量还款基础功能测试

#### TC-001 批量主动还款正常流程测试
- **前置条件**：用户钱包余额充足，存在多笔待还款账单，账单状态为正常待扣款状态
- **测试步骤**：
  1. 准备3-5笔测试账单数据，设置用户钱包余额充足
  2. 调用批量主动还款接口，传入账单列表
  3. 验证还款处理结果和返回信息
  4. 查询账单状态变化和钱包余额变化
  5. 检查`hcb_bill_check_trade_data`表中的记录
- **预期结果**：所有账单扣款成功，钱包余额正确扣减，账单状态更新为已支付，`hcb_bill_check_trade_data`表中每笔订单只有一条记录

#### TC-002 批量主动还款部分成功测试
- **前置条件**：用户钱包余额仅够支付部分账单，存在多笔待还款账单
- **测试步骤**：
  1. 准备5笔测试账单，总金额设置为大于钱包余额
  2. 调用批量主动还款接口
  3. 验证部分成功的处理逻辑和返回结果
  4. 检查成功和失败账单的状态分别更新
  5. 验证钱包余额的正确扣减
- **预期结果**：余额足够的账单扣款成功，余额不足的账单保持原状态，钱包余额正确扣减，成功的订单在`hcb_bill_check_trade_data`表中有记录，失败的订单不产生重复记录

#### TC-003 批量主动还款余额不足测试
- **前置条件**：用户钱包余额为0或不足，存在多笔待还款账单
- **测试步骤**：
  1. 设置钱包余额为0或小于最小账单金额
  2. 调用批量主动还款接口
  3. 验证处理结果和错误信息
  4. 检查钱包余额和账单状态是否保持不变
- **预期结果**：所有账单扣款失败，钱包余额不变，账单状态不变，`hcb_bill_check_trade_data`表中不产生记录

### 1.2 批量还款异常处理测试

#### TC-004 批量主动还款无效账单测试
- **前置条件**：传入已支付或无效的账单号
- **测试步骤**：
  1. 准备包含已支付账单号的请求参数
  2. 调用批量主动还款接口
  3. 验证系统对无效账单的处理逻辑
  4. 检查错误提示信息的准确性
  5. 确认不会产生重复扣款
- **预期结果**：接口返回相应错误信息，不产生重复扣款，`hcb_bill_check_trade_data`表中不产生重复记录

#### TC-005 批量主动还款账单状态校验测试
- **前置条件**：存在不同状态的账单（待扣款、扣款中、已支付、已失败）
- **测试步骤**：
  1. 准备不同状态的账单列表
  2. 调用批量主动还款接口
  3. 验证系统对不同状态账单的处理逻辑
  4. 检查状态流转的正确性
- **预期结果**：只有符合条件的账单被处理，已支付账单不被重复扣款，状态更新符合业务逻辑

## 2. 防重处理功能测试

### 2.1 重复请求防重测试

#### TC-006 相同参数重复调用防重测试
- **前置条件**：用户钱包余额充足，存在待还款账单
- **测试步骤**：
  1. 准备测试账单数据，记录调用前状态
  2. 第一次调用批量主动还款接口
  3. 立即使用相同参数再次调用接口
  4. 验证防重处理效果和返回结果
  5. 检查钱包余额和流水记录
- **预期结果**：第一次调用成功扣款，第二次调用被防重机制拦截，钱包只扣款一次，`hcb_bill_check_trade_data`表中每笔订单只有一条记录

#### TC-007 部分重复账单防重测试
- **前置条件**：用户钱包余额充足，第一次还款成功部分账单
- **测试步骤**：
  1. 第一次提交5笔账单，确保成功还款3笔
  2. 第二次提交包含已成功账单的请求
  3. 验证防重处理逻辑对部分重复的处理
  4. 检查已成功和未成功账单的不同处理
- **预期结果**：已成功的账单不再重复扣款，未成功的账单正常处理，`hcb_bill_check_trade_data`表中不产生重复记录

#### TC-008 时间窗口防重测试
- **前置条件**：用户钱包余额充足，防重时间窗口设置（如5分钟）
- **测试步骤**：
  1. 第一次调用批量主动还款接口
  2. 在防重时间窗口内再次调用相同请求
  3. 等待防重时间窗口过期后再次调用
  4. 验证不同时间点的处理逻辑
  5. 检查防重机制的时效性
- **预期结果**：防重时间窗口内的重复请求被拦截，时间窗口过期后的请求正常处理（如账单状态允许），数据记录符合防重逻辑

### 2.2 防重机制验证测试

#### TC-009 防重键值生成逻辑测试
- **前置条件**：了解防重键值的生成规则
- **测试步骤**：
  1. 使用相同参数多次生成防重键值
  2. 验证防重键值的一致性
  3. 使用不同参数生成防重键值
  4. 验证防重键值的唯一性
  5. 测试参数顺序变化对防重键值的影响
- **预期结果**：相同参数生成相同防重键值，不同参数生成不同防重键值，防重机制能正确识别重复请求

#### TC-010 防重记录清理机制测试
- **前置条件**：防重记录有过期清理机制
- **测试步骤**：
  1. 执行批量主动还款生成防重记录
  2. 等待防重记录过期时间
  3. 验证过期记录是否被正确清理
  4. 测试清理后相同请求的处理
- **预期结果**：过期的防重记录被正确清理，不影响后续正常请求的处理

## 3. 任务首次扣款失败后主动还款测试

### 3.1 定时扣款与主动还款衔接测试

#### TC-011 任务扣款失败转主动还款测试
- **前置条件**：用户钱包余额不足，存在待扣款账单，任务首次扣款定时任务正常运行
- **测试步骤**：
  1. 设置用户钱包余额不足
  2. 等待任务首次扣款定时任务执行
  3. 验证账单被标记为扣款失败状态
  4. 用户充值钱包余额到充足状态
  5. 调用批量主动还款接口处理失败账单
  6. 验证完整流程的处理结果
- **预期结果**：任务首次扣款标记账单为失败状态，主动还款成功处理失败账单，钱包流水记录正确，账单状态最终更新为已支付，`hcb_bill_check_trade_data`表中数据正确

#### TC-012 多次扣款失败后主动还款测试
- **前置条件**：账单经历过多次扣款失败，用户钱包现有充足余额
- **测试步骤**：
  1. 模拟账单多次扣款失败的历史记录
  2. 调用批量主动还款接口
  3. 验证历史失败记录不影响主动还款
  4. 检查主动还款基于当前状态的处理逻辑
- **预期结果**：主动还款忽略历史失败记录，基于当前状态正常处理，不产生重复扣款

#### TC-013 定时扣款与主动还款状态冲突处理测试
- **前置条件**：定时扣款任务和主动还款可能同时处理同一账单
- **测试步骤**：
  1. 准备处于扣款处理中状态的账单
  2. 在定时任务处理过程中调用主动还款
  3. 验证状态冲突的处理机制
  4. 检查最终的数据一致性
- **预期结果**：系统能正确处理状态冲突，避免重复扣款，确保数据一致性

### 3.2 业务流程完整性测试

#### TC-014 账单状态一致性验证测试
- **前置条件**：完整的账单状态流转链路
- **测试步骤**：
  1. 追踪账单从创建到最终支付的完整状态流转
  2. 验证定时扣款失败对状态的影响
  3. 验证主动还款成功对状态的恢复
  4. 检查状态变更的时间点和触发条件
- **预期结果**：账单状态流转逻辑清晰正确，每次状态变更都有明确的触发条件和时间记录

#### TC-015 钱包流水完整性测试
- **前置条件**：涉及定时扣款失败和主动还款成功的完整流程
- **测试步骤**：
  1. 记录定时扣款失败时的钱包流水
  2. 记录主动还款成功时的钱包流水
  3. 验证流水记录的完整性和准确性
  4. 检查流水类型和关联关系
- **预期结果**：钱包流水完整记录整个处理过程，流水类型准确，关联关系正确

## 4. 并发处理测试

### 4.1 并发防重测试

#### TC-016 相同账单并发还款测试
- **前置条件**：用户钱包余额充足，测试环境支持并发请求
- **测试步骤**：
  1. 准备相同的账单列表
  2. 同时发起多个（3-5个）批量主动还款请求
  3. 验证并发防重处理效果
  4. 检查并发请求的响应结果
  5. 验证最终的数据一致性
- **预期结果**：只有一个请求成功处理，其他请求被防重机制拦截，钱包只扣款一次，`hcb_bill_check_trade_data`表中每笔账单只有一条记录

#### TC-017 不同账单并发还款测试
- **前置条件**：用户钱包余额充足，准备不同的账单组合
- **测试步骤**：
  1. 准备不同的账单列表（无重复账单）
  2. 同时发起多个批量主动还款请求
  3. 验证并发处理能力和响应时间
  4. 检查所有账单的处理结果
  5. 验证钱包余额和流水记录的准确性
- **预期结果**：所有请求都应该成功处理，钱包余额正确扣减，所有账单状态正确更新，数据记录完整无遗漏

### 4.2 高并发压力测试

#### TC-018 高并发场景压力测试
- **前置条件**：准备大量测试账单和用户，测试环境支持高并发
- **测试步骤**：
  1. 模拟100+并发用户同时进行主动还款
  2. 监控系统性能指标（响应时间、吞吐量、错误率）
  3. 验证防重机制在高并发下的稳定性
  4. 检查数据一致性和完整性
  5. 验证系统资源使用情况
- **预期结果**：系统稳定运行无崩溃，防重机制正常工作，数据一致性得到保证，响应时间在可接受范围内

#### TC-019 并发边界条件测试
- **前置条件**：用户钱包余额刚好等于账单总额
- **测试步骤**：
  1. 设置钱包余额精确等于待还款总额
  2. 并发发起相同账单的还款请求
  3. 验证边界条件下的防重处理
  4. 检查余额扣减的精确性
  5. 验证不会出现超扣或负余额情况
- **预期结果**：只有一个请求成功，钱包余额刚好扣完，不出现超扣或负余额情况

## 5. 数据一致性验证测试

### 5.1 核心数据表一致性测试

#### TC-020 交易数据表一致性测试
- **前置条件**：完成批量主动还款操作
- **测试步骤**：
  1. 执行多次批量主动还款（包含重复请求）

  4. 检查数据的完整性和准确性
  5. 验证没有孤立或冗余的数据记录
- **预期结果**：`hcb_bill_check_trade_data`表中无重复`biz_order_no`，交易数据与实际扣款一致，订单表和明细表数据关联正确

#### TC-021 钱包流水一致性测试
