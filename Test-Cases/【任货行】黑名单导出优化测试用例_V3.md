# 【任货行】黑名单导出优化测试用例

## 涉及接口与数据表

**定时任务接口**：`/test/etcCardBalanceFlowStatistics`

**数据表**：
| 表名 | 说明 | 存储位置 |
|------|------|----------|
| hcb_etc_card_balance_flow_statistics | 历史钱包流水归档统计表 | MongoDB |
| hcb_etccardbalanceflow_backup | 历史钱包流水归档表 | MongoDB |
| hcb_etccardbalanceflow | MySQL钱包流水表 | MySQL |
| hcb_black_record | 黑名单记录表 | MySQL |
| hcb_hlj_truckuserwallet | 用户钱包表 | MySQL |

**归档统计表字段(hcb_etc_card_balance_flow_statistics)**：
| 字段 | 说明 |
|------|------|
| truckUserWalletId | 用户钱包ID |
| blackTime | 下黑时间 |
| lateFee | 滞纳金 |
| lawyerFee | 律师诉讼费 |
| monthFee | 月管理费 |
| lateFeeRefund | 滞纳金退费 |
| statisticsDate | 统计日期 |
| status | 状态 |

**流水类型(OPERATOR_TYPE)**：
| 类型码 | 说明 | 对应统计字段 |
|--------|------|--------------|
| 2 | 通行扣款 | lateFee(滞纳金) |
| 71 | 起诉服务费扣款 | lawyerFee(律师诉讼费) |
| 146 | 先享月费扣款 | monthFee(月管理费) |
| 145 | 诉讼费退款 | lateFeeRefund(滞纳金退费) |

## 1. 数据归档功能测试

### 1.1 归档任务执行测试

#### TC-001 归档任务正常执行(用户钱包ID本月未统计)
- **前置条件**：
  - 存在黑名单/已注销用户
  - 用户truckUserWalletId本月未归档(statisticsDate非本月)
- **测试步骤**：
  1. 调用接口`/test/etcCardBalanceFlowStatistics`
  2. 从hcb_etccardbalanceflow_backup表统计用户最近下黑时间
  3. 按下黑时间统计各类流水金额
  4. 验证hcb_etc_card_balance_flow_statistics表新增记录
- **预期结果**：
  - 新增记录包含正确的truckUserWalletId
  - lateFee/lawyerFee/monthFee/lateFeeRefund字段金额准确
  - statisticsDate为当前月份

#### TC-002 归档任务跳过(用户钱包ID本月已统计)
- **前置条件**：用户statisticsDate已是本月
- **测试步骤**：
  1. 调用归档接口
  2. 检查该用户是否被跳过
- **预期结果**：跳过已统计用户,统计表数据未重复

#### TC-003 遍历黑名单用户归档
- **前置条件**：hcb_black_record表存在多条黑名单记录

  3. 验证所有符合条件的用户都被处理

### 关键验证MongoDB
```javascript
// 验证统计表数据
db.hcb_etc_card_balance_flow_statistics.findOne({truckUserWalletId: "xxx"})

// 验证流水表统计
db.hcb_etccardbalanceflow_backup.aggregate([
  {$match: {TRUCKUSERWALLET_ID: "xxx"}},
  {$group: {_id: "$OPERATOR_TYPE", total: {$sum: {$toDouble: "$AMOUNT"}}}}
])
```
