# 【任货行、任通行、新发行】二次激活记录优化测试用例

## 1. 任货行导出功能优化测试

### 1.1 导出字段新增验证测试

#### TC-001 年度二次激活次数字段新增测试
- **前置条件**：任货行系统有完整的二次激活记录数据，包含不同年度的激活记录
- **测试步骤**：
  1. 登录任货行系统-二次激活记录管理页面
  2. 选择查询条件，查询包含多个年度的二次激活记录
  3. 点击导出功能按钮
  4. 下载导出的Excel文件
  5. 验证文件中是否包含"年度二次激活次数"字段
  6. 验证该字段的数据准确性
- **预期结果**：导出文件中新增"年度二次激活次数"字段，字段数据准确，显示每个车辆在当前年度的总激活次数

#### TC-002 年度激活次数计算准确性测试
- **前置条件**：准备测试车辆在2024年有不同次数的二次激活记录
- **测试步骤**：
  1. 设置测试车辆A在2024年有5次二次激活记录
  2. 设置测试车辆B在2024年有0次二次激活记录
  3. 设置测试车辆C跨2023-2024年有激活记录
  4. 导出二次激活记录
  5. 验证年度激活次数统计的准确性
- **预期结果**：年度激活次数统计准确，按年度正确计算，不跨年度累加

#### TC-003 导出文件格式完整性测试
- **前置条件**：任货行系统有各类型的二次激活记录数据
- **测试步骤**：
  1. 查询所有类型的二次激活记录
  2. 执行导出操作
  3. 验证导出文件的格式和字段完整性
  4. 检查原有字段是否保持不变
  5. 确认新增字段的位置和命名
- **预期结果**：导出文件格式正确，包含所有原有字段，新增年度激活次数字段位置合理，命名规范

### 1.2 下载中心集成测试

#### TC-004 导出任务提交下载中心测试
- **前置条件**：任货行系统下载中心功能正常运行
- **测试步骤**：
  1. 在二次激活记录页面执行导出操作
  2. 验证导出操作是否提示任务已提交至下载中心
  3. 进入下载中心页面
  4. 查找对应的导出任务记录
  5. 验证任务状态和信息显示
- **预期结果**：导出操作成功提交至下载中心，任务状态正确显示，信息完整准确

#### TC-005 下载中心任务状态跟踪测试
- **前置条件**：下载中心有二次激活记录导出任务
- **测试步骤**：
  1. 提交二次激活记录导出任务
  2. 在下载中心观察任务状态变化（处理中→已完成）
  3. 验证任务完成时间记录
  4. 测试文件下载功能
  5. 验证下载文件的完整性
- **预期结果**：任务状态跟踪准确，完成时间记录正确，文件下载正常，内容完整

#### TC-006 大批量导出下载中心处理测试
- **前置条件**：任货行系统有大量二次激活记录数据（>10000条）
- **测试步骤**：
  1. 设置查询条件获取大量数据
  2. 执行导出操作
  3. 观察下载中心的处理性能
  4. 验证大文件的生成和下载
  5. 检查系统稳定性
- **预期结果**：大批量导出正常处理，下载中心稳定运行，大文件生成成功，下载无异常

## 2. 任通行导出功能新增测试

#### TC-008 系统字典数据配置验证测试
- **前置条件**：已执行提供的SQL插入语句配置系统字典

  3. 验证字典数据的正确性
  4. 检查字典状态是否为启用
  5. 确认字典配置对导出功能的影响
- **预期结果**：系统字典数据配置正确，dict_label为'二次激活记录'，dict_value为'189'，status为'0'（启用状态）

#### TC-009 导出功能完整流程测试
- **前置条件**：任通行系统有二次激活记录测试数据
- **测试步骤**：
  1. 在二次激活记录页面设置查询条件
  2. 执行数据查询操作
  3. 点击导出功能按钮
  4. 等待导出处理完成
  5. 下载并验证导出文件
- **预期结果**：导出功能流程完整，文件生成成功，包含年度二次激活次数字段，数据准确完整

### 2.2 导出数据准确性测试

#### TC-010 导出数据完整性验证测试
- **前置条件**：任通行系统有完整的二次激活记录数据
- **测试步骤**：
  1. 查询特定条件的二次激活记录并记录总数
  2. 执行导出操作
  3. 对比导出文件数据条数与查询结果
  4. 验证关键字段数据的一致性
  5. 检查年度激活次数计算的准确性
- **预期结果**：导出数据完整，条数一致，关键字段数据准确，年度激活次数计算正确

#### TC-011 多年度数据导出测试
- **前置条件**：任通行系统有跨多年度的二次激活记录
- **测试步骤**：
  1. 准备2022-2024年的二次激活记录数据
  2. 设置不同年度查询条件
  3. 分别导出不同年度的数据
  4. 验证年度激活次数字段的正确性
  5. 检查跨年度数据不会混淆
- **预期结果**：不同年度数据导出正确，年度激活次数按对应年度统计，数据不混淆

#### TC-012 特殊数据场景导出测试
- **前置条件**：准备特殊场景的测试数据
- **测试步骤**：
  1. 准备无激活记录的车辆数据
  2. 准备激活次数为0的车辆数据
  3. 准备高频激活的车辆数据（>20次）
  4. 执行导出操作
  5. 验证特殊场景数据的处理
- **预期结果**：特殊场景数据正确处理，无激活记录显示0次，高频激活数据准确统计

## 4. 年度二次激活次数排序功能测试

### 4.1 排序功能基础测试

#### TC-019 升序排序功能测试
- **前置条件**：三个平台都有不同年度激活次数的测试数据
- **测试步骤**：
  1. 在二次激活记录页面查询数据
  2. 点击年度二次激活次数字段的排序功能
  3. 选择升序排序（由低到高）
  4. 验证排序结果的正确性
  5. 检查相同次数数据的排序稳定性
- **预期结果**：升序排序功能正常，数据按激活次数从低到高排列，相同次数数据排序稳定

#### TC-020 降序排序功能测试
- **前置条件**：系统有不同年度激活次数的数据
- **测试步骤**：
  1. 在二次激活记录页面查询数据
  2. 点击年度二次激活次数字段的排序功能
  3. 选择降序排序（由高到低）
  4. 验证排序结果的正确性
  5. 确认最高激活次数排在前面
- **预期结果**：降序排序功能正常，数据按激活次数从高到低排列，排序逻辑正确

#### TC-021 排序状态保持测试
- **前置条件**：已设置年度激活次数排序
- **测试步骤**：

  5. 验证排序状态的保持情况

### 4.2 排序性能和兼容性测试

#### TC-022 大数据量排序性能测试
- **前置条件**：系统有大量二次激活记录数据
- **测试步骤**：

  2. 执行年度激活次数排序操作

- **预期结果**：大数据量排序性能良好，响应时间<5秒，排序结果准确，资源消耗合理

- **测试步骤**：
  1. 在任货行、任通行、新发行分别测试排序功能

  3. 对比三个平台的排序结果
  4. 验证排序交互方式的一致性
  5. 检查排序逻辑的统一性
- **预期结果**：三个平台排序功能一致，交互方式统一，排序逻辑相同，用户体验一致

#### TC-024 排序与其他功能集成测试
- **前置条件**：排序功能与查询、导出等功能并存
- **测试步骤**：

  2. 在排序状态下执行导出操作

  5. 确认排序不影响其他功能
