# BUG-【先享月费】多账单扣款余额不足时所有账单被错误标记为失败

## 【问题描述】
在"先享月费账单扣款"定时任务中，当同一用户有多笔待扣款账单，且在扣款过程中冻结金额不足时，系统会**错误地将所有账单（包括已成功扣款的账单）都标记为失败**，导致严重的数据不一致问题：钱已经扣了，但账单状态显示失败。

**严重等级**：🔴 **P0级 - 数据不一致，影响财务对账**

## 【请求参数】
- **触发接口**：`curl http://127.0.0.1:8082/hcbTimer-business/test/startMonthlyTask.do?type=2`
- **测试用户**：骆志敏（身份证：441622199602075195）

## 【复现步骤】

### 1. 准备测试数据

在`hcb_monthly_paylater_wallet_bill`表中插入3笔待扣款账单：
```sql
-- 账单1：苏ZXM309 (LTK) - 100元 - 状态：待扣款(0)
-- 账单2：苏ZYC900 (XTK) - 100元 - 状态：待扣款(0)
-- 账单3：苏ZCS486 (TXB) - 150元 - 状态：待扣款(0)
-- 总金额：350元
```

在`hcb_hlj_truckuserwallet`表中设置冻结金额：
```sql
UPDATE hcb_hlj_truckuserwallet 
SET FREEZE_AMOUNT = '250.00',           -- 冻结金额250元
    WITHHOLD_FREEZE_AMOUNT = '0.00'     -- 代扣冻结0元
WHERE ID_CODE = '441622199602075195';
-- 可用冻结金额 = 250元（只够扣前2笔，不够扣第3笔）
```

### 2. 执行账单扣款定时任务
```bash
curl http://127.0.0.1:8082/hcbTimer-business/test/startMonthlyTask.do?type=2
```

### 3. 查询扣款结果

查看账单状态：
```sql
SELECT 
    car_num AS 车牌号,
    bill_amount AS 账单金额,
    actual_amount AS 实扣金额,
    bill_status AS 状态码,
    CASE bill_status 
        WHEN 0 THEN '待扣款'
        WHEN 2 THEN '已扣款'
        WHEN 3 THEN '扣款失败'
    END AS 状态说明,
    remark AS 备注
FROM hcb_monthly_paylater_wallet_bill
WHERE id_code = '441622199602075195'
ORDER BY create_time;
```

查看扣款流水：
```sql
SELECT 
    source_id AS 账单ID,
    trade_amount AS 交易金额,
    before_amount AS 扣款前余额,
    surplus_amount AS 扣款后余额
FROM hcb_monthly_paylater_wallet_flow
WHERE id_code = '441622199602075195'
ORDER BY create_time;
```

查看冻结金额变化：
```sql
SELECT FREEZE_AMOUNT AS 冻结金额
FROM hcb_hlj_truckuserwallet 
WHERE ID_CODE = '441622199602075195';
```

### 🐛 数据不一致问题
- ❌ **账单1**：实扣0元，但状态显示"已扣款"
- ❌ **所有账单备注**：都是"冻结金额不足"，但都标记为"已扣款"（矛盾！）
- ❌ **流水记录**：只有2条（150元+100元），说明只扣了2笔，但3笔账单都显示成功
- ❌ **冻结金额**：从250元变成0元，说明钱已扣完，但账单1实扣显示0元

## 【预期结果】

### 账单状态（正确的）
| 车牌号 | 账单金额 | 实扣金额 | 预期状态 | 说明 |
|--------|---------|---------|---------|------|
| 苏ZCS486 | 150元 | 150元 | ✅ 已扣款 | 第1笔，余额充足，扣款成功 |
| 苏ZYC900 | 100元 | 100元 | ✅ 已扣款 | 第2笔，余额充足，扣款成功 |
| 苏ZXM309 | 100元 | 0元 | ❌ 扣款失败 | 第3笔，余额不足，扣款失败 |

### 扣款流水
- 应该有2条扣款成功的流水记录（对应前2笔账单）

### 冻结金额
- 扣款后应该是：50元（250 - 150 - 100）

### 业务逻辑
- 按顺序扣款，能扣多少扣多少
- 已成功扣款的账单状态应该是"已扣款"
- 余额不足无法扣款的账单状态应该是"扣款失败"
- **不应该把已成功的账单标记为失败**

## 【影响范围】
- **数据不一致**：账单状态与实际扣款情况不符
- **财务风险**：已扣款但显示失败，可能导致重复扣款或对账错误
- **用户体验**：用户看到扣款失败，但钱已被扣

## 【修复建议】
请开发排查扣款逻辑，确保：
1. 已成功扣款的账单不会被错误标记为失败
2. 账单状态与实际扣款情况一致
3. 实扣金额字段准确记录
