# BUG-018：钱包2.0代扣模式微信支付成功后日志展示错误

## 【问题描述】
在钱包2.0代扣模式下，用户使用微信支付成功后，未按照需求上传支付成功日志，导致无法追踪真实的支付成功情况。

**需求要求：**
支付成功之后（支付方式：微信、银行卡），上传一次日志（公共接口），日志内容：
```
{
  phone: 上一次手机号,
  req: 请求参数-json字符串,
  source: '快速缴费充值成功'
}
```

## 【请求参数】

**微信支付调起接口：**
```bash
POST /rtx-app/deducrecord/wechatPayDeductRepayment

{
  "userWalletId": 1906909894016225282,
  "wxOpenId": "o9h285Ka3kzx-FTZRUDE9WKt0nvI",
  "billDeducts": [
    {
      "deductBillId": 1985966853188685826,
      "operatorCode": "LTK"
    }
  ]
}
```

## 【复现步骤】
1. 登录任通行小程序
2. 进入"快速缴费充值"页面
3. 选择钱包2.0代扣模式用户（如：黑A113355）
4. 查看欠费账单（1笔，1元）
5. 点击"微信支付"按钮
6. 完成微信支付（支付成功）
7. 返回小程序
8. 查看MongoDB日志（`rtx_app_request_log202511` 集合）

## 【日志记录&截图标注】

**❌ 实际记录的日志：**
```
接口: com.rtx.app.controller.EtcDeducrecordController.selectFailBills
时间: 2025-11-05 15:13:19
用户钱包ID: 1906909894016225282

返回内容:
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "payAmount": 0,
    "billDeducts": []
  },
  "success": true
}
```

**问题：**
- ❌ 记录了 `selectFailBills`（查询欠费账单接口）
- ❌ **缺少支付成功日志**

## 【预期结果】

**应该记录的日志：**
```json
{
  "phone": "13800138000",
  "req": "{\"userWalletId\":1906909894016225282,\"billDeducts\":[{\"deductBillId\":1985966853188685826,\"operatorCode\":\"LTK\"}],\"payResult\":\"SUCCESS\",\"transactionId\":\"4200002450202511050123456789\"}",
  "source": "快速缴费充值成功"
}
```

**具体期望：**
1. 用户完成微信支付后，调用公共日志接口上传支付成功日志
2. 日志包含：phone（手机号）、req（请求参数JSON字符串）、source（固定值：'快速缴费充值成功'）
3. 符合需求要求的日志格式
