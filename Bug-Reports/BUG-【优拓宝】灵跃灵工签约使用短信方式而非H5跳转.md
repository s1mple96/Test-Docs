# BUG-【优拓宝】灵跃灵工签约使用短信方式而非H5跳转

## 1. BUG描述

优拓宝小程序接入灵跃灵工提现渠道后，用户发起签约时，系统沿用了旧的短信签约逻辑（发送短信到用户手机，用户在短信中完成签约），而灵跃灵工的签约方式应为 H5 页面跳转签约。

后端接口已正确返回了 H5 签约链接（`signUrl`），但前端未跳转至该链接，导致用户无法完成签约。

**影响**：
- ✅ 签约接口调用正常，成功返回 H5 签约链接
- ❌ 前端未跳转至 H5 签约链接，用户停留在原页面
- ❌ 用户收到弹窗提示「签约协议有更新，请重新签约。请在收到的短信内完成签约操作。」，与灵跃灵工签约流程不符
- ❌ 用户无法完成签约，无法进行提现操作

---

## 2. 复现步骤

### 前置条件
- 测试环境：192.168.1.60
- 系统：优拓宝小程序（hcb 项目）
- 提现渠道：灵跃灵工（FlexibleWithdraw）
- 测试账号：`431023199911026516`
- `APPLET_ID`：`31b346ec25b54069b9522ba6d88e32ef`（优拓宝推广）

### 操作步骤

**Step 1**：使用测试账号打开优拓宝小程序，进入提现页面，点击「提现」按钮发起签约

```
请求参数：
{
  "fenMiAppCode": "YTB",
  "withdrawCode": "FlexibleWithdraw",
  "appType": "1",
  "merchantNo": "YTB0001",
  "APPLET_ID": "31b346ec25b54069b9522ba6d88e32ef"
}
```

**Step 2**：观察接口响应

```json
{
  "ret": "1",
  "msg": "请求成功!",
  "data": {
    "bankReturnMsg": "签约发起成功",
    "signStatus": 2,
    "signUrl": "https://hlg-zuul.asign.cn/web/short/ZrEVJb424346"
  }
}
```

**Step 3**：观察前端实际行为

前端弹出弹窗，提示内容为：
```
温馨提示
签约协议有更新，请重新签约。
请在收到的短信内完成签约操作。
```

---

## 3. 预期结果

**预期**：
- ✅ 接口返回 `signStatus=2` 且 `signUrl` 有值时，前端应跳转至 `signUrl` 对应的 H5 页面（`https://hlg-zuul.asign.cn/web/short/ZrEVJb424346`），引导用户在 H5 页面完成灵跃灵工签约
- ✅ 用户在 H5 页面完成签约后，返回小程序继续提现流程

---

## 4. 实际结果

**实际**：
- ❌ 前端未跳转至 `signUrl` 的 H5 签约页面
- ❌ 弹出短信签约提示弹窗（「请在收到的短信内完成签约操作」），与灵跃灵工 H5 签约流程不匹配
- ❌ 用户无法通过 H5 完成签约，导致后续提现功能不可用
