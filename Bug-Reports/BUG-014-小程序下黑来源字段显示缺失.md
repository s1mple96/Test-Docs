# BUG-014：小程序下黑来源字段显示缺失

## 【问题描述】
小程序中用户查看黑名单信息时，`black_resoure`（下黑来源）字段存在null、5、6三种值，但页面未进行渲染处理，导致用户看到空白或未定义的显示效果，影响用户体验和信息完整性。

## 【请求参数】
**涉及接口：**
- 用户信息查询接口：`GET /rtx-app/etc/user/getEtcUserByCarNumAndColor`
- 黑名单详情接口：`GET /rtx-app/etc/user/getBlacklistInfo`

**关键字段：**
- `black_resoure`: 下黑来源
  - `null`: 未设置/历史数据
  - `"5"`: 补换下黑
  - `"6"`: 第三方自助下黑

**当前返回示例：**
```json
{
  "code": 200,
  "data": {
    "carNum": "黑A113355",
    "status": "2",
    "blackResoure": null,  // ❌ 页面显示空白
    "blackTime": "2025-10-20 15:30:00"
  }
}
```

## 【复现步骤】
1. **准备测试数据**：
   - 在数据库中设置用户的`black_resoure`为null、5、6
   ```sql
   -- 测试数据准备
   UPDATE rtx_etc_card_user 
   SET status='2', black_resoure=NULL 
   WHERE car_num='测试车牌1';
   
   UPDATE rtx_etc_card_user 
   SET status='2', black_resoure='5' 
   WHERE car_num='测试车牌2';
   
   UPDATE rtx_etc_card_user 
   SET status='2', black_resoure='6' 
   WHERE car_num='测试车牌3';
   ```

2. **小程序端操作**：
   - 登录小程序
   - 进入"我的车辆"页面
   - 查看黑名单车辆详情
   - 点击"下黑原因"或"黑名单详情"

3. **观察显示效果**：
   - `black_resoure = null`: 显示空白或"undefined"
   - `black_resoure = "5"`: 显示"5"或空白
   - `black_resoure = "6"`: 显示"6"或空白

4. **对比其他下黑来源**：
   - `black_resoure = "4"`: ✅ 正确显示"账户欠费"
   - `black_resoure = "2"`: ✅ 正确显示"注销下黑"

## 【预期结果】

### 1. 前端渲染逻辑修复
```javascript
// ✅ 修复后的映射关系
const blackResoureMap = {
  '0': '未下黑',
  '1': '手工下黑',
  '2': '注销下黑',
  '3': '银行卡冻结',
  '4': '账户欠费',
  '5': '补换下黑',           // ✅ 新增
  '6': '第三方自助下黑',     // ✅ 新增
  'null': '历史数据',        // ✅ 新增（兼容历史数据）
};

// 改进的渲染逻辑
getBlackResoureText(value) {
  if (value === null || value === undefined) {
    return '历史数据';  // ✅ 处理null值
  }
  return blackResoureMap[value] || '未知原因';  // ✅ 兜底显示
}
```

### 2. 完整的显示效果
| black_resoure | 期望显示文本 | 说明 |
|--------------|------------|------|
| `null` | 历史数据 | 历史黑名单用户（迁移前的数据） |
| `"0"` | 未下黑 | 正常状态 |
| `"1"` | 手工下黑 | 人工操作下黑 |
| `"2"` | 注销下黑 | 用户注销业务 |
| `"3"` | 银行卡冻结 | 银行卡异常 |
| `"4"` | 账户欠费 | 账单欠费下黑 |
| `"5"` | 补换下黑 | 补换卡业务下黑 |
| `"6"` | 第三方自助下黑 | 第三方平台下黑 |

### 3. 多端一致性
- ✅ **小程序端**：完整显示所有下黑来源
- ✅ **H5端**：保持与小程序一致
- ✅ **APP端**：统一显示逻辑
- ✅ **后台管理端**：已支持（无需修改）

### 4. 用户体验改进
- 用户能够清楚看到下黑原因
- 历史数据有明确标识
- 不再出现空白或undefined

---

**技术分析：**
- **根本原因**：产品遗漏，小程序前端代码未同步后端枚举定义的新增值
- **涉及组件**：小程序页面渲染逻辑、枚举映射配置
- **影响范围**：所有查看黑名单详情的小程序用户

**影响评估：**
- 🟡 **严重级别**：中等（不影响功能，但影响用户体验）
- 📱 **用户影响**：约228个黑名单用户看到空白显示
- 💼 **业务影响**：用户投诉增加，客服工作量加大
- 🐛 **修复难度**：低（前端配置修改）

**修复建议：**
1. 小程序端添加缺失的枚举映射
2. 添加null值的兜底处理
3. 统一前后端枚举定义
4. 建立枚举同步机制，避免后续遗漏
