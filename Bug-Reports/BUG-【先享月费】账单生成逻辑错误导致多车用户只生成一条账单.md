# BUG-【先享月费】账单生成逻辑错误导致多车用户只生成一条账单

## 【问题描述】
先享月费账单生成定时任务执行时，对于拥有多辆车的用户，系统只为该用户生成1条账单，而不是按照需求文档要求的按车辆（wallet_id + car_num）生成多条账单。导致多辆车的失败金额被错误汇总到1条账单中，其他车辆的账单缺失。

## 【请求参数】
```bash
# 账单生成定时任务触发接口
curl http://127.0.0.1:8082/hcbTimer-business/test/startMonthlyTask.do?type=1
```

**风控配置**：
- 任务代码：`Fee_MonthlyPayLater_Wallet`
- 运营商配置：`LTK,XTK`

## 【复现步骤】
1. 准备测试数据：
   - 测试用户：骆志敏（钱包ID: `7b4bfaa4a8de11f084f1141877512abb`）
   - 车辆1：苏ZXM309（LTK运营商），9-11月失败账单，共3个月，300元
   - 车辆2：苏ZYC900（XTK运营商），9-10月失败账单，共2个月，200元

2. 执行账单生成定时任务
   ```bash
   curl http://127.0.0.1:8082/hcbTimer-business/test/startMonthlyTask.do?type=1
   ```

3. 查看生成的先享月费账单
   ```sql
   SELECT * FROM hcb_monthly_paylater_wallet_bill 
   WHERE wallet_id = '7b4bfaa4a8de11f084f1141877512abb';
   ```

4. **实际结果**：只生成1条账单
   - 车牌号：苏ZXM309
   - 运营商：LTK
   - 汇总月数：5个月（错误：把两辆车的月数加在一起）
   - 账单金额：500元（错误：把两辆车的金额加在一起）

## 【日志记录&截图标注】
**日志信息**：
```
2025-11-13 16:25:00 [http-nio-8082-exec-10] ERROR - MonthlyPaylaterWalletBillBizService|monthlyPayLaterBillGenerate|先享月费钱包账单生成，userList size:1
```

**生成的账单数据**：
| 字段 | 值 |
|------|-----|
| 车牌号 | 苏ZXM309 |
| 运营商 | LTK |
| 汇总月数 | 5个月 ❌ |
| 账单金额 | 500元 ❌ |

**原始失败账单数据**：
| 车牌号 | 运营商 | 账单数 | 失败金额 |
|--------|--------|--------|----------|
| 苏ZXM309 | LTK | 3 | 300元 |
| 苏ZYC900 | XTK | 2 | 200元 |

**问题说明**：
1. ❌ 只生成了1条账单，应该生成2条
2. ❌ 苏ZYC900的账单完全缺失
3. ❌ 账单金额500元 = 300元 + 200元（错误地把两辆车的金额加在一起）
4. ❌ 汇总月数5个月 = 3个月 + 2个月（错误地把两辆车的月数加在一起）

## 【预期结果】
按照需求文档 `02-新增定时任务-账单生成.md`，应该按 `wallet_id + car_num` 汇总，生成**2条账单**：

| 车牌号 | 运营商 | 汇总月数 | 账单金额 |
|--------|--------|----------|----------|
| 苏ZXM309 | LTK | 3个月 | 300元 |
| 苏ZYC900 | XTK | 2个月 | 200元 |

**需求依据**：
- 需求文档明确要求：按 `wallet_id + car_num` 进行汇总
- 说明：一个用户名下可能有多辆车，每辆车单独生成一条先享月费账单

**影响范围**：
- ✅ 单车用户：正常（只有1辆车，生成1条账单）
- ❌ 多车用户：异常（有N辆车，只生成1条账单）

**业务影响**：
1. 多车用户的账单缺失（除第1辆车外，其他车辆的账单都没生成）
2. 账单金额错误（多辆车的金额被错误汇总到1条账单）
3. 无法按车辆统计先享月费
4. 扣款时只能扣第1辆车的账单，其他车辆无法扣款
