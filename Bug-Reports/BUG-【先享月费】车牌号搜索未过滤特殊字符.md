# BUG-【先享月费】车牌号搜索未过滤特殊字符导致搜索失败

## 【问题描述】
先享月费账单列表页面，从列表中三击全选复制车牌号后粘贴到搜索框，搜索不到数据。原因是复制时带入了制表符（Tab键），但前端和后端均未对搜索条件进行trim()处理，导致搜索条件不匹配。

## 【请求参数】
```
carNum=浙D11943[Tab]    ← 包含制表符，搜不到
carNum=浙D11943[Space]  ← 包含空格，能搜到
carNum=浙D11943         ← 正确格式
```

实际curl请求中的参数：
```
--data-urlencode 'carNum=浙D11943	'
```
（注意：车牌号后面有制表符）

## 【复现步骤】
1. 登录后台系统（账号：test / 密码：a12345678）
2. 进入"钱包账户管理" → "先享月费账单"
3. 在列表中找到一条车牌号为"浙D11943"的记录
4. 三击全选该车牌号（会选中整个单元格内容，包括制表符）
5. 复制粘贴到搜索框的"车牌号"字段
6. 点击搜索按钮
7. 发现搜索不到数据

**对比测试**：
1. 清空搜索框
2. 手动输入"浙D11943"，然后按一下空格键
3. 点击搜索
4. 能搜索到数据

## 【日志记录&截图标注】
1. 复制的车牌号包含制表符：`浙D11943[Tab]`
2. 请求参数：`carNum=浙D11943	`（末尾有制表符）
3. 数据库中实际存储：`浙D11943`（无空格无制表符）
4. 搜索失败原因：制表符与空格不匹配

## 【预期结果】
1. 前端应该对搜索框输入进行trim()处理，自动去除首尾空白字符（空格、制表符、换行符等）
2. 或后端接收参数时进行trim()处理
3. 无论用户复制时带入什么空白字符，都应该能正常搜索到数据
