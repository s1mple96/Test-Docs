# BUG-010：RTX导出文件"操作类型"字段显示枚举值未转义

## 【问题描述】
在任通行(RTX)系统二次激活记录导出功能中，导出的Excel文件中"操作类型"字段显示的是枚举值（如：SECOND_ACTIVATE、MANUAL_ACTIVATE等），而不是用户友好的中文描述，导致导出数据可读性差，用户无法直观理解操作类型含义。

## 【复现步骤】
1. 登录任通行(RTX)管理后台
2. 进入二次激活记录管理页面
3. 设置查询条件，查询二次激活记录数据
4. 点击"导出"按钮
5. 下载生成的Excel文件
6. 打开Excel文件查看"操作类型"列内容

## 【实际结果】
**导出文件中"操作类型"字段显示：**
- `SECOND_ACTIVATE`
- `MANUAL_ACTIVATE` 
- `AUTO_ACTIVATE`
- `FORCE_ACTIVATE`
- 其他枚举值...

## 【预期结果】
**导出文件中"操作类型"字段应显示：**
- `二次激活`
- `手动激活`
- `自动激活` 
- `强制激活`
- 对应的中文描述...

## 【影响范围】
- **用户体验**：导出数据可读性差，用户需要额外查阅文档理解枚举含义
- **数据分析**：业务人员无法直观进行数据分析和统计
- **报表展示**：导出数据无法直接用于对外报表和汇报

## 【建议修复方案】
1. **数据转义处理**：在导出服务中增加枚举值到中文描述的转换逻辑
2. **字典映射**：建立操作类型枚举与中文描述的映射关系
3. **统一规范**：确保与任货行、新发行系统的操作类型字段显示保持一致

**参考字典映射：**
```java
// 建议的枚举转换映射
Map<String, String> operationTypeMap = new HashMap<>();
operationTypeMap.put("SECOND_ACTIVATE", "二次激活");
operationTypeMap.put("MANUAL_ACTIVATE", "手动激活");
operationTypeMap.put("AUTO_ACTIVATE", "自动激活");
operationTypeMap.put("FORCE_ACTIVATE", "强制激活");
```

## 【测试验证建议】
1. 验证所有操作类型枚举值都有对应的中文转义
2. 确认导出文件中操作类型列显示为中文描述
3. 验证转义后的数据统计和筛选功能正常
4. 检查与其他平台导出格式的一致性
