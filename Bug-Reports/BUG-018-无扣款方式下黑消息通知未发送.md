# BUG-018：无扣款方式下黑消息通知未发送

## 【问题描述】
在任通行(RTX)系统"无扣款方式下黑"定时任务(ID: 231)执行时，用户虽然被成功下黑（status=2, black_resoure=7），但小程序订阅消息未发送，导致用户不知道自己已被限制通行，严重影响用户体验。

**根本原因**：代码中使用了错误的枚举比较方法（`getName()` 而非 `getValue()`），导致状态检查永远失败，消息发送逻辑被错误跳过。

1. 准备测试用户（黑A113355）满足以下条件：
   - 钱包模式：2.0代扣（`wallet_mode=2, wallet_deduction=2`）
   - 用户状态：正常（`status=1`）
   - 无有效银行卡（`rtx_user_bind_bank`中无`bank_status=1`记录）
   - 微信代扣已关闭（`rtx_wechat_withhold_users.withhold_switch=1`）
   - 有微信openId
2. 手动执行定时任务ID 231："用户无扣款方式进行下黑"
3. 查看数据库`rtx_etc_card_user`表，确认用户状态已更新为黑名单

## 【技术分析】

### 问题代码位置
- **文件**：`rtx-quartz/src/main/java/com/rtx/quartz/task/EtcCardUserBlackListTask.java`
- **方法**：`sendWeChatMsg(Long userId)`
- **行号**：第291行

### 错误代码
```java
private void sendWeChatMsg(Long userId) {
    try {
        EtcCardUser etcCardUser = this.etcCardUserService.selectEtccardUserById(userId);
        
        // ❌ 致命错误：使用了 getName() 而不是 getValue()
        if (!EtcCardUserStatusEnum.BLACKLIST.getName().equals(etcCardUser.getStatus())) {
            log.info("sendWeChatMsg-无扣款方式下黑,用户状态非黑名单,不发送微信通知, etcCardUserId={}", userId);
            return;  // ❌ 永远执行return，消息永远不会发送
        }
        // ... 以下代码永远不会执行
    }
}
```

### 逻辑错误分析

**枚举定义**：
```java
// EtcCardUserStatusEnum.java
public enum EtcCardUserStatusEnum {
    BLACKLIST("黑名单", "2"),  // getName() = "黑名单", getValue() = "2"
    
    private final String name;   // 中文名称，用于显示
    private final String value;  // 数据库值，用于存储和比较
}
```

**数据库字段**：
- 类型：`char(1)`
- 存储值：`"2"`（不是`"黑名单"`）

**错误比较逻辑**：
```
数据库查询：etcCardUser.getStatus() = "2"
代码检查：EtcCardUserStatusEnum.BLACKLIST.getName() = "黑名单"
比较结果："黑名单".equals("2") = false
取反后：!false = true
结果：永远执行return，消息永远不发送 ❌
```

**对比表格**：

| 对比项 | 错误代码 | 正确代码 |
|-------|---------|---------|
| 使用的方法 | `getName()` | `getValue()` |
| 返回值 | `"黑名单"` | `"2"` |
| 与数据库比较 | `"黑名单".equals("2")` = false | `"2".equals("2")` = true |
| 取反后 | `!false` = **true** ❌ | `!true` = **false** ✅ |
| 是否return | **永远return** ❌ | **不return** ✅ |
| 消息发送 | **永远不发送** ❌ | **正常发送** ✅ |

## 【建议修复方案】

### 修复代码（一行）
**文件**：`rtx-quartz/src/main/java/com/rtx/quartz/task/EtcCardUserBlackListTask.java`  
**行号**：第291行

```java
// 原代码（错误）
if (!EtcCardUserStatusEnum.BLACKLIST.getName().equals(etcCardUser.getStatus())) {
    return;
}

// 修改为（正确）
if (!EtcCardUserStatusEnum.BLACKLIST.getValue().equals(etcCardUser.getStatus())) {
    return;
}
```

**改动说明**：将 `getName()` 改为 `getValue()`

### 为什么要用 getValue()？
1. **数据库存储的是 `value`**：`char(1)` 类型只能存储1个字符（如`"2"`），无法存储3个汉字（如`"黑名单"`）
2. **性能优化**：单字符比较比多字节字符串比较更快
3. **稳定性**：`value` 是固定编码，不会因文案优化而变化
4. **规范性**：同一文件中其他地方（第218行、第237行）都正确使用了 `getValue()`
