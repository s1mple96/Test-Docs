# BUG-【任货行】绑卡签约成功后页面未跳转

## 1. BUG描述

在任货行小程序"我的银行卡"绑定新银行卡流程中，完成签约验证码校验后，接口返回成功，但页面没有自动跳转，用户停留在验证码输入页面。

**影响**：
- ✅ 签约接口正常返回成功（ret=1）
- ✅ 签约数据正常写入
- ❌ 页面不跳转，用户无法感知签约已成功
- ❌ 用户体验中断，需手动返回

---

## 2. 复现步骤

### 前置条件
- 测试环境: 192.168.1.60
- 系统: 任货行小程序
- 路径: 我的 → 我的银行卡 → 绑定新的银行卡
- 测试账号: truckUserId=6ff0e1869e3149a1ae2e2009ac03a882
- 测试数据: bankSignOrderId=2029108191362727936

### 操作步骤

**Step 1**: 打开任货行小程序，进入 我的 → 我的银行卡 → 绑定新的银行卡

**Step 2**: 填写银行卡信息，点击"发送验证码"，收到短信验证码

**Step 3**: 输入验证码，点击"确认绑定"，调用签约验证接口

```json
{
  "bankSignOrderId": "2029108191362727936",
  "verifyCode": "746983",
  "verifyCodeNo": "2026030400010334",
  "relativeurl": "com.hcb.bank.verifyV2",
  "caller": "chefuAPP",
  "timestamp": 1772612071898,
  "hashcode": "4e890047b5dd816a9f4dc0ca76117b94"
}
```

**Step 4**: 观察接口返回结果与页面行为

接口返回：
```json
{
  "ret": "1",
  "msg": "操作成功",
  "data": "2009463593736851456"
}
```

**Step 5**: 观察页面——停留在验证码输入页，未跳转

---

## 3. 预期结果

**预期**：
- ✅ 接口返回 ret=1（操作成功）后，页面自动跳转到"我的银行卡"列表页或显示"绑卡成功"提示页
- ✅ 新绑定的银行卡出现在银行卡列表中

## 4. 实际结果

**实际**：
- ❌ 接口返回 ret=1、msg=操作成功，签约成功
- ❌ 页面无任何跳转，仍停留在验证码输入页面
- ❌ 无成功提示弹窗，用户无法感知绑卡已成功
