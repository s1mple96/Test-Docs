# BUG-【任通行后台】切换车辆类型后未重新查询用户导致导出数据为空

## 1. BUG描述

在数据导出中心，先查询客车用户信息后发起导出，再切换车辆类型为「货车」直接点击导出，导出任务成功创建但文件内容为空（0条数据）。

**影响**：
- ✅ 客车类型正常查询并导出（通行账单 91 条、钱包流水 318 条）
- ❌ 切换为货车后未重新查询，导出任务创建成功但文件 0 条数据
- ❌ 用户无法感知数据为空的原因，误以为该用户无货车数据

---

## 2. 复现步骤

### 前置条件
- 测试环境：`192.168.1.60`
- 系统：任通行后台 → 数据导出中心
- 测试账号：admin
- 测试数据：身份证 `441622199602075195`（该身份证同时有客车和货车数据）

### 操作步骤

**Step 1**：选择车辆类型「客车」，输入身份证 `441622199602075195`，点击「查询」，查询成功后，选择导出类型「通行账单」+「钱包流水」，点击「导出」

**Step 2**：不刷新页面，直接将车辆类型切换为「货车」，**不重新点击「查询」**，直接点击「导出」

```bash
# 第一次请求（客车，正常）
curl -X POST 'http://788360p9o5.yicp.fun/rtx-admin/center/export/createTask' \
  -H 'Authorization: Bearer eyJhbGciOiJIUzUxMiJ9...' \
  -H 'Content-Type: application/json;charset=UTF-8' \
  -d '{
    "carType": "1",
    "idCard": "441622199602075195",
    "exportTypes": ["1","2"],
    "carNums": [],
    "walletId": "1906909894016225282"
  }'

# 第二次请求（切换货车后未重新查询，walletId 仍为客车的值）
curl -X POST 'http://788360p9o5.yicp.fun/rtx-admin/center/export/createTask' \
  -H 'Authorization: Bearer eyJhbGciOiJIUzUxMiJ9...' \
  -H 'Content-Type: application/json;charset=UTF-8' \
  -d '{
    "carType": "2",
    "idCard": "441622199602075195",
    "exportTypes": ["1","2"],
    "carNums": [],
    "walletId": "1906909894016225282"
  }'
```

**Step 3**：在导出记录列表查看任务结果

**数据查询结果**：

```
# 客车导出任务（正常）
id: 2043963790818787329  type: 1001（客车通行账单）  progress: 当前执行成功数量:91   export_url: 有值
id: 2043963790827175938  type: 1003（客车钱包流水）   progress: 当前执行成功数量:318  export_url: 有值

# 切换货车后的导出任务（异常）
id: 2043963807218515969  type: 1002（货车通行账单）  progress: 当前执行成功数量:0   export_url: 有值（文件为空）
id: 2043963807222710273  type: 1004（货车钱包流水）   progress: 当前执行成功数量:0   export_url: 有值（文件为空）
```

---

## 3. 预期结果

**预期**：
- ✅ 切换车辆类型后，前端应清空上次查询结果，禁止直接点击「导出」
- ✅ 或切换车型后自动重置 walletId，提示用户需重新点击「查询」再导出
- ✅ 货车导出应使用货车对应的 walletId（`7b4bfaa4a8de11f084f1141877512abb`），导出结果应有数据

---

## 4. 实际结果

**实际**：
- ❌ 切换车辆类型后，前端未清空查询状态，直接点击「导出」仍可提交
- ❌ 提交的导出请求中 `walletId` 仍为客车的值（`1906909894016225282`），而非货车的值
- ❌ 货车系统根据该 walletId 查不到任何车辆信息，导出文件 0 条数据
- ❌ 页面无任何错误提示，用户无法得知导出结果为空的原因
