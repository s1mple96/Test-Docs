# BUG-010：小程序非强制协议取消后不再弹出

## 【问题描述】
在小程序登录过程中，当用户面对非强制协议选择"取消"后，成功登录小程序。但是在后续使用其他功能时，该协议框没有再次弹出，导致用户无法重新签署非强制协议。

## 【请求参数】
**涉及接口（推测）：**
- 登录接口：`/api/user/login`
- 协议检查接口：`/api/agreement/check`
- 功能访问接口：`/api/function/*`

**关键参数：**
- 用户ID：已登录用户
- 协议类型：非强制协议 (`is_mandatory = 0`)
- 协议状态：用户选择取消 (`sign_status = 0` 或未签署)

## 【复现步骤】
1. 打开小程序，进入登录页面
2. 输入用户名和密码，点击登录
3. 登录过程中弹出非强制协议弹窗
4. 点击"取消"按钮，拒绝签署协议
5. 成功进入小程序主页
6. 点击任意其他功能模块（如：我的、设置、业务功能等）
7. 观察是否重新弹出协议框

## 【日志记录&截图标注】

**预期行为流程：**
```
用户登录 → 弹出非强制协议 → 用户点击取消 → 登录成功 
→ 使用其他功能 → 应该重新弹出协议框 → 用户可选择签署或继续取消
```

**实际行为流程：**
```
用户登录 → 弹出非强制协议 → 用户点击取消 → 登录成功 
→ 使用其他功能 → ❌ 协议框不再弹出 → 用户无法签署协议
```

**问题现象：**
- ✅ 登录时协议弹窗正常显示
- ✅ 用户可以选择取消
- ✅ 取消后可以正常登录
- ❌ 后续使用功能时协议不再弹出
- ❌ 用户失去重新签署协议的机会

**数据库状态检查：**
```sql
-- 检查用户协议签署状态
SELECT 
    u.user_id,
    u.phone,
    crd.agreement_id,
    crd.is_sign,
    crd.sign_status,
    crd.sign_time
FROM hcb_truckchannel u
LEFT JOIN hcb_channel_resign_detail crd ON u.TRUCKCHANNEL_ID = crd.user_id
WHERE u.PHONE = '用户手机号'
  AND crd.agreement_id = '非强制协议ID';
```

## 【预期结果】
1. **协议持久提醒**：非强制协议被取消后，应该在用户后续使用功能时继续提醒
2. **用户选择权**：用户应该有多次机会选择是否签署非强制协议
3. **功能不受影响**：即使用户持续拒绝签署，也不应影响小程序基本功能使用
4. **状态记录**：系统应该记录用户的拒绝行为，但保持协议未签署状态

**具体期望：**
- 用户取消非强制协议后，在每次使用新功能时都应弹出协议确认
- 用户可以在任何时候选择签署或继续取消
- 系统应提供协议查看和签署的入口（如设置页面）
- 协议状态应正确记录为"未签署"而非"已拒绝"

---

**技术分析：**
- **可能原因**：协议检查逻辑可能将"取消"状态误判为"已处理"
- **涉及逻辑**：小程序协议检查机制、用户状态管理
- **建议检查**：协议弹窗触发条件、用户协议状态判断逻辑

**影响范围：**
- 用户体验：影响用户对协议的知情权和选择权
- 合规风险：可能导致协议签署率降低
- 功能完整性：协议管理功能不完善
