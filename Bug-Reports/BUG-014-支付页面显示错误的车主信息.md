# BUG-014：支付页面显示登录账号信息而非账单车主信息

## 【问题描述】
在小程序中，用户A登录后查询用户B名下车辆的账单并进入支付页面时，页面顶部显示的姓名和车牌号错误地显示为用户A的信息，而非账单所属车辆（用户B）的信息。

## 【复现步骤】
1. 使用账号A登录小程序（如：张三，车牌：苏A12345）
2. 通过账单查询功能，查询账号B名下车辆的账单（如：李四，车牌：苏ZAA340）
3. 选择账号B的账单，点击"去还款"或"立即支付"
4. 进入支付页面 `moduleService/pay/pay`
5. 查看页面顶部显示的姓名和车牌号

## 【实际结果】
支付页面显示：
- 姓名：**张三**（登录账号A的姓名）
- 车牌：**苏A12345**（登录账号A的车牌）

## 【预期结果】
支付页面应该显示：
- 姓名：**李四**（账单车主B的姓名）
- 车牌：**苏ZAA340**（账单对应的车牌）

---

## 【技术分析】

**问题根因：**
支付页面使用了全局状态管理中的 `currentETCInfo`（当前选中的ETC车辆信息），该信息是登录用户自己选中的车辆，而非账单对应的车辆信息。

**相关代码位置：**
- 文件：`moduleService/pay/pay.vue`
- 显示逻辑：
  ```html
  <text class="car-holder">{{currentETCInfo.cardHolder}}</text>
  <text class="car-num">{{currentETCInfo.carNum}}</text>
  ```

**数据来源：**
- 错误来源：`mapGetters(['currentETCInfo'])` - 全局状态，登录用户的当前车辆
- 正确来源：应从账单详情或URL参数中获取对应车辆信息

---

## 【建议修复方案】

### 方案1：从账单详情接口获取车主信息
```javascript
// 在 getRechargeTipsParam() 方法中返回车主信息
data() {
  return {
    billOwnerInfo: {}, // 新增：账单车主信息
  }
},

async getRechargeTipsParam() {
  const params = {};
  if (this.isShowPayFromTravelRepayment) {
    params.idCode = this.currentETCInfo.idCode;
  }
  const res = await this.$HTTP('getRechargeTipsParam', params, 'GET', true);
  
  if (res.code === 200) {
    // 保存账单车主信息
    this.billOwnerInfo = {
      cardHolder: res.data.cardHolder || res.data.userName,
      carNum: res.data.carNum,
      vehicleColor: res.data.vehicleColor
    };
  }
}
```

### 方案2：从页面跳转参数传递车主信息
```javascript
// 账单列表页跳转时传递车主信息
navigateToPay(bill) {
  uni.navigateTo({
    url: `/moduleService/pay/pay?fromPage=travelRepayment&cardHolder=${bill.cardHolder}&carNum=${bill.carNum}&vehicleColor=${bill.vehicleColor}`
  });
}

// 支付页面接收参数
onLoad(options) {
  if (options.cardHolder && options.carNum) {
    this.billOwnerInfo = {
      cardHolder: options.cardHolder,
      carNum: options.carNum,
      vehicleColor: options.vehicleColor
    };
  }
}
```

### 模板修改
```html
<!-- 使用账单车主信息而非登录用户信息 -->
<text class="car-holder">{{billOwnerInfo.cardHolder || currentETCInfo.cardHolder}}</text>
<text class="car-num">{{billOwnerInfo.carNum || currentETCInfo.carNum}}</text>
```

---

**影响范围：** 所有通过查询他人车辆账单进入支付页面的场景

**优先级：** 高（影响用户体验和信息准确性）
