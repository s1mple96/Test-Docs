# BUG-【任通行】代扣验证码错误时提示文案不正确

## 1. BUG描述

调用代扣支付接口 `/withhold/pay` 时，故意输入错误验证码，接口返回的 `msg` 为「签约处理中，请换张银行卡再试！」，与真实错误原因（验证码错误）不符。

**影响**：
- ✅ 接口能正常返回 500 及业务错误
- ❌ 验证码错误场景下提示文案错误，用户无法得知是验证码问题

---

## 2. 复现步骤

### 前置条件
- 接口地址: https://788360p9o5.yicp.fun/rtx-app/withhold/pay
- 系统: 任通行
- 场景: 代扣支付时故意输入错误验证码（如 1234）

### 操作步骤

**Step 1**: 使用错误验证码调用代扣支付接口

```bash
curl -X POST 'https://788360p9o5.yicp.fun/rtx-app/withhold/pay' \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer eyJhbGciOiJIUzUxMiJ9...' \
  -d '{"signOrderId":"2029755439054548993","verifyCode":"1234","verifyCodeNo":"2026030620153592911","bankCardNo":"6222034000046037473","location":1}'
```

**Step 2**: 观察响应

实际返回：
```json
{
  "code": 500,
  "msg": "签约处理中，请换张银行卡再试！",
  "data": null,
  "success": false
}
```

---

## 3. 预期结果

当验证码输入错误时，应明确提示验证码错误，便于用户重试。

**预期**：
- ✅ `msg` 为「验证码错误，请重新输入」或与之含义一致的文案
- ✅ 用户能区分是验证码错误，而非签约状态问题

---

## 4. 实际结果

验证码错误时，接口仍返回与「签约处理中」相关的文案，导致用户误以为是银行卡或签约状态问题。

**实际**：
- ❌ `msg` 为「签约处理中，请换张银行卡再试！」
- ❌ 与真实失败原因（验证码错误）不一致
