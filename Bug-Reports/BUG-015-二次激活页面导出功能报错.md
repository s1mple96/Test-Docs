# BUG-015：新发行系统二次激活页面导出功能报错

## 【问题描述】
在新发行系统的二次激活页面，点击"导出"按钮时，接口返回500错误，提示 `NoSuchMethodError`，导出功能无法正常使用。

## 【复现步骤】
1. 登录新发行系统后台：https://788360p9o5.yicp.fun/report-admin
2. 进入菜单：**二次激活** 页面
3. 点击页面上的"导出"按钮
4. 触发导出接口请求

## 【实际结果】
接口返回错误：
- **接口地址：** `https://788360p9o5.yicp.fun/report-admin/active/record/export?pageNum=1&pageSize=10&orderByParam=0`
- **HTTP状态码：** 500
- **错误信息：**
```json
{
    "code": 500,
    "msg": "Handler dispatch failed; nested exception is java.lang.NoSuchMethodError: com.alibaba.fastjson.JSON.toJSONBytes(Ljava/lang/Object;)[B",
    "data": null,
    "success": false
}
```

## 【预期结果】
导出功能正常执行，返回Excel文件或导出成功的响应。

---

## 【技术分析】

**问题根因：**
接口报 `NoSuchMethodError: com.alibaba.fastjson.JSON.toJSONBytes(Ljava/lang/Object;)[B` 错误，说明代码中调用了 `JSON.toJSONBytes()` 方法，但当前环境中的 fastjson 版本不支持该方法。

**可能的原因：**
1. **Fastjson版本不匹配**：`JSON.toJSONBytes()` 方法是 fastjson 1.2.70+ 版本才引入的API，而当前项目可能使用了更低版本的fastjson
2. **依赖冲突**：项目中可能存在多个不同版本的fastjson依赖，导致运行时加载了低版本的fastjson类
3. **编译和运行环境不一致**：代码编译时使用了高版本fastjson，但运行时类路径中的fastjson是低版本

**涉及的方法：**
```java
// 导出功能中可能使用了此方法
byte[] jsonBytes = JSON.toJSONBytes(exportData);
```

---

## 【建议修复方案】

### 方案1：升级 Fastjson 版本（推荐）
检查项目的 `pom.xml` 或 `build.gradle`，将 fastjson 依赖升级至 1.2.70 或更高版本：

**Maven配置：**
```xml
<dependency>
    <groupId>com.alibaba</groupId>
    <artifactId>fastjson</artifactId>
    <version>1.2.83</version> <!-- 建议使用最新稳定版 -->
</dependency>
```

**Gradle配置：**
```gradle
implementation 'com.alibaba:fastjson:1.2.83'
```

### 方案2：使用兼容的替代方法
如果无法升级fastjson版本，可以修改代码使用兼容的方法：

**修改前：**
```java
byte[] jsonBytes = JSON.toJSONBytes(exportData);
```

**修改后：**
```java
// 方式1：先转成JSON字符串再转字节数组
String jsonString = JSON.toJSONString(exportData);
byte[] jsonBytes = jsonString.getBytes(StandardCharsets.UTF_8);

// 方式2：使用JSONWriter
byte[] jsonBytes = JSON.toJSONString(exportData).getBytes("UTF-8");
```

### 方案3：排查依赖冲突
使用Maven命令检查fastjson依赖情况：
```bash
# Maven项目
mvn dependency:tree | grep fastjson

# 排除冲突的低版本依赖
<dependency>
    <groupId>某个依赖</groupId>
    <artifactId>artifact-name</artifactId>
    <exclusions>
        <exclusion>
            <groupId>com.alibaba</groupId>
            <artifactId>fastjson</artifactId>
        </exclusion>
    </exclusions>
</dependency>
```

## 【附加说明】

### Fastjson版本对照
| 版本 | `toJSONBytes()` 方法 | 发布时间 |
|------|---------------------|----------|
| 1.2.69 及以下 | ❌ 不支持 | 2020年之前 |
| 1.2.70 及以上 | ✅ 支持 | 2020-06-01 |
| 1.2.83（推荐） | ✅ 支持 | 2022-05-31 |

### 安全提示
如果升级fastjson版本，建议使用 **1.2.83** 或更高版本，因为低版本存在多个已知的安全漏洞（如反序列化漏洞）。
