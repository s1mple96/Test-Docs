# BUG-011：已登录用户不弹出新建协议弹窗

## 【问题描述】
当系统管理员新建协议后，已经登录的小程序用户在使用过程中没有收到新协议的弹窗提醒，导致用户无法及时了解和签署新发布的协议内容。

## 【请求参数】
**涉及接口（推测）：**
- 协议创建接口：`/hcbadmin/channelResignTask/save.do`
- 用户协议检查接口：`/api/agreement/checkNew`
- 小程序功能访问接口：`/api/function/*`

**关键参数：**
- 新协议ID：`agreementId`
- 用户ID：已登录用户的 `user_id`
- 协议创建时间：`create_time`
- 用户最后登录时间：`last_login_time`

## 【复现步骤】
1. **准备阶段**：确保有用户已经登录小程序
2. **管理端操作**：
   - 登录管理后台系统
   - 进入"渠道重签任务管理"
   - 点击"新增渠道重签任务"
   - 填写协议信息并保存，创建新的协议任务
3. **小程序端验证**：
   - 保持用户登录状态
   - 在小程序中点击各种功能模块
   - 切换页面、刷新应用
   - 观察是否弹出新建协议的签署提醒

## 【日志记录&截图标注】

**预期行为流程：**
```
管理端创建新协议 → 系统标记所有用户需要签署 → 用户使用小程序功能 
→ 系统检测到新协议 → 弹出协议签署弹窗 → 用户选择签署或取消
```

**实际行为流程：**
```
管理端创建新协议 → 系统标记所有用户需要签署 → 用户使用小程序功能 
→ ❌ 系统未检测到新协议 → ❌ 不弹出协议弹窗 → 用户无感知新协议
```

**问题现象：**
- ✅ 管理端可以正常创建新协议
- ✅ 协议数据正常保存到数据库
- ❌ 已登录用户不会收到新协议弹窗
- ❌ 用户无法及时了解新协议内容
- ❌ 可能影响协议签署覆盖率

**数据库状态检查：**
```sql
-- 检查新创建的协议任务
SELECT 
    crt.id as task_id,
    crt.agreement_id,
    crt.create_time,
    crt.sign_status,
    am.AGREEMENTMOULD_NAME as agreement_name,
    am.AGREEMENTMOULD_VERSION as version
FROM hcb_channel_resign_task crt
LEFT JOIN hcb_agreementmouldmeg am ON crt.agreement_id = am.AGREEMENTMOULDMEG_ID
WHERE crt.create_time >= CURDATE()
ORDER BY crt.create_time DESC;

-- 检查用户是否有对应的签署记录
SELECT 
    crd.user_id,
    crd.resign_task_id,
    crd.is_sign,
    crd.sign_status,
    crd.create_time
FROM hcb_channel_resign_detail crd
WHERE crd.resign_task_id = '新创建的任务ID'
  AND crd.user_id = '测试用户ID';
```

**可能的技术原因：**
1. **协议检查机制缺失**：小程序可能缺少主动检查新协议的机制
2. **推送机制未实现**：系统可能没有向已登录用户推送新协议通知
3. **缓存问题**：用户协议状态可能被缓存，未及时更新
4. **轮询频率不足**：协议检查的频率可能过低

## 【预期结果】
1. **实时感知**：已登录用户应该能够及时感知到新建的协议
2. **主动提醒**：系统应该主动向用户推送新协议签署请求
3. **多触发点**：在用户进行关键操作时都应该检查是否有待签署协议
4. **状态同步**：用户协议状态应该与后台数据实时同步

**具体期望：**
- 管理员创建新协议后，已登录用户在下次操作时应收到弹窗提醒
- 弹窗应该清晰显示新协议的内容和签署要求
- 用户可以选择立即签署、稍后提醒或查看详情
- 系统应该记录用户的选择并合理安排后续提醒

**建议的触发时机：**
- 用户切换功能页面时
- 用户执行重要业务操作前
- 小程序启动或从后台切回前台时
- 定期的后台检查（如每小时一次）

---

**技术分析：**
- **可能原因**：缺少新协议的实时检查机制或推送系统
- **涉及组件**：小程序协议检查模块、用户状态管理、消息推送系统
- **建议方案**：实现轮询检查、WebSocket推送或启动时检查机制

**影响范围：**
- 合规风险：新协议无法及时覆盖所有用户
- 用户体验：用户可能错过重要协议更新
- 业务影响：可能影响协议签署的及时性和覆盖率
