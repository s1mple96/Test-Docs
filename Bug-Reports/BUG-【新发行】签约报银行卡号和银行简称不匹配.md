# BUG-【新发行】签约报银行卡号和银行简称不匹配

## 1. BUG描述

调用代扣签约发短信或签约验证时，接口返回错误提示「银行卡号和银行简称不匹配」，无法完成签约流程。测试卡为浙商银行（卡号 622309 开头）。

**影响**：
- ❌ 申办场景下无法完成绑卡签约（发短信或验证环节报错）
- ❌ 浙商银行测试卡无法走通签约流程

---

## 2. 复现步骤

### 前置条件
- 测试环境: 192.168.1.60
- 系统: 新发行（分米ETC）
- 数据库: fenmi_etc、fenmi_pay
- 测试数据: 吕玲 / 511529196111020062 / 13173765674；浙商银行卡 622309331002366579 或 6223091910000008252
- 支付通道: 平安付合单（bizSource=2 蒙通卡 / bizSource=1 通行宝）

### 操作步骤

**Step 1**: 调用签约发短信接口

```http
POST /fenmi/etc/app/withhold/signSendSms
Content-Type: application/json

{
  "holderIdCode": "511529196111020062",
  "holderName": "吕玲",
  "holderPhoneNumber": "13173765674",
  "bankCardNo": "622309331002366579",
  "bizSource": "2"
}
```

或完成发短信后调用签约验证接口时，同样可能报该错误。

**Step 2**: 观察接口返回

接口返回 500 或业务失败，提示「银行卡号和银行简称不匹配」。

**Step 3**: 查看日志

```bash
tail -n 200 /home/fenmi/fenmi-etc/log.txt
tail -n 200 /home/fenmi/fenmi-pay/log.txt
```

---

## 3. 预期结果

- 根据卡号识别为浙商银行后，与渠道侧银行简称/编码一致，签约发短信或签约验证可正常通过。
- 接口返回 200，可收到短信或签约成功。

**预期**：
- ✅ 浙商卡（622309）可正常签约发短信/验证
- ✅ 不出现「银行卡号和银行简称不匹配」提示

## 4. 实际结果

- 接口返回失败，提示「银行卡号和银行简称不匹配」。

**实际**：
- ❌ 报错文案：银行卡号和银行简称不匹配
- ❌ 浙商银行测试卡（622309 开头）在该提示下无法完成签约
