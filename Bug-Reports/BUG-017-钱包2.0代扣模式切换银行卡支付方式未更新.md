## 🐛 问题描述

**场景**: 钱包2.0代扣模式下，用户在主动还款页面切换银行卡进行支付时

**预期行为**: 
- 用户切换选择银行卡后，支付方式应自动更新为"银行卡支付"
- 或者支付方式选项应清空，提示用户重新选择

**实际行为**:
- 用户切换选择银行卡后，支付方式仍然显示为默认的"微信支付"（勾选状态）
- 导致用户可能误以为会使用微信支付，但实际会使用银行卡支付
- 界面显示与实际支付方式不一致

---

## 📱 复现步骤

### 环境
- **钱包模式**: 钱包2.0
- **扣款方式**: 代扣模式（`wallet_deduction=2`）
- **测试账号**: 18604515585
- **测试车辆**: 黑A113355

### 操作步骤
1. 登录小程序，进入"主动还款"页面
2. 选择需要还款的账单
3. 点击"支付方式"选择区域
4. 默认状态下，"微信支付"处于勾选状态
5. **关键步骤**: 切换选择"银行卡支付"（如选择工商银行、建设银行等）
6. 观察支付方式区域的显示状态

### 实际结果
- 切换银行卡后，"微信支付"的勾选状态**未清除**
- 或者"银行卡支付"的勾选状态**未激活**
- 界面显示与选择不一致

---

## 🎯 影响范围

### 1. 用户体验
- ❌ 界面显示与实际支付方式不一致，造成用户困惑
- ❌ 用户可能误认为使用微信支付，但实际扣款来自银行卡
- ❌ 降低用户对支付功能的信任度

### 2. 业务影响
- ⚠️ 可能导致用户选错支付方式
- ⚠️ 增加客服咨询量（"为什么我选的微信支付却扣的银行卡？"）
- ⚠️ 影响支付成功率（用户可能因界面异常放弃支付）

### 3. 影响用户群体
- **钱包2.0代扣模式用户**（绑定了银行卡和微信支付）
- **主动还款功能用户**
- **预估影响**: 中等（使用主动还款的用户占比约20%-30%）

## 🔍 技术分析

### 可能的原因

#### 1. **前端状态未同步**
```javascript
// 伪代码示例
// 问题代码可能是这样的：
selectPayMethod(method) {
  this.selectedBankCard = method.bankCardId;
  // ❌ 漏掉了清除其他支付方式的选中状态
  // this.selectedPayType = 'BANK_CARD'; // 应该更新支付类型
}
```

**根因**: 切换银行卡时，只更新了银行卡ID，未更新支付方式类型标识

#### 2. **单选/多选逻辑混乱**
- 微信支付和银行卡支付应该是**互斥**的（单选逻辑）
- 但前端可能使用了多个独立的`checkbox`，而非`radio`或统一的状态管理
- 导致可以同时勾选多个支付方式

#### 3. **UI组件状态绑定问题**
```vue
<!-- 可能的问题代码 -->
<checkbox :checked="isWechatPay" @change="selectWechat">微信支付</checkbox>
<checkbox :checked="isBankCardPay" @change="selectBankCard">银行卡支付</checkbox>

<!-- 问题：两个checkbox状态独立，切换时未互斥 -->
```

**应该改为**:
```vue
<radio-group v-model="paymentMethod">
  <radio value="WECHAT">微信支付</radio>
  <radio value="BANK_CARD">银行卡支付</radio>
</radio-group>
```

#### 4. **默认值覆盖问题**
- 页面初始化时，`paymentMethod`默认值为`WECHAT`
- 用户切换银行卡后，未触发`paymentMethod`的更新
- 导致界面仍显示微信支付为选中状态

## ✅ 修复建议

### 方案1: 统一支付方式状态管理（推荐）

**修改逻辑**:
```javascript
// data
data() {
  return {
    paymentMethod: '', // 当前选中的支付方式: 'WECHAT' | 'BANK_CARD'
    selectedBankCardId: null, // 选中的银行卡ID
  }
},

// methods
selectWechatPay() {
  this.paymentMethod = 'WECHAT';
  this.selectedBankCardId = null; // 清除银行卡选择
},

selectBankCard(bankCardId) {
  this.paymentMethod = 'BANK_CARD';
  this.selectedBankCardId = bankCardId;
},

// computed
isWechatPaySelected() {
  return this.paymentMethod === 'WECHAT';
},

isBankCardPaySelected() {
  return this.paymentMethod === 'BANK_CARD';
}
```

**UI绑定**:
```vue
<view class="pay-method-item" @tap="selectWechatPay">
  <image :src="isWechatPaySelected ? 'checked.png' : 'unchecked.png'" />
  <text>微信支付</text>
</view>

<view class="pay-method-item" 
      v-for="card in bankCards" 
      :key="card.id" 
      @tap="selectBankCard(card.id)">
  <image :src="selectedBankCardId === card.id ? 'checked.png' : 'unchecked.png'" />
  <text>{{card.bankName}} ({{card.cardNo}})</text>
</view>
```

### 方案2: 使用原生radio-group组件

```vue
<radio-group @change="onPayMethodChange">
  <label class="pay-method-item">
    <radio value="WECHAT" :checked="paymentMethod === 'WECHAT'" />
    <text>微信支付</text>
  </label>
  
  <label class="pay-method-item" v-for="card in bankCards" :key="card.id">
    <radio :value="'BANK_CARD_' + card.id" :checked="selectedBankCardId === card.id" />
    <text>{{card.bankName}} ({{card.cardNo}})</text>
  </label>
</radio-group>
```

```javascript
onPayMethodChange(e) {
  const value = e.detail.value;
  if (value === 'WECHAT') {
    this.paymentMethod = 'WECHAT';
    this.selectedBankCardId = null;
  } else if (value.startsWith('BANK_CARD_')) {
    this.paymentMethod = 'BANK_CARD';
    this.selectedBankCardId = parseInt(value.replace('BANK_CARD_', ''));
  }
}
```

### 方案3: 添加互斥清除逻辑（临时方案）

```javascript
selectWechatPay() {
  this.isWechatPayChecked = true;
  this.isBankCardPayChecked = false; // ✅ 清除银行卡选中状态
  this.selectedBankCardId = null;
},

selectBankCard(bankCardId) {
  this.isWechatPayChecked = false; // ✅ 清除微信支付选中状态
  this.isBankCardPayChecked = true;
  this.selectedBankCardId = bankCardId;
}
```
