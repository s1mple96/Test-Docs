# BUG-【公共平台】补换订单退款接口500但实际已退款

## 1. BUG描述

公共平台管理后台对补换设备订单执行退款接口 `/fenmi/etc/replaceEquipment/refund` 返回 `500`，响应 `msg=null`；但从业务现象看资金侧已正常退款。

**影响**：
- ✅ 资金侧：退款实际已到账（现象：用户/支付通道显示已退）
- ❌ 接口侧：返回 500 且无错误信息，后台操作方无法确认结果/无法继续后续流程

---

## 2. 复现步骤

### 前置条件
- 测试环境：192.168.1.60
- 系统：公共平台（ETC 管理后台）
- 服务：分米ETC（日志路径见第4部分）
- 测试数据：补换订单ID：`2041820564361367553`

### 操作步骤

**Step 1**：调用补换订单退款接口

```bash
curl -X POST 'https://788360p9o5.yicp.fun/fenmi/etc/replaceEquipment/refund' \
  -H 'Content-Type: application/json;charset=UTF-8' \
  -H 'dmType: admin' \
  -H 'Authorization: Bearer <已打码>' \
  -d '{"id":"2041820564361367553","refundType":"0"}'
```

**Step 2**：查询退款记录（客观描述数据状态）

```sql
SELECT id, refund_trx_no, org_trx_no, refund_status, refund_amount, biz_source_id, org_biz_type, refund_success_time, remark, create_time, update_time
FROM fenmi_etc.etc_refund_record
WHERE biz_source_id='2041820564361367553'
ORDER BY id DESC
LIMIT 5;
```

**数据查询结果**：
```
存在退款记录（示例）：
- id=2041820852963037186
- refund_trx_no=PAYRE30022026040818100100000006
- org_trx_no=ET2026040818093600000000001
- refund_status=2
- refund_amount=12.10
- refund_success_time=NULL
```

---

## 3. 预期结果

- 接口返回成功（`code=200/success=true`），并返回明确的业务提示信息
- 若退款已受理/退款中，也应返回可识别的状态与文案，而不是 500 且 `msg=null`
