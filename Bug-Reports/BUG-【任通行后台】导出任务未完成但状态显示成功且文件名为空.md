# BUG-【任通行后台】导出任务未完成但状态显示成功且文件名为空

## 1. BUG描述

在任通行管理后台“数据导出中心”导出货车通行账单时，导出任务在文件尚未生成/上传完成、下载地址为空的情况下，任务状态已显示为“成功”（status=2），且列表“文件名”为空，导致用户误以为导出已完成但实际无法下载文件。

**影响**：
- ✅ 页面能生成导出任务记录
- ❌ 任务未完成时提前显示“成功”，文件名为空、下载不可用/无效，影响用户判断与后续操作

---

## 2. 复现步骤

### 前置条件
- 测试环境: 192.168.1.60
- 系统: 任通行管理后台（rtx-admin）→ 数据导出中心
- 数据库: rtx
- 账号: admin（或任一有导出权限账号）
- 导出类型: 货车用户通行账单（type=1002）

### 操作步骤

**Step 1**：在数据导出中心选择“货车”用户，完成身份证查询、选择车辆，点击“导出”

**Step 2**：在“导出任务记录”列表观察该任务行
- 现象：状态显示“成功”，但“文件名”为空（截图示例：任务ID `2043892420315344897`）

**Step 3**：查询数据库确认任务仍无文件信息

```sql
SELECT id, status, progress, export_name, export_url, over_time
FROM rtx.rtx_etc_export_record
WHERE id = 2043892420315344897
LIMIT 1;
```

**数据查询结果**（客观描述）：
```
status=2
progress=当前执行成功数量持续增长（如 800000+）
export_name=NULL
export_url=NULL
```

---

## 3. 预期结果

- 导出任务在文件未生成/未上传完成（export_name/export_url 为空）时：
  - 状态应显示“进行中/处理中”
  - 列表文件名可为空，但不应显示“成功”
- 仅当文件生成并上传完成、export_url 有值后，状态才显示“成功”，并可正常下载

---

## 4. 实际结果

- 导出任务在 export_name/export_url 仍为空时，页面状态已显示“成功”（status=2）
- 列表中文件名为空，用户无法确认导出文件，点击下载可能无效或无法下载
