# 🎯 Test-Docs (测试文档库)

这个仓库是我个人的测试工程实践记录，主要用于存储编写的测试用例、Bug 深度分析报告以及相关的技术测试文档。

---

## 📂 目录结构说明

| 文件夹 | 说明 | 内容格式 |
| :--- | :--- | :--- |
| [**Test-Cases**](./Test-Cases/) | **测试用例** | 包含功能测试、接口测试、自动化测试脚本说明等 |
| [**Bug-Reports**](./Bug-Reports/) | **缺陷分析** | 记录复现步骤、根本原因分析（Root Cause）及解决方案 |
| [**Assets**](./Assets/) | **静态资源** | 存放文档中引用的图片、截图、录屏文件 |

---

## 📝 文档规范

### 1. 测试用例 (Test Cases)
每个测试用例模块建议包含：
- **前置条件**
- **测试步骤**
- **预期结果**
- **备注（如优先级、测试数据）**

### 2. Bug 报告 (Bug Reports)
优秀的 Bug 报告应遵循：
- **缺陷描述**：简洁明了的标题
- **环境说明**：系统版本、浏览器内核等
- **复现路径**：1, 2, 3 步骤
- **实际执行结果 vs 预期结果**
- **原因剖析**：从代码或逻辑层面分析为什么会出错
