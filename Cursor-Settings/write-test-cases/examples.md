## 完整工作流程示例

### 场景: 编写任货行账单解析功能测试用例

#### 第0步: 分析需求文档 🔥 最重要

**原始需求**:
```
优化任货行账单解析功能，支持内蒙古MTK和江苏TXB两个运营商的账单文件解析。
- MTK: JSON格式，编码GBK，金额单位为"分"
- TXB: TXT格式（^分隔），编码UTF-8，金额单位为"分"
账单文件通过FTP下载到指定目录，定时任务每小时解析一次。
解析成功后写入数据库，解析失败需记录错误日志。
```

**需求分析结果**:

```markdown
需求点清单:

1. 运营商支持
   ✓ 支持MTK（内蒙古）账单解析
   ✓ 支持TXB（江苏）账单解析
   
2. 文件格式
   ✓ MTK: JSON格式
   ✓ TXB: TXT格式（^分隔）
   
3. 文件编码
   ✓ MTK: GBK编码
   ✓ TXB: UTF-8编码
   
4. 金额单位
   ✓ 两个运营商都是"分"，需转换为"元"
   
5. 文件来源
   ✓ FTP下载到指定目录
   ✓ MTK目录: /home/data/test/MTK/
   ✓ TXB目录: /home/data/test/TXB/
   
6. 解析频率
   ✓ 定时任务每小时执行一次
   
7. 解析结果
   ✓ 解析成功写入数据库
   ✓ 解析失败记录错误日志
   
8. 隐含需求（需通过代码和数据库确认）
   ✓ 分页查询是否有ORDER BY？（防止数据遗漏）
   ✓ 重复文件如何处理？
   ✓ 文件解析中途失败如何处理？
   ✓ 解析记录表结构？
   ✓ 账单数据表结构？
```

**提取测试场景**:

从需求分析可以提取出约15+个测试场景:
- MTK正常解析 (JSON格式、GBK编码、金额单位)

- 文件编码错误 (MTK用UTF-8、TXB用GBK)
- 文件格式错误 (JSON格式错误、TXT分隔符错误)
- 金额单位转换 (分→元)
- 分页查询ORDER BY检查
- 大数据量测试
- 重复文件处理
- 解析失败日志记录
