**测试人员**: 骆志敏

## 1. 需求概述

**需求内容**: 任通行针对长期未登录等风险用户，优化账户/额度相关展示策略；涉及是否展示钱包余额、标签打标与解标、定时任务与账户信息页表现。

**涉及系统**: 任通行（RTX）小程序、RTX 应用与 Quartz 定时任务、MySQL（`rtx` 库）

**涉及模块**:
- 标签：`rtx_tag_relation`（用户维度 `type='2'`、`white_type=0` 等）
- 字典：`sys_dict_data` 中 `not_show_wallet_balance_type` / `not_show_wallet_balance_tag`
- 接口：小程序侧 `isShowWalletBalance`（示例：`/rtx-app/my/isShowWalletBalance?userWalletId=...`）
- 页面：小程序「账户信息」

---

## 2. 业务流程（摘要）

### 2.1 主流程

```
风险策略 / 定时任务 → 维护标签（如 RiskyDormantWallet、InactiveRiskWallet）→ 接口按字典配置判断是否展示余额 → 小程序账户信息页展示或隐藏「账户余额」
```

**验证中涉及的关键点**:
1. 字典配置「不展示余额」所依据的 `tag_key`（当前环境示例：`RiskyDormantWallet`）。
2. 钱包维度在 `rtx_tag_relation` 中是否存在上述标签，与 `isShowWalletBalance` 返回值一致（无命中标签 → `data: true` 表示展示余额；有命中 → 应 `false` 隐藏）。
3. 定时任务区分：与「休眠钱包风险」相关的任务与「非活跃风险标签」类任务用途不同，执行时需核对 jobId/任务名称，避免用错任务。

### 2.2 异常 / 环境类

- 日志查询若 Loki 无数据，需确认 Promtail 等服务是否存活（本轮曾出现 Promtail 未运行、重启后恢复的情况）。
- Windows 本机 curl 触发 Quartz 若遇 JSON 解析等问题，可在测试机对本机 Quartz 端口调用执行接口（含鉴权头）作为替代。

## 3. 业务逻辑（核心）

### 3.1 关键业务规则

| 规则 | 说明 | 验证情况 |
|------|------|----------|
| 展示余额 | 钱包在 `rtx_tag_relation` 中**未**命中字典配置的隐藏类 `tag_key` 时，接口倾向返回展示（示例 `data: true`） | ✅ 与 SQL 核对一致（示例钱包无 RiskyDormantWallet 命中时返回 true） |
| 隐藏余额 | 命中字典配置的 `tag_key`（如 RiskyDormantWallet）时，应不展示余额区域 | 按打标数据验证（打标后应 `false`） |
| 数据维度 | 校验 `source_id` 与 `userWalletId`、以及 `type`/`white_type` 与线上一致 | ✅ 测试中使用约定条件查询 |

### 3.2 数据与配置

**涉及的表/配置**:
- `rtx.sys_dict_data`：`not_show_wallet_balance_type`、`not_show_wallet_balance_tag`
- `rtx.rtx_tag_relation`：按 `source_id`（钱包 ID）、`tag_key`、`type`、`white_type` 核对

**常用核对 SQL（摘录）**:
```sql
-- 字典：哪些 tag 触发隐藏余额
SELECT dict_label, dict_value FROM rtx.sys_dict_data
WHERE dict_type = 'not_show_wallet_balance_type'
  AND dict_label = 'not_show_wallet_balance_tag' LIMIT 5;

-- 某钱包是否命中 RiskyDormantWallet 等
SELECT tag_key, COUNT(*) AS cnt FROM rtx.rtx_tag_relation
WHERE source_id = 1906909894016225282 AND type = '2' AND white_type = 0
  AND tag_key IN ('RiskyDormantWallet','InactiveRiskWallet')
GROUP BY tag_key;
```
