# 【任货行】先享月费湖南和江苏扣款金额、频率迭代 - 测试总结

**测试时间**: 2026-03-17

## 1. 需求概述

**需求内容**: 
针对任货行（HCB）先享月费功能进行迭代，主要涉及湖南（XTK）和江苏（TXB）运营商的扣款金额上调、账单生成与扣款频率调整、以及历史数据初始化。

**涉及系统**: 
- 任货行定时任务 (hcbTimer-business / hcbTimer-bill)
- 任货行接口服务 (hcb-api)

**涉及模块**:
- 账单生成模块: 汇总历史失败月管理费账单
- 账单扣款模块: 从用户钱包执行扣款
- 退款模块: 登录或通行后自动退款
- 历史数据初始化: 1-3月存量数据更新

---

## 2. 业务流程

### 2.1 主流程

```
历史失败账单 → 偶数月生成汇总账单 → 奇数月执行钱包扣款 → 用户登录/通行触发退款
```

**详细步骤**:
1. **账单生成**: 定时任务在偶数月2号触发，查询过去1个月内状态为“失败”的月管理费账单并汇总。
2. **账单扣款**: 定时任务在奇数月1号触发，从用户 `hcb_hlj_truckuserwallet` 的冻结金额中扣除。
3. **退款触发**: 用户登录小程序或产生通行账单后，系统调用 `autoRefund` 接口或通过 MQ 消息触发退款。

## 3. 业务逻辑 (核心)

### 3.1 关键业务规则

| 规则 | 说明 | 验证结果 |

| 退款逻辑 | 登录或通行后，按实扣金额原路退回先享月费钱包 | ✅ 通过 |
| 失效逻辑 | **扣款前发生通行，账单失效且不扣款**（通过标签校验实现） | ✅ 通过 |
| 扣款优先级 | **优先扣除余额(BALANCE)，余额不足再扣冻结(FREEZE)** | ✅ 通过 |
| 扣完即止 | **冻结金额不足30元时，扣完现有金额即标记成功** | ✅ 通过 |

### 3.2 数据流转

**涉及的表**:
- `hcb_complete_wallet_bill`: 原始月管理费账单表（bill_type=1）
- `hcb_monthly_paylater_wallet_bill`: 先享月费汇总账单表
- `hcb_hlj_truckuserwallet`: 用户钱包表（扣除 FREEZE_AMOUNT）
- `hcb_monthly_paylater_wallet_flow`: 先享月费专用流水表
- `hcb_complete_wallet`: 全量钱包表（更新 paid_monthly_amount）

**关键字段**:
- `bill_status`: 0=待扣款, 1=成功, 2=失败, 3=已退款
- `operator_code`: XTK, TXB, LTK, MTK
- `actual_amount`: 实际从钱包扣除的金额

## 5. 关键发现

### 5.1 BUG记录

| BUG编号 | 问题描述 | 严重程度 | 状态 |
|---------|---------|---------|------|
| BUG-001 | `spring-scheme-business.xml` 缺少 `monthlyPaylaterDeductTaskJobTrigger` 配置，导致扣款任务无法自动触发 | 严重 | 待修复 |
| BUG-002 | `TestController` 中 `startMonthlyTask` 接口将 `date` 设为必填，导致不带日期的扣款触发报 400 | 低 | 建议优化 |

### 5.2 代码逻辑确认

**确认的逻辑**:
1. **扣完即止逻辑**:
   - 位置: `MonthlyPaylaterWalletBillBizServiceImpl.java:253`
   - 描述: 当钱包余额不足 30 元但大于 0 时，系统会实扣可用金额（如 20 元）并标记 `bill_status = 1` (成功)。
   - 结论: **符合脑图需求**。业务上要求“扣完现有金额即标记成功”，不再追缴差额。
2. **通行失效逻辑**:
   - 描述: 扣款前若发生通行，用户 `Silent_Wallet` 标签被移除，扣款任务校验标签失败后会将账单标记为失败（status=2），备注“已有通行”。
   - 结论: **符合预期**。通过标签体系实现了账单的动态失效。
