# 【新发行】数据导出中心 - 测试总结

**测试时间**: 2026-04-15  
**测试环境**: 192.168.1.60（测试机）、网关域名 `788360p9o5.yicp.fun`、`fenmi_etc` / `fenmi_etc_admin`

---

## 1. 需求概述

**需求内容**: 新发行后台「数据导出中心」能力验证——按身份证查询用户、提交导出任务、跨年通行账单导出、导出完成/重发邮件通知等。

**涉及系统**: 新发行（fenmi-etc、fenmi-etc-admin、网关）

**涉及模块**:
- 导出中心：`CenterExportController`（`/center/export/*`）
- 异步导出：`fenmi-etc-admin` MQ 消费者 + `CenterDeductRecordExportStrategy`
- 权限：若依 `sys_menu` / `sys_role_menu`、接口 `@RequiresPermissions`
- 邮件：`EXPORT_CENTER_EMAIL_RECEIVE`、`SimpleMailSender`、`resendEmail`

---

## 2. 业务流程（摘要）

1. **查询车辆**：`GET /center/export/queryUserInfo?idCard=...`（需 `center:export:query`）→ 返回 `userIds` 等。  
2. **创建任务**：`POST /center/export/createTask`（需 `center:export:export`）→ 写导出任务、投递 MQ。  
3. **异步导出**：etc-admin 消费任务 → Feign 调 `fenmi-etc` `/deducrecord/exportPagelist` → 写 OSS → 更新任务状态。  
4. **邮件**：配置项 `EXPORT_CENTER_EMAIL_RECEIVE`；重发走 `GET /center/export/resendEmail?taskId=...`（需 `center:export:send`）。

---

## 3. 业务逻辑（核心）

| 规则 | 说明 |
|------|------|
| 导出查询权限 | `queryUserInfo` 使用 **`center:export:query`**，与菜单里「导出/列表/发邮件」三颗按钮的 `perms` 无必然包含关系。 |
| 默认导出时间 | `createTask` 未传日期时：开始日=用户 `createTime` 日期，结束日=当天；`bill_year` 由起止日期的年份拼 `IN` 列表。 |
| 异步失败常见原因 | `ExportPageDataContext`：`count=0` →「查无要导出数据」；或 Feign 调通行记录失败。 |
| 重发邮件成功含义 | `SimpleMailSender` 中 `Transport.send` 未抛异常即接口返回成功；**成功路径几乎无业务日志**。 |
| 自动导出后邮件 | `CenterDeductRecordExportStrategy.afterExport` **未调用发信**，仅打日志；**不能依赖导出完成自动收件**。 |

**涉及库表（测试核对过的）**:
- `fenmi_etc.etc_user`、`fenmi_etc.etc_deduct_record`：造数/导出数据。
- `fenmi_etc_admin.sys_menu`、`fenmi_etc_admin.sys_role_menu`、`fenmi_etc_admin.sys_user_role`：权限与角色。
- `fenmi_etc_admin.sys_config`：`config_key = EXPORT_CENTER_EMAIL_RECEIVE` 为导出收件邮箱。

- **造数**：为车牌 **苏Z9914K**（`etc_user.id=1876452208004415489`）在 `fenmi_etc.etc_deduct_record` 插入跨年 `bill_year`（2024/2025/2026）测试行，`trading_number` 前缀 `FENMI_TEST_Z9914K_` 便于清理。  
- **权限**：角色「测试权限专用」`role_id=6`、用户 **骆志敏** `user_id=24`；核对菜单 1861～1864 与 `center:export:query` 缺失问题。  
- **邮件**：调用 `resendEmail` 返回成功但 163 未收；最终定位为 **直接改库邮箱未刷新缓存**，实际仍发到旧收件地址。  
- **日志**：网关在 `fenmi-gateway/log.txt` 可看到请求转发至 `192.168.1.60:9207`；fenmi-etc 侧因日志策略/二进制日志，`grep` 需 `grep -a`，且发信成功难从应用日志反查。

## 5. 关键发现

### 5.1 权限（已定位）

- **`fenmi_etc_admin.sys_menu` 中不存在 `perms = center:export:query`**，导致「无权限查询用户信息」。  
- 需在菜单增加带 **`center:export:query`** 的按钮（或等价配置），并在 `sys_role_menu` 中授权。

### 5.2 邮件（本轮结论）

- **根因（已解决）**：仅更新 **`sys_config` 中 `EXPORT_CENTER_EMAIL_RECEIVE`**，**未刷新/清理相关配置缓存**，接口仍读到旧值，邮件发到旧邮箱。  
- **产品/实现注意**：`resendEmail` 正文中**未拼接 `exportUrl`**，用户即使收信也可能找不到下载链接；导出完成后的 `afterExport` **未真正发 SMTP**。

### 5.3 日志与运维

- 网关日志路径示例：`/home/fenmi/fenmi-gateway/log.txt`。  
- `fenmi-etc/log.txt` 可能被识别为二进制，检索时用 **`grep -a`**。

---

## 6. 注意事项

1. **改 `EXPORT_CENTER_EMAIL_RECEIVE`**：改库后务必 **刷新配置缓存**（或按你们规范重启/清 Redis/Nacos 等），否则仍按旧邮箱发信。  
2. **验权限**：以 **`getInfo` 返回的 `permissions`** 与 **`sys_menu.perms`** 为准，不要只看菜单树展示。  
3. **验导出数据**：默认日期范围可能不含 `bill_year=2024` 的分片；跨年场景需配合调整导出起止日期。  
4. **清理造数**（测试库）：`DELETE FROM fenmi_etc.etc_deduct_record WHERE trading_number LIKE 'FENMI_TEST_Z9914K_%';`（执行前请再 `SELECT` 确认，勿在生产执行）。

## 7. 测试结论

**整体评估**: ✅ 本轮导出中心相关验证与问题闭环完成（邮件问题根因为配置缓存，已按用户反馈解决）。
