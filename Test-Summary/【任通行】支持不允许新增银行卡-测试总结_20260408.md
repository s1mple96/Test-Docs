# 【任通行】支持不允许新增银行卡 - 测试总结

**测试时间**: 2026-04-08  
**测试环境**: 192.168.1.60（rtx 库）

---

## 1. 需求概述

**需求内容**: 通过字典配置「禁止新增银行卡」的渠道与白名单；当用户名下所有**有效车辆**（正常、黑名单、待激活）均属已配置渠道，且**无白名单车牌**时，任通行小程序隐藏两处新增银行卡入口：扣款设置「新增代扣签约」、账户信息「新增签约银行卡」。

**涉及系统**: 任通行小程序 + 管理后台（字典）

**涉及模块**: 首页/车辆列表接口（`enableBindBankCard`）、字典 `sys_dict_type` / `sys_dict_data`、车主与车辆 `rtx_driver_user_info`、`rtx_etc_card_user`

---

## 2. 业务流程（核心逻辑）

1. 后台维护字典：`enable_bind_bank_card_channel`（键值=渠道 `channel_id`）、`enable_bind_bank_card_whitelist`（键值=车牌，与标签一致）。
2. 接口按 `driver_user_id` 拉取 `rtx_etc_card_user` 全量车辆循环判断：
   - 车牌命中白名单 → **允许绑定**（直接放行）。
   - 非有效状态（未发行、已注销、已解绑等）→ **跳过**，不参与渠道判断。
   - 有效状态（正常/黑名单/待激活）中，任一辆 `channel_id` **不在**禁止列表 → **允许绑定**。
   - 否则（存在有效车且均在禁止渠道、且无白名单命中）→ **不允许绑定**，前端隐藏按钮。

**代码位置（只读参考）**: `IndexServiceImpl.isEnableBindBankCard`、`IndexController` 写入 `EtcUserInfoVO.enableBindBankCard`、`SmsCodeUserDetailsServiceImpl`（登录按 `rtx_driver_user_info.phone` 查用户）。

---

## 3. 数据与配置

| 类型 | 说明 |
|------|------|
| 禁止渠道（测试库示例） | `0000`（字典「默认渠道-广东」等） |
| 白名单（测试库示例） | `蒙A44343`、`蒙A113355`、`黑A113355` 等 |
| 车辆表 | `rtx_etc_card_user`：`driver_user_id`、`car_num`、`channel_id`、`status` |
| 登录表 | `rtx_driver_user_info`：与 `driver_user_id` 对齐方可手机号登录 |

## 4. 测试过程摘要

**用例文档**: `temp/测试用例/【任通行】支持不允许新增银行卡测试用例_V1.0.md`（约 25 条）。

**本轮对话中涉及的准备与支撑**:

- **登录「用户不存在」**: `rtx_etc_card_user` 存在该车但 `rtx_driver_user_info` 缺同 `id` 记录；已补插 `id=1840644592836440065` 与手机 `13877857502`。
- **TC-007**: 库内原无「单车+黑名单+0000」；已用测试号 **18631601378** / `driver_user_id=1447478959637221378`，车牌 **陕F989898**，将 `channel_id` 改为 `0000`（**测完可执行用例文档中的 UPDATE 回滚原渠道**）。

## 5. 关键发现

| 类型 | 说明 |
|------|------|
| 数据一致性 | 存在「有车表 `driver_user_id` 但车主表无对应 `id`」时，短信登录报「登录用户：xxx 用户不存在」，需补 `rtx_driver_user_info`。 |
| 造数注意 | TC-007 类场景需同时满足单车、有效状态为黑名单/正常/待激活之一、渠道在禁止列表、车牌不在白名单。 |

## 6. 注意事项

- 字典 **`dict_value` 与 `rtx_etc_card_user.channel_id`、白名单与 `car_num` 需严格一致**（含大小写、空格）。
- 修改测试车辆 `channel_id` 后，其他依赖该渠道的业务测试前建议 **回滚**。
- 无车用户测 TC-015 时，须使用 **`rtx_driver_user_info` 存在且可登录** 的账号。
